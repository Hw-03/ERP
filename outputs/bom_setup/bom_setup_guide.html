BOM 세팅 도구
사용 설명서

이 도구는 각 완제품·반제품에 필요한 부품 목록(BOM, Bill of Materials)을 처음 세팅할 때 쓰는 오프라인 도구입니다. 인터넷이나 서버 없이 파일만 열면 바로 사용할 수 있고, 작업 내용은 자동으로 저장됩니다. 이 설명서는 처음 사용하시는 분도 화면을 그대로 보고 따라할 수 있도록 만들어졌습니다.

3시나리오별 따라하기

실제 작업 따라하기

자주 마주치는 4가지 상황을 단계별로 안내합니다. 각 단계의 화면을 보면서 그대로 따라해보세요.

시나리오 A
조립 부서의 새 완제품 BOM을 처음부터 만들기
예시: "진단킷 본체 V2" (7-AS-0023) 의 BOM을 처음 등록한다
1
메인 화면 상단에서 A 조립 부서 탭을 클릭
부서 탭이 파란색으로 강조되고, 좌측 목록과 우측 통계가 조립 부서 기준으로 바뀝니다.
2
좌측 검색창에 "진단킷" 입력 → 목록에서 해당 품목 클릭
검색은 품목명 일부만 입력해도 즉시 필터링됩니다. ERP 코드(7-AS-0023)로도 검색됩니다. 클릭하면 편집 화면으로 이동합니다.
3
편집 화면에서 "+ 하위 품목 추가" 클릭하여 우측 패널 열기
패널에는 "추천" 탭이 기본 활성화되어, 이 상위 품목과 관련성이 높은 부품이 점수 순으로 정렬됩니다.
4
부품 한 개씩 클릭 → 소요량 입력 → 추가
예: "형광튜브 35W" 클릭 → 소요량 "2" 입력 → 추가. 왼쪽 BOM 그리드에 즉시 한 줄이 늘어납니다. 5개 부품을 같은 방법으로 추가합니다.
5
우측 상단 "검토 · 완료 →" 클릭
검증 결과가 나타납니다. 모두 ✓ 정상이면 다음 단계로, 오류가 있다면 "← 편집으로" 눌러서 수정합니다.
6
검증 통과 후 "✓ 완료로 표시" 클릭
메인 화면으로 돌아오며, 이 품목이 "완료" 탭으로 이동합니다. 통계 카드의 "완료" 숫자가 +1 됩니다.
시나리오 B
추가한 부품의 소요량 수정하기
예시: "M3 나사" 소요량을 12 → 8로 변경
1
편집 화면 좌측 BOM 그리드에서 해당 행의 "× 12 EA" 버튼 클릭
소요량 부분이 입력 가능한 상태로 바뀝니다.
2
숫자를 "8"로 수정
기존 숫자를 지우고 새 값을 입력합니다.
3
Enter 키를 누르거나 저장 버튼 클릭
즉시 반영되고 자동저장됩니다. 취소하려면 Esc.
시나리오 C
잘못 추가한 부품 삭제하기
1
편집 화면 좌측 BOM 그리드에서 해당 행 끝의 × 버튼 클릭
빨간색 작은 사각형 버튼입니다. 한 번 클릭하면 즉시 삭제됩니다.
2
삭제 후 자동저장 시각이 갱신되는지 확인
상단 앱바의 "자동저장됨 HH:MM:SS" 시각이 변경되었으면 정상 저장된 것입니다.
!
실수로 삭제했어요! 삭제는 즉시 적용되며 되돌리기(Undo) 기능은 없습니다. 같은 부품을 우측 패널에서 다시 추가해주세요.
시나리오 D
완료 처리한 BOM을 다시 수정하기
1
메인 화면 우측 상태 탭에서 "완료" 클릭
완료된 품목 목록이 표시됩니다.
2
수정할 품목 클릭 → 편집 화면으로 이동
기존에 등록한 BOM이 그대로 보입니다.
3
필요한 수정(추가 / 삭제 / 소요량 변경) 후 다시 "검토 · 완료 →"
검증 통과 후 "완료로 표시"를 다시 누르면 갱신됩니다.
4저장 · 백업 · PC 이동

작업 내용을 안전하게 지키는 법

이 도구는 자동저장과 수동 백업, 두 가지 방식으로 작업 내용을 보호합니다. 두 방식의 차이를 정확히 이해하시면 데이터를 잃을 일이 없습니다.

4-1. 자동저장 (현재 PC · 같은 브라우저)

부품을 추가/삭제/수정하거나 완료로 표시할 때마다, 자동으로 이 브라우저의 저장소에 기록됩니다. 실수로 창을 닫거나 컴퓨터를 껐다 켜도 같은 PC·같은 브라우저에서 열면 그대로 복원됩니다.

자동저장은 이 브라우저 안에만 보관됩니다 다른 PC나 다른 브라우저(Chrome ↔ Edge)에서 열면 빈 화면이 됩니다. PC를 옮기려면 다음 항목의 "진행도 저장" 파일이 필요합니다.

5검증 오류 도감

검토 화면에서 만날 수 있는 오류

검토 화면은 6가지 검사를 자동으로 수행합니다. 각 오류가 어떻게 보이고, 왜 발생하며, 어떻게 해결하는지 안내합니다.

⚠
1. 자기 참조 (Self Reference)
⚠
오류
상위 품목(7-AS-0023)이 자기 자신을 하위 부품으로 포함하고 있습니다.
왜 발생하나요?
실수로 상위 품목과 같은 부품을 하위 부품 목록에 추가한 경우. 예: "진단킷 본체 V2"를 만들면서 "진단킷 본체 V2"를 부품으로 넣음.
어떻게 해결하나요?
"← 편집으로" 클릭 후, 잘못 추가된 행의 × 버튼으로 삭제합니다.
⚠
2. 소요량 오류 (Invalid Quantity)
⚠
오류
7-RM-0042의 소요량이 0이거나 음수입니다.
왜 발생하나요?
소요량 입력란에 0이나 -1 같은 값을 입력했거나, 빈 값으로 추가된 경우.
어떻게 해결하나요?
편집 화면에서 해당 행의 소요량을 클릭하여 1 이상의 정수로 수정합니다.
⚠
3. 중복 항목 (Duplicate)
⚠
오류
7-VS-0011 (CMOS 센서 모듈 8MP)이 두 번 이상 추가되어 있습니다.
왜 발생하나요?
실수로 같은 부품을 두 번 클릭하여 추가한 경우.
어떻게 해결하나요?
한 줄만 남기고 나머지는 삭제. 합쳐야 한다면 남긴 한 줄의 소요량을 두 줄의 합으로 수정합니다.
⚠
4. 순환 참조 (Cycle)
⚠
오류
A → B → C → A 형태로 BOM이 서로 물고 도는 구조가 발견되었습니다.
왜 발생하나요?
A의 부품으로 B를 넣고, B의 부품으로 C를 넣었는데, C의 부품으로 다시 A를 넣은 경우. 부품 계층이 무한 반복됩니다.
어떻게 해결하나요?
순환 고리를 끊을 수 있는 한 군데를 골라 해당 부품을 제거합니다. 보통은 가장 마지막에 추가한 관계가 잘못된 경우가 많습니다.
!
5. 공정 순서 역전 (Process Order)
!
경고
조립(AS) 단계 부품이 원자재(RM) 단계 부품의 하위로 들어가 있습니다.
왜 발생하나요?
예: 포장재(P 부서)를 원자재(RM)보다 먼저 사용해야 하는 듯이 BOM이 짜였을 때. 보통은 분류 실수.
어떻게 해결하나요?
실제로 그 순서가 맞다면 무시하고 진행해도 됩니다 (경고일 뿐 차단 아님). 잘못된 분류라면 부품 코드를 다시 확인 후 교체합니다.
!
6. 미완성 BOM (Empty)
!
경고
하위 부품이 한 개도 추가되지 않았습니다.
왜 발생하나요?
상위 품목을 선택만 하고 부품은 추가하지 않은 채 검토 화면으로 이동한 경우.
어떻게 해결하나요?
"← 편집으로" 클릭 후 부품을 1개 이상 추가합니다. 실제로 부품이 없는 단순 원자재라면 굳이 BOM 세팅을 할 필요가 없으니 메인 화면으로 돌아갑니다.
!
오류와 경고의 차이 오류(빨강)는 데이터가 분명히 잘못된 상태 — 가급적 수정하세요. 경고(노랑)는 의심스러운 부분이 있다는 안내 — 실제로 맞다면 그대로 진행해도 됩니다. 둘 다 "완료로 표시"는 가능합니다.
6마우스 · 키보드 동작

알아두면 빠른 동작

Enter
소요량 입력 후 부품 추가
우측 패널에서 부품을 선택하고 소요량을 입력한 뒤 Enter 키를 누르면 즉시 추가됩니다.
Esc
소요량 수정 / 부품 선택 취소
소요량 편집 중 Esc를 누르면 변경 없이 닫힙니다.
클릭
소요량 숫자 → 즉시 편집
BOM 그리드에서 "× 12 EA" 같은 버튼을 클릭하면 그 자리에서 숫자가 입력 모드로 바뀝니다.
×
한 번 클릭으로 행 삭제
확인창 없이 즉시 삭제됩니다. 되돌리기는 없으니 주의하세요.
7자주 묻는 질문

FAQ

Q
실수로 창을 닫았어요. 작업 내용이 날아갔나요?
아닙니다. 자동저장 덕분에 같은 PC·같은 브라우저에서 다시 열면 그대로 복원됩니다. 단 다른 브라우저(Chrome → Edge 등)나 다른 PC에서는 보이지 않으니, 그런 경우엔 "진행도 저장" 파일이 필요합니다.
Q
완료 처리한 BOM을 다시 수정하고 싶어요.
메인 화면 우측 상태 탭에서 "완료"를 클릭하고, 해당 품목을 다시 클릭하면 편집 화면으로 들어갑니다. 수정 후 다시 "완료로 표시"하면 갱신됩니다.
Q
처음부터 모든 작업을 초기화하고 싶어요.
화면에 별도 "초기화" 버튼은 없습니다. 브라우저에서 F12를 눌러 개발자 도구를 열고, Console 탭에서 localStorage.removeItem('bom_setup_v1')을 입력 후 Enter, 그다음 페이지를 새로고침하면 빈 상태로 시작합니다.
Q
인터넷 연결이 없어도 되나요?
네. 품목 데이터 722개가 파일 안에 모두 들어있어 인터넷·서버 없이도 완전히 동작합니다. 처음 열 때 폰트만 잠깐 다르게 보일 수 있는데 기능에는 영향 없습니다.
Q
BOM 내보내기 파일은 어디에 사용하나요?
JSON은 시스템 담당자(개발팀)에게 전달하면 데이터베이스에 일괄 등록됩니다. CSV는 엑셀에서 바로 열어 확인용으로 사용하세요.
Q
품목이 722개인데 새 품목을 추가할 수 있나요?
이 도구에 내장된 품목은 고정된 722개입니다. 새 품목 추가는 HTML 파일 자체를 수정해야 하므로 개발팀에 요청해주세요.
Q
"BOM 내보내기" 버튼이 보이지 않아요.
완료로 표시한 항목이 1건 이상 있어야 이 버튼이 나타납니다. 한 건이라도 완료해보세요.
Q
검토 화면에서 오류가 있어도 "완료로 표시"를 누를 수 있나요?
네, 누를 수는 있습니다. 다만 빨간색 "오류"는 데이터가 분명히 잘못된 상태이므로, 가급적 수정 후 진행하시는 것을 권장합니다. 노란색 "경고"는 실제로 의도한 게 맞다면 무시해도 됩니다.
8문제 해결

이런 증상이 보일 때

!
자동저장 녹색 점이 보이지 않아요
원인
아직 한 번도 변경한 적이 없거나, 브라우저가 시크릿 모드여서 저장이 거부됨.
해결
시크릿/프라이빗 모드를 끄고 일반 모드에서 다시 열어주세요. 부품 한 개를 추가/삭제해보고 시각이 표시되는지 확인합니다.
!
다른 PC에서 열었더니 빈 화면이에요
원인
자동저장은 PC1의 브라우저 안에만 있습니다. PC2에는 데이터가 없으므로 빈 화면이 정상입니다.
해결
PC1에서 미리 "진행도 저장"한 JSON 파일을 PC2로 옮긴 뒤, "진행도 불러오기"로 복원하세요. (4-2 항목 참고)
!
"BOM 내보내기" 버튼이 안 보여요
원인
완료로 표시된 항목이 0건이면 이 버튼은 표시되지 않습니다.
해결
상위 품목 1건을 끝까지 완료해보세요 (편집 → 검토 → 완료로 표시). 그러면 버튼이 나타납니다.
!
진행도 불러오기 후 화면이 깨져 보여요
원인
JSON 파일이 손상되었거나, 다른 도구의 파일을 잘못 불러온 경우.
해결
F5(새로고침)으로 페이지를 다시 열고, 정상적인 bom_draft_*.json 파일을 다시 불러옵니다. 백업 파일이 여러 개 있다면 가장 최근 것을 사용하세요.
!
하위 품목 추가 우측 패널이 좁아 보여요
원인
브라우저 창이 너무 좁아서 패널이 축소된 상태.
해결
브라우저 창을 최대화(F11 또는 우상단 □ 버튼)하면 됩니다. 권장 해상도: 1280×800 이상.
