## 🔴 F04 — 생산 가능 수량 모달

**스크린샷**: `F04-capacity-modal.png`

| 기준 | 판정 | 이탈 내용 |
|------|------|----------|
| V1 글자 잘림 | 🔴 | 하위 확장 행(>)의 첫 열이 텅 빔 — 어떤 AF 모델인지 알 수 없음 |
| V2 줄바꿈 | 🔴 | 첫 열 헤더 "조립완제품·병목"이 한 글자씩 세로로 쌓임 (열 너비 거의 0) |
| V3 여백 | 🔴 | 4열 테이블이 393px에 모두 들어가지 않아 첫 열이 밀림 |
| V4 정렬 | ⚠️ | 숫자 열은 정렬됐으나 첫 열 이름이 보이지 않아 무의미 |
| V6 모달 비례 | 🔴 | 테이블 구조 자체가 모바일에 맞지 않음 |
| V8 전체 | 🔴 | 가장 이상한 화면 — 사용자가 직접 지목한 문제 |

**원인**: 4열(모델명 + 출하준비 + 빠른조립 + 총생산) 테이블이 393px에 억지로 들어가면서
숫자 3열이 고정 너비를 차지하고 첫 열(모델명)이 잔여 너비를 받아 사실상 0이 됨.

**개선 방향**:
- 모델별 카드 레이아웃으로 전환: 모델명을 카드 헤더로, 3개 수량을 가로 3칸으로 배치
- 또는 테이블 헤더 열을 고정 숨김 없이 좌우 스크롤(`overflow-x-auto`) + 첫 열 sticky
- 파일: `frontend/app/mes/_components/CapacityDetailModal.tsx` (또는 모바일 전용 뷰)

## 🟡 F05 / F10 — 입출고 Step1 · 불량 허브 빈 공간

**스크린샷**: `F05-warehouse-step1.png`, `F10-defect-hub.png`

| 기준 | 판정 | 이탈 내용 |
|------|------|----------|
| V5 빈 화면 | 🟡 | 카드 2–3개 아래 화면 절반이 텅 빔 — 뭔가 빠진 것처럼 보임 |

**원인**: 선택지가 2–3개뿐인 단순 허브 화면. 콘텐츠 영역이 `flex-1 overflow-auto`라
하단이 채워지지 않은 채 노출됨.

**개선 방향**:
- 카드를 `flex-1`로 늘려 화면 높이를 채우거나
- 카드 아래에 안내 텍스트/힌트를 추가해 빈 공간을 채움
- 또는 카드를 화면 중앙에 수직 배치(`justify-center`)해 의도적 여백으로 처리
