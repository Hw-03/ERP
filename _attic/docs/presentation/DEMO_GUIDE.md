# DEXCOWIN MES — 시연 기능 정리 (발표자용)

> 라이브 시연 때 화면을 돌며 **무엇을 보여주고 설명할지** 빠짐없이 짚기 위한 참조 문서.
> 데스크탑 전용 (모바일 시연 안 함). 진입: 브라우저 → `/legacy` → **PIN 로그인** → 좌측 사이드바, URL `?tab=…`.
> 발표 슬라이드(`index.html`)의 시연 체크리스트 6항목은 이 문서의 **빠른 동선**과 1:1 대응.

도메인 용어 (틀리면 안 됨):
- 공정 단계: **R 원자재 · A 중간공정 · F 공정완료** (코드 뒷글자)
- 부서 코드 앞글자: 조립 A · 고압 H · 진공 V · 튜닝 N · 튜브 T · 출하 P
- 제품 모델: DX3000(3) · COCOON(7) · SOLO(8) · ADX4000W(4) · ADX6000FB(6)
- 품목코드 = `모델-공정(부서+단계)-일련번호` 예: `3-HF-0001`

---

## ⏱ 빠른 시연 동선 (6분 · 슬라이드 9 체크리스트와 1:1)

발표 중 alt+tab으로 MES 이동 → 아래 순서대로 → 슬라이드 복귀 시 `T`키로 체크.

| # | 슬라이드 항목 | MES 동작 | 메뉴 |
|---|---|---|---|
| 1 | PIN 로그인 | PIN 입력 → 대시보드 진입 | 로그인 |
| 2 | 창고 입출고 요청 | 입출고 → 요청 작성 → "창고 입출고" → 창고→부서 → 품목·수량 → 제출 | §2 |
| 3 | 창고 정/부 승인 처리 | 입출고 → "창고 승인함" → 요청 승인(PIN) | §2 |
| 4 | 부서 재고 현황 확인 | 대시보드 → 부서 필터 → 우측 품목 상세의 부서별 재고 | §1 |
| 5 | BOM 입고 → 하위 자동출고 | 입출고 → "부서 입출고" → 생산 → 완제품 입고 시 하위 자동 출고 확인 | §2 |
| 6 | 불량 등록 | 입출고 → "불량" → 불량 격리 → 제출 | §2 |

> 시연 계정: **조립 부서 + 생산부 정/부(이필욱·김건호)** 로 진행. (부서 결재가 같은-부서로 스코프돼 있어 타 부서 요청은 시연 막힘 — 발표 전 라우팅 수정 전제. 안 됐으면 조립으로만.)

## §1. 대시보드  `?tab=dashboard`

**들어가는 법**: 로그인 직후 기본 화면. 좌측 사이드바 "대시보드".
**화면 구성**: 상단 KPI 그룹 / 필터(접이식) / 검색 / 재고 테이블 / 행 클릭 시 우측 상세 패널.

**보여줄 기능 체크리스트**:
- [ ] KPI 그룹 클릭 — **전체 / 재고 많음 / 재고 적음 / 재고 없음** 전환 (클릭 시 자동 필터)
- [ ] "생산 가능 용량" 패널 → 클릭 시 부서별 용량 모달
- [ ] 필터 펼치기 — **부서**(다중 체크, "창고" 포함) / **모델 분류** / **공정 단계(R·A·F)**, "전체 해제"
- [ ] 검색창 — 품목명·품목코드로 실시간 필터
- [ ] 재고 테이블 행 클릭 → 우측 패널
- [ ] 우측 패널: 승인 대기 수량 / 사용 가능 재고 / **부서별 재고 위치(정상·불량 탭)** / 최근 거래 5건
- [ ] 우측 패널 "입출고 진행" → 이 품목 들고 입출고 탭으로 이동
- [ ] 패널 닫기 — X / ESC / 행 재클릭

**시연 멘트**: "엑셀 안 열어도 어느 부서에 몇 개 있는지 한 화면에서 보입니다. 빨강은 재고 부족."
**대표 흐름**: 재고 적음 KPI 클릭 → 품목 행 클릭 → 우측에서 부서별 분포 보여주기.
**주의**: 샘플 데이터 — 수치 자체보다 "한눈에 보인다"를 강조.

## §2. 입출고  `?tab=warehouse`

**들어가는 법**: 사이드바 "입출고". 상단 섹션 탭(권한별): **요청 작성 / 작업 중 / 내 요청 / 창고 승인함 / 부서 승인함**.
**화면 구성**: 요청 작성은 5단계 위저드. 승인함은 대기 요청 목록.

### 2-A. 요청 작성 — 5단계 위저드

- [ ] **1단계 작업 유형** (큰 카드 4개):
  - 원자재 입고 — 발주 품목 입고 *(창고 정/부만 보임)*
  - 창고 입출고 — 창고↔부서
  - 부서 입출고 — 부서 내 작업
  - 불량 — 불량 재고 격리
- [ ] **2단계 세부 작업 + 부서 지정**:
  - 원자재 입고 → 외부 입고
  - 창고 입출고 → **창고→부서**(BOM 1단계 하위 자동 포함) / **부서→창고**(반납 하위만)
  - 부서 입출고 → **생산**(하위 출고+결과 입고) / **분해**(상위 출고+회수 입고) / 수량보정 입고 / 수량보정 출고
  - 불량 → **불량 격리**(창고 승인) / **공급처 반품**
- [ ] **3단계 품목 선택(장바구니)**: 검색 → 선택 → 수량(±) → 우측 장바구니. BOM 자동 전개 시 "상위 1 + 하위 N" 표시. 라인 제외(체크 해제) + 제외 사유.
- [ ] **4단계 확인**: 작업 요약 + 필요 승인(즉시/창고/부서) + **PIN 입력** + 비고
- [ ] **5단계 결과**: 요청 번호 + 상태(승인 대기 / 즉시 완료)

### 2-B. 작업 중 / 내 요청
- [ ] **작업 중** — 임시 저장 요청 카드: 이어하기 / 삭제 / 상세
- [ ] **내 요청** — 상태 칩(대기·승인·반려·완료) 필터, 상세 보기, 반려건 삭제

### 2-C. 창고 승인함 / 부서 승인함
- [ ] (창고 정/부 권한) **창고 승인함** — 요청 행: **승인**(PIN) / **반려**(사유+PIN) / 상세
- [ ] (부서 정/부 권한) **부서 승인함** — 본인 부서 요청만, 승인/반려 동일
- [ ] 배치 처리 — 묶음 요청은 batch 단위로 상세 표시

**시연 멘트**: "대장·엑셀 안 쓰고 요청만 올리면 끝. 승인해야 재고가 움직입니다."
**대표 흐름(시연 핵심)**: 창고 입출고 → 창고→부서(조립) → 품목+수량 → 제출 → 창고 승인함에서 같은 요청 승인 → 재고 반영 확인.
**BOM 자동출고 보여주기**: 부서 입출고 → 생산 → 완제품(예 `3-HF-0001`) 선택 → 하위 자재가 장바구니에 자동(투입 자재 태그) → 제출하면 완제품 입고+하위 출고 동시.
**승인 규칙(설명용)**: 창고↔부서·불량격리 = **창고 승인**, BOM 아닌 낱개 조정 = **부서(생산부 정/부) 승인**, 그 외 = 즉시. 동시 결재 없음(둘 중 하나).
**주의**: ① 원자재 입고 카드는 창고 정/부 계정에서만 보임 ② 부서 승인은 같은-부서만 가능(시연은 조립/생산부 정·부) ③ 비가역(공급처 반품·불량 격리)은 경고 문구 뜸 — 천천히.

## §3. 입출고 내역  `?tab=history`

**들어가는 법**: 사이드바 "입출고 내역".
**화면 구성**: 상단 통계 바 / 기간 칩 / 필터·달력(접이식) / 거래 테이블 / 행 클릭 시 우측 상세.

**보여줄 기능 체크리스트**:
- [ ] 기간 칩 — 이번 달 / 지난 3·6개월 / 올해 / 전체
- [ ] 필터 — 부서(거래 수 표시) / 모델 분류 / 거래 종류, "전체 해제"
- [ ] 달력 — 거래 있는 날 배지, 날짜 클릭 시 그 날짜만
- [ ] 검색 — 품목·메모
- [ ] 거래 테이블 행 클릭 → 우측 상세 (거래 정보 + 라인)
- [ ] **배치 그룹** — 묶음 거래 접기/펼치기, 그룹 헤더 클릭 시 배치 상세
- [ ] **BOM 드릴** — 상위 거래 → 하위 라인 파고들기(뒤로 가능)
- [ ] (권한) 거래 **수정** + **정정 기록 보기**
- [ ] 더보기 — 추가 로드

**시연 멘트**: "누가·언제·왜 움직였는지 전부 남습니다. 임의로 숫자 못 맞춥니다."
**대표 흐름**: 방금 시연한 입출고 건을 내역에서 찾아 "기록이 남았다" 보여주기.
**주의**: 정정 기록은 수정 권한 계정에서만.

## §4. 주간보고  `?tab=weekly`

**들어가는 법**: 사이드바 "주간보고".
**보여줄 기능 체크리스트**:
- [ ] 주 선택(월요일 기준)
- [ ] KPI — 총 생산량 / 최다 생산 모델 / 활성 부서
- [ ] **생산 매트릭스** — 행=제품 모델, 열=부서, 셀=생산 수량(진하기)
- [ ] 부서별 추이 / 모델별 비교
- [ ] CSV 다운로드 / 인쇄

**시연 멘트**: "주 단위로 어느 모델이 어느 부서에서 얼마나 나왔는지 자동 집계."
**주의**: 샘플 데이터라 수치 비어 있을 수 있음 — 기능(자동 집계)만 보여주고 넘어가기.

## §5. 관리  `?tab=admin`  *(PIN 게이트)*

**들어가는 법**: 사이드바 하단 "관리" → **PIN 입력**. 진입 후 좌측 7섹션.
**섹션 순서**: 모델 관리 / 품목 관리 / 직원 관리 / 부서 관리 / BOM 관리 / 내보내기 / 설정.

- [ ] **모델 관리** — 모델 추가·수정, 부서 할당(체크), 활성/비활성
- [ ] **품목 관리** — 품목 검색·테이블, 우측 상세(코드·이름·카테고리·단가·공급처) 편집/저장, 추가/삭제, "BOM 편집" 진입
- [ ] **직원 관리** — 직원 상세, **창고 권한(없음/정/부)** · **부서 권한(없음/정/부)** 라디오, 활성 토글, PIN
- [ ] **부서 관리** — 부서 추가, 색상 피커, 활성/비활성, 삭제(위험)
- [ ] **BOM 관리(워크벤치)** — 부서 탭 → 부모 품목 선택 → 직계 자식 목록, 자식 추가/수량 편집, 완료 상태 토글, **사용처(Where Used) 모드**(역참조)
- [ ] **내보내기** — 품목·직원·BOM·거래 CSV
- [ ] **설정** — 관리자 PIN 변경 / 데이터베이스 리셋(위험 — 시연 시 누르지 말 것)

**시연 멘트**: "부서·직원·품목·BOM을 여기서 직접 관리. 권한도 여기서 정·부 지정."
**대표 흐름(30초)**: 부서 관리 한 번 → 직원 관리에서 권한 라디오 → BOM 워크벤치에서 부모-자식 한 번.
**주의**: ⚠️ **설정 > 데이터베이스 리셋 절대 클릭 금지**. BOM 워크벤치 부서 탭은 R(원자재) 제외 부모만.
