01 · 배경

왜 이 개선안이 필요한가

표준 BOM은 필요하지만 실제 오더는 표준과 항상 같지 않습니다.

현재 발생하는 문제

기존 BOM을 매번 고치면 과거 출하 구성을 알 수 없고, 가능한 조합을 모두 미리 만들면 관리 대상이 끝없이 늘어납니다.

배터리 2개 → 이번 주문은 1개실드 포함 → 이번 주문은 미포함국가별 충전기·전원 코드 변경별도 악세사리·카톤 포장 추가

정리해야 할 기준

표준 구성은 그대로 재사용하고, 변경 주문은 그 오더만의 PA·PF와 BOM으로 남겨 실제 출하 내용을 추적합니다.

변경 없음 → 기존 PA·PF 사용PA 내부 변경 → 오더 전용 PA·PF 생성PF 단계만 변경 → 오더 전용 PF 생성출하 완료 후에도 BOM과 이력 유지
핵심 질문 · “이번에 실제로 무엇을 넣어 출하했는가?”를 BOM과 재고 이력으로 함께 남깁니다.
