## 1. 분석 배경

### 1.1 기존 MES 시스템 개요

덱스코윈에서 2021~2022년 도입한 MES (Manufacturing Execution System). 3개 프로그램으로 구성:

- **MES** — 관리 프로그램 (PC, 사무직)
- **POP** — 작업자 프로그램 (현장 PC, 생산 작업자)
- **PDA** — 창고관리 프로그램 (핸디 단말, 창고 담당자)

### 1.2 기존 MES의 핵심 문제 (현장 담당자 피드백)

1. **UX 부담**: 그리드 셀 색상으로 "필수입력 / 수정불가 / 등록후수정가능"을 구분 → 사용자가 규칙을 외워야 함
2. **C/S 구조의 한계**: 서버가 있어도 각 PC에 .msi 설치 필요, 모바일 미지원, Windows 전용
3. **8클릭 등록 흐름**: 메뉴선택 → 조회 → 작업지시서 → 시작 → 종료 → 수량입력 → 저장 → 확인
4. **2차 피해**: 작업자가 실시간 입력을 안 하고 퇴근 직전 몰아서 입력 → **실시간 재고/공정 데이터가 사실상 가짜**

### 1.3 현재 MES가 맡은 미션

위 3가지 문제를 해결하기 위해 재구축. 따라서 단순 "기능 따라잡기"가 아니라 **"같은 일을 더 빠르고 마찰 없이"** 가 핵심 가치 제안.

### 4.3 핵심 함의

1. **"넓이"로는 MES를 절대 못 따라잡음**.
   - 78개 항목 중 우리가 0점인 것이 약 30개. 다 만들면 1년+ 작업.
   - "MES와 동등 기능" 을 목표로 잡는 순간 본 프로젝트는 실패.

2. **차별화 전략은 "깊이 + UX"**.
   - 자재·재고 코어는 이미 MES 수준 (호각). 여기에 모바일 1탭 UX를 더하면 명확히 우월.
   - "8 클릭 → 1 스캔" 이 우리 가치 제안의 핵심.

3. **품질·설비는 다 만들지 말고 "회사가 실제 쓰는 것만"**.
   - 매뉴얼에 있다고 현장에서 다 쓰는 게 아님 (PPT엔 다 들어있어도).
   - 담당자 확인 후 진짜 쓰는 기능만 골라 구현 권장.

4. **POP을 만들 거면 "흐름"부터 설계**.
   - 반제품바코드(#29) → 공정순서 검증(#34) → 자재투입(#31) → 양품/불량 보고(#32/33)는 한 세트.
   - 단편적으로 한두 개씩 만들면 가치 안 나옴. 전체 흐름 한 번에 하거나 안 하거나.
