# 창고 박스별 재고 자동 차감 설계

작성: 2026-06-22  
상태: 설계 확정 / 구현 대기

---

## 배경 및 문제

현재 출고하면 `Inventory.warehouse_qty`(창고 총재고)만 자동 차감되고,
창고 지도의 `WarehouseBoxItem.quantity`(박스별 수량)는 입출고와 무관하다.
관리자가 수동 편집할 때만 바뀌기 때문에 시간이 지나면 실물과 어긋난다.

---

## 설계 원칙

- **진실의 원천 = `warehouse_qty`(창고 총재고)**. 실물과 다르면 실사로 덮어씀.
- **박스 합 = 총재고**여야 한다. 어긋나면 경고.
- 부서 재고는 박스 개념 없음. 창고 박스만 대상.

---

## 자동 차감 규칙 (R1~R10)

### R1. 차감 순서

한 품목이 든 박스들을 아래 키로 정렬해 순서대로 차감한다.

```
층↓(높은 층 먼저) → 줄↑(row_no A→) → 자리↑(jari_index 왼→오) → 스택↓(stack_order 위 박스 먼저)
```

정면도 예시 (3줄 × 2층, 칸당 자리 3개):

```
        줄A       줄B       줄C
2층 │ ①②③    ④⑤⑥    ⑦⑧⑨   ← 여기서 시작
1층 │ ⑩⑪⑫    ⑬⑭⑮    ⑯⑰⑱
      (자리 왼·중·오)
```

같은 자리에 박스가 쌓여 있으면 위 박스(stack_order 높은 것)부터 차감.

> 현장 약속: 작업자도 실제로 이 순서대로 꺼낸다 (위·왼쪽 박스부터).  
> 약속이 지켜지지 않으면 실사로 보정(R9).

### R2. 차감 동작

순서상 첫 번째 비어있지 않은 박스부터 깎는다.  
한 박스가 0이 되면 다음 박스로 넘어간다.

예) 20 / 100 / 100 에서 50 출고 → 0 / 70 / 100

### R3. 빈 박스

수량이 0인 박스는 자리를 유지한다(삭제하지 않음). 차감 시 건너뜀.

### R4. 입고 배치

입고 후 담당자가 관리자 탭에서 앵글·자리·수량을 수동 지정한다.  
배치 전까지는 R10 불일치 경고가 뜬다.

### R5. 출고량 > 박스 합

배치된 박스 수량 합보다 출고량이 많으면 **출고를 막는다**.

> 오류 메시지: "배치 수량 부족 — 관리자 탭에서 먼저 박스에 배치해주세요."

### R6. 출고 취소

취소 시 차감했던 박스·수량을 그대로 원복한다.  
→ 트랜잭션에 **박스 단위 차감 내역**을 기록해야 한다(현재 `inventory_effect`는 warehouse/부서 단위까지만 — 박스 단위 추가 필요).

### R7. 한 박스에 여러 품목

한 박스에 여러 품목이 담길 수 있다.  
"왼쪽 순서" 규칙은 **품목별로 독립적으로** 적용된다.

### R8. 앵글 범위

한 품목은 **한 앵글 안에만** 있다고 가정한다. 앵글 간 정렬은 없음.  
(향후 여러 앵글에 걸치게 되면 `display_order` 순서 추가 필요)

### R9. 실사 보정

실물을 세어 시스템과 다르면:

1. 관리자 탭에서 해당 박스 수량을 실측값으로 직접 수정.
2. 박스 합이 바뀌면 `warehouse_qty`와 차이가 생겨 R10 경고 발생.
3. 입출고 화면에서 차이만큼 **재고 조정(ADJUST)** 거래로 기록(메모 필수).  
   - 원자재(R 코드)는 주간보고 범위 밖이라 통계 영향 없음(확인됨).

### R10. 불일치 경고

`박스 합 ≠ warehouse_qty`이면 창고 지도·관리자 탭에 경고를 표시한다.  
이미 `/api/warehouse-map/reconcile` 엔드포인트가 over/under 감지를 구현하고 있음.

## UX 설계

### 구조 변경 — 관리자 탭 창고 섹션 제거

현재 관리자 탭의 "창고" 섹션(`AdminWarehouseStructureSection`, `AdminWarehousePlacementSection`)을  
**창고 지도 탭으로 통합**한다. 관리자 탭에서 창고 관련 메뉴는 삭제.

### 권한

창고 지도 탭의 **편집 기능 전체**는 `warehouse_role = primary | deputy`인 사용자에게만 노출된다.  
일반 직원은 지금과 동일하게 읽기 전용 지도만 본다.  
(`WarehouseQueuePanel.tsx`에서 이미 사용 중인 패턴 그대로 적용.)

---

### 창고 지도 탭 — 권한자 화면 구성

#### 편집 모드 토글

지도 우상단에 **"편집 모드"** 토글. 권한자에게만 표시.

| 모드 | 동작 |
|------|------|
| 읽기 | 지금과 동일. 박스 클릭 → 내용 팝업. |
| 편집 | 박스 드래그 이동 가능. 자리 클릭 → 박스 추가/편집/삭제. 앵글 구조 편집 패널 노출. |

#### 드래그로 박스 이동

편집 모드에서 박스를 다른 자리로 드래그해 이동.

- 대상 자리 **용량 초과 시 드롭 막기** (사이즈 유닛 합산 검사)
- 같은 자리 내 **스택 순서 변경**은 드래그 대신 ↑↓ 버튼 (더 명확함)
- 이동 즉시 저장 (별도 "저장" 버튼 없음)

#### 품목별 보기 탭

관리자 화면 상단에 **"위치별 | 품목별"** 토글.

- **위치별** (기본): 앵글 구조를 탐색해서 박스 편집 (현재 방식)
- **품목별**: 품목 검색 → 해당 품목이 담긴 박스 목록 (위치 + 수량) → 클릭해 박스 편집

품목별 보기는 "이 부품 몇 박스, 어디 있지?" 를 한 번에 해결하는 뷰.

#### 미배치 품목 알림

편집 모드 진입 시 상단에 **"배치 필요 품목"** 섹션 고정.

- 조건: `warehouse_qty > 0`인데 박스 합이 `warehouse_qty`보다 적은 품목
- 클릭하면 품목별 보기로 전환하며 해당 품목 자동 선택

입고 완료 후 배치 연결 플로우:
```
입고 확정 → "창고 배치 필요 N건" 배지(편집 모드 토글 옆) → 편집 모드 진입 → 미배치 품목 섹션
```

### 에러 UX

#### R5 출고 차단

```
출고 저장 → "배치 수량 부족" 에러 모달
→ 모달 안에 "창고 지도에서 배치하기 →" 버튼
→ 창고 지도 탭 편집 모드, 해당 품목 품목별 보기로 직행
```

#### R10 불일치 경고

```
창고 지도 박스 위 ⚠️ 배지 (박스합 ≠ warehouse_qty 인 품목)
→ 클릭 → 편집 모드 진입, 해당 박스 바로 선택
```
