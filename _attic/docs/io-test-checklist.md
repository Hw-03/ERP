# 입출고 기능 사전 테스트 체크리스트

> **목적**: 시연 전 직접 화면을 조작해 버그·어색한 UX를 발견한다.
> **대상**: 개발자 직접 수행
> **기본 PIN**: `0000` (전 직원 동일)

---

## 사전 준비

### 재고 및 배치 초기화

```powershell
cd c:\ERP\backend

# 1. 재고 리셋 (입출고 내역 포함 전체 초기화)
python scripts/reset_test_stock.py --yes

# 2. 재고 분포 보정 (품절 100 / 정상 400 / 부족 ~439)
python scripts/rebalance_test_stock.py --yes

# 3. 미완료 배치 정리 (draft/submitted/reserved 삭제)
python scripts/clear_pending_batches.py --yes
```

### 계정 정보

| 사번 | 이름 | 부서 | PIN | 결재 권한 | 비고 |
|------|------|------|-----|---------|------|
| E22 | 이필욱 | 조립/부장 | 0000 | 창고결재(대리) + 조립 부서결재(주) | 주 결재 계정 |
| E04 | 김건호 | 조립/과장 | 0000 | 조립 부서결재(부) | |
| E01 | 김민재 | 조립/대리 | 0000 | 없음 | 일반 조립 작업 |
| E10 | 이지훈 | 진공/대리 | 0000 | 없음 | 진공 부서 작업 |
| E11 | 김지현 | 고압/주임 | 0000 | 없음 | 고압 부서 작업 |
| E15 | 김도영 | 튜브/주임 | 0000 | 없음 | 튜브 부서 작업 |
| E13 | 오세현 | 튜닝/사원 | 0000 | 없음 | 튜닝 부서 작업 |
| E27 | 권동환 | 경영지원 | 0000 | 창고결재(주) | ⚠️ io_enabled=false 가능성 있음 |

> ⚠️ **E27 권동환** — 창고 결재 주담당이지만 `io_enabled=false` 상태일 수 있음.
> 입출고 화면 접근이 막히면 E22로 창고 결재 진행.

## 사전 체크 (시나리오 시작 전)

### 🔴 반드시 확인

- [ ] **E27 입출고 화면 접근 가능 여부**
  - E27(PIN 0000)으로 로그인 → 입출고 탭 클릭
  - 결과: 화면이 열리면 ✅, "권한 없음" 또는 빈 화면이면 ⚠️ (E22로 대체)

- [ ] **조립 외 부서의 수량 보정(adjust) 결재자 없음**
  - 진공·고압·튜브·튜닝 부서 직원이 adjust_in/out 요청 시 결재자가 없어 처리 불가 가능
  - 그룹 B 시나리오는 **조립 부서(E01)** 로만 테스트

- [ ] **disassemble 회수 불가 라벨 표시 여부**
  - B6 실행 후 내역 탭에서 해당 항목 클릭
  - 자식 라인 중 "회수 안 됨" 표시가 있는지 확인 (현재 UI에 표시 안 될 가능성 있음)

- [ ] **BACKFLUSH 재고 부족 시 안내 명확성**
  - A6 실행 시 일부 자식 품목이 재고 부족이면 "부족" 경고 표시
  - 경고에도 불구하고 제출 가능한지, 제출 후 내역이 정확히 표시되는지 확인

## 그룹 A — 핵심 흐름 (반드시 테스트)

### A1: 원자재 입고 (receive_supplier)

- [ ] E22(이필욱)으로 로그인
- [ ] 입출고 → **원자재 입고** 탭 선택
- [ ] 품목 검색: `7-TR` (튜브 원자재 계열) 또는 재고가 있는 TR 품목
- [ ] **낱개** 방식으로 품목 선택, 수량 **50** 입력
- [ ] 제출 (결재 없이 즉시 처리됨)
- [ ] 확인: 입출고 내역 탭 → 해당 품목 "원자재 입고" 항목 존재
- [ ] 확인: 품목 상세 → 창고 수량이 +50 반영됨

---

### A2: 창고 → 부서 낱개 (warehouse_to_dept)

- [ ] E15(김도영)으로 로그인
- [ ] 입출고 → **창고 반출** 탭 선택
- [ ] 품목 검색: A1에서 입고한 TR 품목 (창고 수량 있는 것)
- [ ] **낱개** 방식, 수량 **30** 입력
- [ ] 제출 → "요청함" 상태 확인
- [ ] E22(이필욱)으로 전환 → **승인함** 탭
- [ ] 해당 요청 클릭 → 내용 확인 후 **승인**
- [ ] 확인: 창고 수량 -30
- [ ] 확인: 튜브 부서 수량 +30
- [ ] 확인: 내역 탭 "창고 반출" 항목

---

### A3: 창고 → 부서 BOM (warehouse_to_dept)

- [ ] E15(김도영)으로 로그인
- [ ] 입출고 → **창고 반출** 탭 선택
- [ ] **BOM** 방식으로 튜브 중간공정 품목(7-TA-xxxx) 선택
  - BOM 있는 품목: 7-TA-0001 등 검색
- [ ] 수량 **1** 입력
- [ ] 제출 → E22로 전환 → 승인
- [ ] 확인: 내역 탭에서 BOM 하위 원자재 자동 차감 항목 펼쳐짐
- [ ] 확인: BOM 자식 원자재의 창고 수량 감소

---

### A4: 부서 → 창고 (dept_to_warehouse)

- [ ] E10(이지훈)으로 로그인
- [ ] 입출고 → **창고 반입** 탭 선택
- [ ] 진공 부서에 재고가 있는 품목(V로 시작) 검색
- [ ] 낱개, 수량 **5** 입력
- [ ] 제출 → E22로 전환 → 승인
- [ ] 확인: 진공 부서 수량 -5
- [ ] 확인: 창고 수량 +5
- [ ] 확인: 내역 탭 "창고 반입" 항목

---

### A5: 생산 등록 낱개 (produce)

- [ ] E11(김지현)으로 로그인
- [ ] 입출고 → **생산 등록** 탭 선택
- [ ] 고압 중간공정 품목(3-HA-xxxx 등) **단품** 선택
- [ ] 수량 **1** 입력
- [ ] 제출 (결재 없이 즉시 처리됨)
- [ ] 확인: 고압 부서 수량 +1
- [ ] 확인: 내역 탭 "생산 등록" 항목, 알약 표시 확인

---

### A6: 생산 등록 BOM (produce + BACKFLUSH)

- [ ] E22(이필욱)으로 로그인
- [ ] 입출고 → **생산 등록** 탭 선택
- [ ] **BOM** 방식으로 조립 중간공정 품목(3-AA-xxxx) 선택
- [ ] 수량 **2** 입력
- [ ] 제출 (즉시 처리)
- [ ] 확인: 조립 부서 대상 품목 +2
- [ ] 확인: BOM 자식 품목들 자동 차감(BACKFLUSH)
- [ ] 확인: 내역 탭 — 상위(+) 알약과 하위(-) 알약이 나란히 표시
- [ ] 확인: 재고 부족인 자식이 있으면 "부족" 경고 표시 여부

## 그룹 B — 결재 분기 / 예외 흐름

### B1: 수량 보정 입고 (adjust_in)

- [ ] E01(김민재)으로 로그인
- [ ] 입출고 → **수량 보정** → **입고** 탭 선택
- [ ] 조립 부서 품목(3-AA-xxxx 또는 3-AF-xxxx) 검색
- [ ] 낱개, 수량 **1** 입력, 사유 입력
- [ ] 제출 → "요청함" 상태
- [ ] E22로 전환 → **부서 승인함** 탭에서 해당 요청 확인 → 승인
- [ ] 확인: 조립 부서 수량 +1
- [ ] 확인: 내역 탭 ADJUST 증가 칩 표시

---

### B2: 수량 보정 출고 (adjust_out)

- [ ] E01(김민재)으로 로그인
- [ ] 입출고 → **수량 보정** → **출고** 탭
- [ ] 조립 부서 재고 있는 품목 선택
- [ ] 낱개, 수량 **1**, 사유 입력
- [ ] 제출 → E22 승인 (부서 결재)
- [ ] 확인: 수량 -1
- [ ] 확인: 내역 탭 ADJUST 감소 칩 표시
- [ ] 확인: 이유 메모 없을 때 빈 메모 vs 없음 구분 표시

---

### B3: 불량 격리 (defect_quarantine)

- [ ] E11(김지현)으로 로그인
- [ ] 입출고 → **불량 처리** → **격리** 탭 선택
- [ ] 고압 부서 재고 있는 품목 선택 (3-HA-xxxx 등)
- [ ] 수량 **3** 입력, 격리 사유 선택
- [ ] 제출 → E22로 전환 → 창고 승인함에서 승인
- [ ] 확인: PRODUCTION 수량 -3, DEFECTIVE 수량 +3
- [ ] 확인: 내역 탭 "격리" 항목

---

### B4: 격리 해제 (defect_restore)

- [ ] E11(김지현)으로 로그인 (B3 직후 진행)
- [ ] 입출고 → **불량 처리** → **격리 해제** 탭
- [ ] B3에서 격리한 품목 선택
- [ ] 수량 입력 → 제출 (즉시 처리, 결재 없음)
- [ ] 확인: DEFECTIVE -N, PRODUCTION +N
- [ ] 확인: 내역 탭 표시

---

### B5: 격리 처리 — 폐기 (defect_process)

- [ ] E11(김지현)으로 로그인 (B3 직후, 또는 격리 항목 남아있을 때)
- [ ] 입출고 → **불량 처리** → **폐기** 탭
- [ ] B3에서 격리된 품목 선택
- [ ] 제출 → E22로 전환 → 창고 승인함에서 승인
- [ ] 확인: DEFECTIVE 수량 소멸 (0)
- [ ] 확인: 총재고 감소, 내역 탭 "폐기" 항목

---

### B6: 재작업 — 분해 (disassemble)

- [ ] E10(이지훈)으로 로그인
- [ ] 입출고 → **재작업** 탭 선택
- [ ] **BOM** 방식으로 진공 부서 품목(V 계열) 선택
- [ ] 수량 **1** → 제출 (즉시 처리)
- [ ] 확인: 상위 품목 -1
- [ ] 확인: BOM 자식 품목 회수(+)
- [ ] 확인: 내역 탭 — 상위(-) danger 알약 + 하위(+) primary 알약
- [ ] ⚠️ 확인: "회수 안 됨" 라벨 표시 여부 (현재 표시 안 될 수 있음)

## 내역 탭 시각 검수

각 시나리오 완료 후 내역 탭에서 아래 항목 확인:

| 작업 유형 | 예상 표시 | 확인 |
|---------|---------|------|
| PRODUCE BOM | 상위(+) 알약 + 하위(-) 알약 나란히 | [ ] |
| DISASSEMBLE | 상위(-) danger + 하위(+) primary | [ ] |
| WAREHOUSE_TO_DEPT | "창고 반출 N품목" + 이동 방향 | [ ] |
| DEPT_TO_WAREHOUSE | "창고 반입 N품목" | [ ] |
| ADJUST | 증가/감소 칩 + 이유 메모 표시 | [ ] |
| 다품목 배치 | "하위 N건 (혼합)" + 클릭 시 상세 패널 | [ ] |
| 品절 품목 입출고 | 수량 0에서 시작하는 흐름 표시 | [ ] |
