2026.07.04 ~ 07.10 MES 개발 현황을 보고드립니다.
업무에 늘 수고가 많으십니다.

이번 주엔 출하·입출고 중 품목을 다른 품목으로 바꿔야 할 때 처리하는 품목 전환 기능을 새로 만들고,
현장 피드백을 다수 반영하여 출하·입출고 화면 전반의 사용성을 개선하였습니다.

───────────────────────────────────────────────────────────

■ 주요 진행 사항 (7/4 ~ 7/10)

1. 품목 전환 기능 신설 (진행 중, 7/6~7/10)
- 출하·입출고 과정에서 품목을 다른 품목으로 바꿔야 할 때 이를 처리하는 품목 전환 기능을 새로 만들었습니다
- 수동으로 방식을 고르던 것을 서버가 자동으로 판정하는 방식으로 바꾸어 실수 여지를 줄였습니다
- 품목을 검색해서 고를 때 목록 위치와 용어를 정리하여 더 찾기 쉽게 다듬었습니다

2. 출하 요청 작성 화면 개선 — 김건호 과장님 피드백 (7/7~7/9)
- 동반품을 필수 항목에서 선택 항목으로 바꾸어, 동반품 없이도 출하를 진행할 수 있게 하였습니다
- 재고가 부족해도 출하 요청 자체는 가능하도록 하고, 부족한 품목의 필요·가용·부족 수량을 화면에 표시하도록 하였습니다
- 출하 요청 단계와 최종 확인 화면의 정보 구성을 실제 품목 목록 중심으로 다시 정리하였습니다
- 출하 화면에서 깨져 보이던 한글과 내부 코드가 그대로 노출되던 부분을 바로잡았습니다

3. 부서 입출고 BOM·낱개 혼합 요청 지원 — 이형진 사원님 피드백 (7/8~7/9)
- BOM 묶음과 낱개 품목을 같은 요청 목록에 함께 담아 한 번에 부서 입출고를 요청할 수 있게 하였습니다
- 결재 승인 전에는 재고에 반영되지 않고, 승인 시 BOM과 낱개를 함께 반영하도록 하였습니다

4. 입출고 내역 화면 전면 개편 (진행 중, 7/8~7/10)
- 생산·불량·품목 전환 등 서로 다른 작업 유형이 목록에서 제각각 다르게 보이던 것을 같은 방식으로 통일하여 어떤 재고가 어떻게 바뀌었는지 한눈에 읽히도록 정리하였습니다
- 항목을 펼쳤을 때 우측 상세 패널과 자연스럽게 연동되도록 개선하였습니다
- 취소 버튼의 위치와 확인 방식을 단건·묶음 구분 없이 하나로 통일하였습니다

5. 모바일 입출고·불량 화면 오류 수정 — 허동현 사원님 피드백 (7/7~7/8)
- 부서 입출고에서 출고 처리 후 다음 화면으로 넘어가지 않던 문제를 바로잡고, 품목을 담은 뒤 이어서 다른 품목을 검색하기 편하도록 개선하였습니다
- 불량 바로 폐기 화면에서 검색 후 목록이 스크롤되지 않던 문제를 수정하였습니다
- 창고·부서 입출고 작성 중 저장이 되지 않던 문제를 해결하여, 작성하던 내용을 이어서 처리할 수 있게 하였습니다

6. 모바일 알림·헤더 정리 (7/6)
- 상단 헤더 구성을 더보기 중심으로 정리하여 화면을 간결하게 하였습니다

7. 모바일 레이아웃 밀도 정리 — 이형진 사원님 피드백 포함 (7/6, 7/9)
- 모바일 화면에서 품목명이 잘리던 곳들을 확인하여, 기본 2줄로 보여주고 탭하면 전체를 볼 수 있게 통일하였습니다
- 입출고·불량 처리 단계 표시와 화면 프레임·바텀시트 폭을 정리하여 화면에 더 많은 내용을 안정적으로 담을 수 있게 하였습니다
- 대시보드 재고 목록 정렬 방식을 다듬었습니다

8. 대시보드·화면 스크롤 편의 개선 — 권동환 사원님 피드백 (7/6~7/7)
- 대시보드 자재 목록에 스크롤 바를 표시했습니다.

9. 화면 전환 속도 개선 (7/6)
- 탭을 전환할 때 이전 화면이 잠깐 남아 보이던 현상을 없애고, 목록을 다시 불러오는 속도를 개선하였습니다

10. 공통 패널 화면 정리 (7/8)
- 우측 상세 패널의 뒤로가기·닫기 버튼을 제목 영역과 통합하여 화면을 더 깔끔하게 정리하였습니다

시스템은 제 자리 PC가 켜져 있을 때 http://192.168.0.63:3000 에서 확인하실 수 있습니다.
