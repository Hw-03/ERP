■ 이번 주 진행 사항 (5/23 ~ 5/29)

1. 직원별 입출고 권한 토글 신설
- 직원 개인 단위로 입출고 화면 접근을 켜고/끌 수 있는 토글을 새로 추가하였습니다.
- 꺼진 직원은 입출고 화면에 진입 불가
- DB 구조 변경·서버 API·테스트 동반

2. 관리자 화면 4종 한꺼번에 보강 (5/23)
- 대시보드에서 입출고 화면으로 바로 이동하는 동선 + 중복 품목 자동 합산
- 품목·모델·직원·부서 변경 시 저장하지 않고 화면을 빠져나가려 하면 "저장 안 됨"
  안내 모달 일괄 도입
- 모델 관리에서 행을 직접 편집하고, 끌어서 순서 바꾸기 기능 추가
- 관리자 사이드바의 "위험 영역" 섹션 제거
- 화면 곳곳에 흩어져 있던 버튼 모양을 공통 부품으로 통일(6곳)
- 입출고 내역 달력에 "연 단위 보기" 추가
- BOM 부서 필터에 "전체" 항목 추가, 부서 색상 입력은 hex 코드 + 30여 색
  프리셋 팔레트로 개선

3. 화면 데이터 갱신 방식 정비 (React Query 1차 도입)
화면이 서버에서 데이터(품목 목록·입출고 내역·재고 수량 등)를 받아오는 부분을
통일·자동화하였습니다.
- 도메인 10여 개를 React Query 기반으로 전환
  (이전: 화면마다 따로 / 이후: 한 군데서 관리 + 자동 새로고침)
- 화면 한 곳에 섞여 있던 "데이터 조회 / 입력 폼 / 명령 실행" 세 가지를
  세 부분으로 나눠 정리
- 변경 폭이 커서 일부 화면은 진입 시 한 박자 늦는 경우가 보여 추가 점검이
  필요한 상태입니다.

4. 백엔드 큰 파일 정리 (5/23~24)

- 로그인(PIN) 인증 코드 흐름을 한 군데로 모음

- 불량 처리 시 수량 입력 기능 추가
- 5/28에는 발표 시연 전에 발견된 자동 결재·재작업 트리뷰·요청자/승인자 표시
