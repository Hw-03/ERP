■ 이번 주 진행 사항 (6/8 ~ 6/10)

1. 불량 처리 패널 재설계 (6/10)
- 처리 패널 UI를 전면 재설계하여 부서별 그룹을 접기·펼치기 가능하게 개선
- reason_category를 optional로 처리하여 입력 단계 유연성 확보
- E2E 테스트 안정화: 정상복귀 제출 셀렉터 모호성 해소

2. 격리 목록 기본값·UX 개선 (6/9)
- 창고 담당자 격리 목록의 기본 스코프를 '전체'로 변경
- 격리 항목을 department='창고'로 그룹 표시
- 격리 목록 UX 개선: 출처 기본값·제목 명확화·배지·체크박스 추가

3. 불량 탭 전체 UX 다듬기 (6/9)
- 카트 흐름·뒤로가기·필터를 정리하고 사용 흐름 개선
- 탭 헤더 제거 + 뷰 전환 슬라이드 애니메이션 적용
- 부분 수량 복귀·부서 필터·KPI 카드 분리로 정보 계층화

4. 창고 지도 화면 다듬기 (6/8~6/9)
- 열 클릭·자리 분할·슬롯 번호로 공간 활용도 개선
- 위치 안내 UX 개선하여 사용자 탐색성 강화

5. 직원별 품목 순서 커스터마이징 (6/8)
- 입출고·격리·불량 품목창에 직원별 정렬 순서 UI 적용
- 드래그 기반 재정렬·초기화 기능으로 개인 맞춤 지원

6. 테스트 자동화 강화 (6/8~6/10)
- io-defect E2E 테스트 안정화
- io 제출 원자성 회귀 테스트 신설
- io_preview·warehouse_map 단위 테스트 신설

7. 백엔드 코드 정리 (6/8)
- Item·Inventory 조회 로직을 repository 계층으로 도입하여 중복 제거
