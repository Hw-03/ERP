안녕하세요. 생산부 김현우 사원입니다.

이번 주는 기준정보 체계와 백엔드 코드 구조를 정리하고,
받은 피드백을 모아 처리하였으며, 개발·운영 환경 분리와
테스트 자동화·결재 규칙 정합으로 시스템 신뢰성을 강화하고,
창고 지도 화면을 신설하였습니다.

───────────────────────────────────────────────────────────

■ 이번 주 진행 사항 (6/1 ~ 6/4)

1. 직원 MES 사용 응대 (6/1)
- 직원들이 MES를 직접 써보면서 생긴 의문·불편·모르겠는 점을 받아 정리
- 5/27 사내 시연 이후 누적된 후속 질문 위주로 진행
- 받은 내용은 보완 검토용으로 별도 정리

2. 기준정보 체계 고정 — 코드 보완 (6/1)
- DB 구조를 직접 살펴보면서 학습 (자료 흐름·테이블 관계 정리)
- 생산·분해 시 BOM 부품을 소속 공정에서 차감하도록
  백엔드 보완 (타 공정 부품 조정은 별개 흐름으로 허용)
- 기준정보 문서 정합: 빠르게 바뀌는 수치(품목 수·공정 종수 등)는 문서에
  박지 않고 기준 수치 조회 도구로 실시간 확인하도록 일원화 + 유령 칸 정리

3. 기준정보 정합 후속 — 수량·식별자·품목코드 (6/2 오전)
- 수량을 시스템 전반에서 정수로 통일: 입력 단계에서 소수 입력을 거부하도록
  차단하고, DB 컬럼 선언 타입과 과거 잔여 데이터까지 정수로 정합
- 품목·이동 식별자 형식이 군데군데 어긋나 있던 부분을 공통 처리로 일원화
- 품목코드(부서 + 모델 + 옵션) 생성을 단일 기준에서만 만들도록 정리하여,
  같은 코드가 다른 곳에서 다르게 만들어지는 가능성 차단

4. 백엔드 코드 구조 정리 — 큰 파일 분리 (6/2 오전)
- 한 파일에 모여 너무 커져 있던 입출고·재고 요청·재고 관련 백엔드 코드를
  책임별 모듈로 분리
- 외부 동작에 영향 없는 내부 정리입니다.

5. 운영 로그·사번 감사 보강 (6/2 오전)
- 5/29에 도입한 사번 자동 기록을, 누락되어 있던 직접 입출고(공정 간 이동 등)
  흐름에도 동일하게 적용
- 모든 품목 이동 경로에서 작업자 사번이 빠짐없이 남도록 보강

6. 불량 전용 탭 신설 (6/2 정오)
- 좌측 사이드바 맨 아래에 "불량" 탭을 신설하여 불량 입력·격리·처리 흐름을
  입출고 화면에서 분리
- 작업 화면 사용성은 입출고와 비슷하게 맞춰, 기존에 익숙한 조작 방식 유지

7. 받은 피드백 정리·처리 (6/1 ~ 6/4)
직원분들께 받은 피드백을 모아 처리한 결과입니다.

· 권동환 사원님 — 창고·부서 요청함에서 새 요청이 맨 위에 오도록 정렬 변경
  → 해결 (6/2)

· 이지훈 대리님 (3건)
  - 입출고 3단계 입고 품목 선택 영역에 자체 스크롤바 추가 → 해결 (6/2)
  - 관리자 BOM 관리에서 자식 행 삭제가 동작하지 않던 문제 → 해결 (6/2).
    진단 결과 식별자 형식 혼재로 일부 행을 찾지 못하던 문제였고, 위 3번
    식별자 일원화 작업으로 근본 원인까지 제거되었습니다.
  - 특정 BOM 데이터 교체 작업 → 진행 예정 (실데이터 입력 영역, 별도 진행)

· 허동현 사원님 (3건)
  - 입출고 내역 숫자 입력 칸에서 키 입력이 잠기는 현상 → 해결 (6/2)
  - 불량 화면 라벨 설명을 마우스를 올려야 보이던 부분을 본문에 항상 표시
    → 해결 (6/2)
  - 구매요청 후 요청자에게 알림이 가는 채널 추가 요청 → 검토 중
    (앱 내 알림함·메일 등 채널 선정 단계)

· 김건호 과장님 (5건)
  - 4단계 BOM 확인 화면에서 박스 어디를 클릭해도 펼쳐지도록 변경 → 해결 (6/4)
  - 휴지통 아이콘 크기·색상 개선(빨강·확대) → 해결 (6/4)
  - 하위의 하위 BOM 펼쳐보기(읽기 전용, 어떤 부품이 들어갔는지 확인용)
    → 해결 (6/4)
  - 결재 반려 후 요청자 화면에 반려 상태가 표시되지 않던 좀비 현상
    → 해결 (6/4, 30초 자동 새로고침)
  - 승인 요청 수신 시 승인 담당자에게 알림 → 검토 중
    (허동현 사원님 알림 건과 함께 채널 선정)

· 남재원 사원님 — 하위의 하위 BOM 펼쳐보기 → 해결 (6/4,
  김건호 과장님 동일 의견과 함께 처리)

· 이필욱 부장님 — 창고 맵 시각화 구두 요청 → 해결 (6/4,
  아래 10번 창고 지도 항목으로 별도 진행)

이 외 점검 중 식별·반영한 사항:
- 모델 관리 화면에서 신규 모델 추가가 동작하지 않던 문제 → 해결 (6/2).
  신제품(모델 9)을 정식 등록하고, 신규 모델이 화면 곳곳에서 자동 인식되도록
  함께 보강하였습니다.
- 재작업 시 회수 외 항목이 화면에서 사라지는 현상 → 점검 중
  (불량 격리 이동 흐름으로 정정 검토)
- 입출고 작업 중간 저장이 이전 내역에 덮어쓰이는 현상 → 점검 중
  (사번 감사 로그 보강과 묶어 처리 예정)

8. 개발·운영 환경 분리 (6/2 오후)
- 개발 환경과 운영 환경을 포트로 분리하여, 개발용 작업이 실제 운영
  데이터에 영향을 주지 않도록 정비
- 브라우저 탭 제목에 환경(개발/운영)을 표시하여 시각적으로 구분
- 운영 환경 시작 스크립트 정비

9. 시스템 신뢰성 강화 — 자동 점검·결재 규칙·내역 일치성 (6/4 오전)
- 화면을 사람이 직접 클릭하며 점검하던 절차 일부를 자동으로 옮겨,
  불량 처리·결재 전체 흐름·생산 잠금 등이 코드 변경 후에도 그대로
  동작하는지 매번 자동 확인되도록 마련
- 결재 규칙(어느 단계에서 누가 결재하는지)을 백엔드 한 곳에서만 정의하도록
  통합하고, 프론트와 어긋나면 코드 단계에서 자동 차단되도록 정비
- 입출고 내역을 엑셀/CSV로 내려받을 때 요청자·승인자 이름이 화면 표시와
  일치하도록 보정

10. 창고 지도 화면 신설 (6/4 오후)
이필욱 부장님 구두 요청 사항 대응입니다.
- 창고 안 앵글(철제 선반) 9개의 위치와 박스 배치를 한 화면에서 보는 지도뷰 도입
- 앵글 구조 편집·박스 위치 배정 기능 마련
- 데이터 모델·서버 API·화면까지 통합

화면은 제 자리 PC가 켜져 있을 때 http://192.168.0.63:3000 에서 확인하실 수 있습니다.
