### 입출고(4단계) — 수량 0 상태에서 키보드 입력 차단
#입출고 #수량 #UX #버그 #해결

수량을 0으로 만들면 - 버튼과 제출확인이 비활성화되는 건 맞는데, 키보드로 수량 직접 입력은 가능해야 함. 현재는 키보드 입력 필드도 함께 막혀서 0에서 다른 숫자로 다시 입력하려면 + 버튼을 눌러야만 함.

→ 해결 (2026-06-22, 커밋 `732950b0`)
- `IoLineRow.tsx` input `disabled={stepperDisabled}` → `disabled={qtyLocked}` 변경. qty=0이 되면 bomSync가 included=false로 자동 설정 → stepperDisabled=true로 키보드까지 막히던 구조. + 버튼과 동일하게 qtyLocked만 차단하도록 수정. qty>0 타이핑 시 bomSync가 included=true 자동 복구.

### 입출고(4단계) — 취소 후 재추가 시 재고 미갱신 (캐시)
#입출고 #재고 #버그 #해결

재현 흐름:
1. 창고·부서 아무 유형이나 낱개 품목 추가 → 요청 제출
2. 내 요청 화면에서 해당 요청을 내가 직접 취소
3. 다시 같은 품목을 낱개 추가 → 4단계로 이동
4. '가능 재고'·'실행 후 재고' 숫자가 취소 이전 값 그대로 표시됨
5. 새로고침하면 올바른 값으로 갱신됨

새로고침 없이 바로 반영되도록 수정. 비슷한 캐시 갱신 문제가 있을 수 있는 곳도 함께 찾아 해결.

→ 해결 (2026-06-22, 커밋 `732950b0`)
- `IoComposeView.tsx`·`MobileIoComposeWizard.tsx` 모두에 step 4 진입 시 `api.getItems()` 재fetch useEffect 추가. 취소·승인 등으로 재고가 바뀐 후 재추가해도 새로고침 없이 최신 재고값 반영.
