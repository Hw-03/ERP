### 대시보드 — 빠른 작업칸 입고/출고 분리 + 최종 단계 바로 진입
#대시보드 #빠른작업 #입고 #출고 #UX #추가요청 #해결

대시보드 빠른 작업칸이 입고·출고로 나뉘고, 누르면 바로 입출고 단계로 넘어갔으면 좋겠다는 요청.

→ 해결 (2026-06-12, 2026-06-15 보강)
- 빠른 작업을 `[입고]`/`[출고]` 2버튼 + 인라인 팝업(부서 입고/창고 반입/원자재 수령 · 부서 출고/창고 반출)으로 분리.
- 선택 시 `IoEntryIntent`로 입출고 위저드에 작업유형·방향·부서를 프리셋하고 진입:
  - BOM 부모(조립품): Step 3(대상 선택)에서 BOM/낱개를 직접 선택. 선택 품목을 피커 검색창에 프리필 + 줄 전체 점멸로 강조(정렬상 뒤쪽 품목도 즉시 표시).
  - BOM 없는(낱개) 품목: 자동 카트 추가 후 Step 4(수량 확인)로 바로 진입.
- 데스크톱 진입 시 탭이 대시보드로 튕기던 버그도 수정(`useIoUrlSync` tabParam로 tab 고정).

## 2026-07-08
### 입출고 — 부서 입출고에서 BOM과 낱개를 동시에 요청 목록에 담기
#입출고 #부서입출고 #BOM #낱개 #요청목록 #재고반영 #추가요청 #해결

BOM과 낱개 품목을 동시에 장바구니/요청목록에 담고, 한 번에 부서 입출고 요청을 보낼 수 있어야 함.

조건:
- BOM과 낱개가 같은 요청 목록에 공존 가능해야 함.
- 요청 제출도 동시에 가능해야 함.
- 승인 전까지는 BOM도 실제 재고에 반영되지 않아야 함.
- 승인 시점에 BOM 구성품 재고 차감/검증 흐름이 낱개 품목과 함께 일관되게 처리되어야 함.

구현 완료 (2026-07-09)
- 부서 입출고에서 BOM 묶음과 낱개 manual 라인을 같은 작성 목록에 담을 수 있게 변경.
- BOM+낱개 혼합 요청은 부서 결재 1건으로 생성하며, 승인 전에는 전체 재고를 미반영.
- 승인 시 BOM 라인과 낱개 라인을 한 batch로 함께 반영.

### 모바일 — 품목명 잘림 없이 전체 표시
#모바일 #품목명 #텍스트 #UX #불편 #해결

모바일 화면에서 품목명이 잘리는 곳들을 전부 확인하고, 사용자가 품목명을 온전히 볼 수 있게 표시해야 함.

확인 범위:
- 출하 요청/요청 목록 화면
- BOM/낱개 선택 화면
- 모바일 공통 품목 카드/리스트
- 긴 품목명, 모델명, 옵션명이 함께 표시되는 케이스

구현 완료 (2026-07-09)
- 모바일 출하 카드의 대표 품목명과 최종 PA/PF명을 기존 입출고 모바일과 같은 `기본 2줄 + 탭 시 전체 펼침` 패턴으로 통일.
