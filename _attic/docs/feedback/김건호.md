### 입출고(결재) — 반려된 요청 좀비 버그
#입출고 #결재 #반려 #버그 #미해결

허동현의 알루미늄 필터 요청을 반려했는데, 요청자 "내 요청" 화면에서 반려 상태가 표시되지 않음. 취소 버튼을 눌러도 실제 취소가 안 되는 좀비 상태로 남아있음.

→ 미해결
- 현재 동작: 반려(REJECTED) 후 요청자 내 요청 목록에서 상태가 갱신되지 않음. 취소 버튼 클릭 시 요청만 가고 실제 상태 변경 안 됨.
- 원인 추정:
  - 요청자 화면이 폴링/리패치를 안 해서 stale 상태 유지.
  - 또는 REJECTED 상태의 요청에 CANCEL 액션이 백엔드에서 허용되지 않는데 프론트가 버튼을 그대로 노출.
- 점검 후보:
  - 백엔드 — REJECTED 상태에서 CANCEL 전환 허용 여부 (`cancel_stock_request` 라우터 상태 검사).
  - 프론트 — 내 요청 목록 리패치 주기 / REJECTED 상태일 때 취소 버튼 표시 조건.
- 실제 잔재: 허동현 알루미늄 필터 요청이 DB에 좀비로 남아있을 가능성 — 확인 후 수동 정리 필요.
- 수정: `BomRow.tsx` — 입력 `step="any"` → `step="0.01"`. 표시에서 `formatQty(row.quantity, { maximumFractionDigits: 2, trimTrailingZeros: true })` 적용 → 1.0000 같은 과도한 자릿수 사라짐.
