### 3. 입출고(이어서 하기) — 같은 draft 재복원 시 첫 클릭 씹힘(이중 클릭)
#입출고 #임시저장 #이어서하기 #단계네비 #버그 #미해결

(김건호 과장 정밀 재현, 2026-06-22 확인)
1. 작성 중 다른 탭 이동 → '작업 중' 1건 생성.
2. '이어서 작업' 클릭 → 4단계로 바로 이동(정상).
3. 거기서 3단계로 이동 → 문제 없음.
4. 2단계로 이동 → 문제 없음.
5. **1단계로 이동 → 사실상 새 작업이 됨**(작업유형부터 재선택, 담은 것 초기화).
6. 이 상태에서 '작업 중' → '이어서 작업'을 다시 클릭하면 → 그 장바구니로 4단계 복원이 아니라 **1단계 새 작성으로 감(첫 클릭 씹힘)**.
7. **한 번 더** 같은 작업을 '이어서 작업' 클릭하면 → 그제서야 4단계로 정상 이동.

→ 미해결 (근본 원인 확정 — 단, "정확히 두 번" 횟수는 리렌더 타이밍 의존이라 화면 재현 권장)
- `DesktopWarehouseView.tsx:72,169-191` — '이어서 작업' = `setRestoreIoDraft(draft)` 만 함. 복원 후 이 값을 **초기화하는 코드 없음**(grep 확인). `restoredDraftRef`(빗장)도 안 풀림.
- `useIoDraftRestore.ts:32-51` — 복원 effect는 의존성 `[draftToRestore?.batch_id]` **변화 시에만** 1회성 발동 + `if (restoredDraftRef.current === batch_id) return` 가드. 같은 draft를 다시 이어서하면 batch_id 불변 → effect 안 깨어남 → `goTo(4)` 미실행 → 1단계 머무름(6번). 한 번 더 거쳐 값/마운트가 갱신돼야 빗장 풀려 4단계 점프(7번).
- 참고: 2·3단계 이동이 정상인 이유 = 같은 작업 내 단계 이동이라 복원 추적값과 무관. (앞선 "부서 변경이 번들 초기화" 가설은 오진 — 트리거는 **1단계 이동**.)
- 해결 방향(미구현): 복원 시 step 추적값을 새 작업 시작/탭 전환 때 함께 리셋하거나, 이어서하기를 batch_id 변화가 아니라 명시 트리거(클릭마다)로 복원하도록 변경.
