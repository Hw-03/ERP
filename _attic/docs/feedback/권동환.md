### 창고 지도 — 통로 적재·PL 적재 관리 기능 추가
#창고지도 #통로적재 #파렛트 #PL적재 #요청 #해결

MES 창고 지도 편집 모드에는 현재 앵글 추가·크기 조정만 있음. 실제 현장에서는 앵글 공간 부족으로 통로에 임시 적재하거나 파렛트 위에 자재를 보관하는 경우가 있어, 창고 지도에서 `통로 적재`와 `PL 적재`를 별도 관리할 수 있으면 좋겠다는 요청.

구현 방향은 사용자 답변 기준으로 정리:
1. 현재 앵글 위치가 보이는 창고 지도에 **파렛트**를 추가해 화면에 표시.
2. 파렛트 적재는 정면도·열·층·자리 같은 세부 배치 없이, 파렛트를 클릭하면 해당 파렛트에 어떤 품목이 몇 개 있는지만 리스트로 표시.
3. 통로 적재도 지도에 표시해야 하므로, 창고 지도에 **통로**라는 개념/구역을 배치.
4. 통로 적재는 전용 탭 또는 전용 화면에서 품목·수량 리스트를 확인·관리하는 방향.

권동환 사원은 위 방향대로 진행하면 된다고 확인.

→ 해결 (2026-06-25, 커밋 `fc248779`)
- 창고 지도에 통로/PL 특수 구역 CRUD와 품목·수량 관리 기능 추가.
- 통로/PL 구역은 지도에 표시되고, 클릭 시 적재 품목 리스트를 관리.
- 백엔드 특수 구역 모델·라우터·마이그레이션·OpenAPI baseline 및 창고 지도 UI/API 반영.

### 관리자(직원 권한) — 입출고 권한이 OFF로 풀림

→ 해결 (2026-06-26, 커밋 `5f96c5ea`)
- 마이그레이션 재실행 시 직원별 `io_enabled` 저장값을 부서 권한으로 다시 덮어쓰지 않도록 권한 보존 처리.

품목 추가 시 `위치 추가` 버튼은 부서가 아니라 위치를 봐야 함. 창고에 놓으려 했는데 부서만 표시됨.

위치는 별도로 존재할 가능성이 있으므로, 품목 추가의 위치 선택 UI가 부서 목록이 아니라 실제 위치 목록을 기준으로 동작하는지 확인 필요.

→ 해결 (2026-07-02)
- 초기 재고 위치 선택에 `창고` 옵션을 추가.
- 백엔드 전송 시 창고 수량은 총 초기 수량에 포함하되, 부서별 초기 위치 배열에는 부서 위치만 보냄.
- `ItemFormFields.test.tsx` 회귀 테스트 추가.

### 출하 관리 화면 신설 및 감사 추적 표준화 검토
#출하 #감사추적 #로그 #마스터데이터 #재고변동 #요청 #미해결

출하 관리 화면 신설 및 개선 관련 피드백.

출하 구성품 변경 이력을 남기는 것은 좋지만, 해당 이력이 특정 화면인 출하 탭에만 파편화되어 구현된 것인지 확인 필요.

MES 같은 전사 시스템은 모든 마스터 데이터와 재고 변동에 대해 생성자, 생성일시, 수정자, 수정일시, 이전 값, 이후 값을 저장하는 표준 log 테이블 구조가 설계되어 있어야 한다는 의견.

구현 전 확인 포인트:
- 현재 출하 구성품 변경 이력이 전사 감사 추적 구조를 쓰는지, 출하 탭 전용 구조로 분리되어 있는지 확인.
- 마스터 데이터 변경과 재고 변동 로그의 공통 감사 필드/이전 값/이후 값 저장 방식 점검.
- 출하 관리 화면 신설 시 감사 추적을 화면별 임시 구현이 아니라 공통 구조로 연결하는 방향 검토.
