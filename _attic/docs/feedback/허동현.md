→ 해결 (2026-05-29) — 사용자 결정: 4자리 고정.
- 백엔드 진실 소스: `backend/app/services/pin_auth.py` 에 `validate_pin()` 추가 (정확히 4자리 숫자 아니면 HTTP 422). `change_employee_pin` 진입점에서 호출. `schemas.py` 의 `EmployeePinChangeRequest.new_pin` 도 `min_length=4, max_length=4`.
- 프론트 일관성: `frontend/lib/auth/constants.ts` 신규 `PIN_LENGTH=4`. 로그인·PIN 변경·관리자 PIN 초기화 폼 모두 이 상수 import. `DesktopTopbar`, `AdminEmployeesSection` 의 maxLength·숫자 필터·제출 비활성화 통일.
- **잠긴 사용자 응급 처리**: 관리자가 `reset_employee_pin` 으로 0000 초기화 → 사용자 로그인 → 4자리 PIN 새로 설정.
