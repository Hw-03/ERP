### 관리자(BOM 관리) — 자식 행 삭제가 동작하지 않음
#관리자 #BOM관리 #삭제 #버그 #추가요청 #해결

발생부 (진공) [COCOON] BOM 수정 중 실패. 의도: `고압보드SMT 完 [Ver 2.12 신형] [COCOON]` → `발생부 고압B/D+튜브 최종 작업完 [Ver 2.12] [COCOON]` 로 교체. 수정 과정에서 기존 자식(`고압보드SMT 完 [Ver 2.12 신형] [COCOON]`) 삭제가 안 됨.

→ 해결 (2026-06-02)
- **원인**: SQLAlchemy PostgreSQL UUID dialect 가 SQLite 에서 `bom_id` 를 no-hyphen hex(32자)로 바인딩하는데, 일부 행이 hyphen 포함(36자) 형식으로 저장돼 있어 ORM filter 가 해당 행을 찾지 못함 → 404 "BOM 항목을 찾을 수 없습니다." UI 토스트 뜨고 사라짐.
- **즉각 수정**: `backend/bootstrap/migrate.py` — `_normalize_bom_uuid()` 추가. bom.bom_id hyphen 포함 848건 → no-hyphen hex 일괄 정규화 (마이그레이션 실행 완료).
- **재발 방지**: `backend/app/models/base.py` — `UUIDString` TypeDecorator 신설. write 시 hyphen 입력도 no-hyphen hex 강제, read 시 uuid.UUID 정규화. 7개 모델 파일 `UUID(as_uuid=True)` → `UUIDString` 전면 교체. 어떤 경로로 삽입돼도 포맷 혼재 불가.
- **범위**: 모든 부서 공통 버그였음 (출하가 됐던 건 운). bom 테이블 전체 1752건 no-hyphen hex 통일 완료.
- BOM 데이터 교체(고압보드SMT → 발생부 고압B/D+튜브) 는 사용자가 직접 진행 (삭제 버그 해소됐으므로 정상 동작).

### 입출고 — BOM 선택 시 다른 제품 낱개 동시 선택 불가
#입출고 #BOM #낱개 #동시선택 #사양 #종결

한 제품의 BOM을 누르면 다른 제품의 낱개를 못 누르는 현상. (BOM·낱개 전반)

→ 종결 (의도된 동작 — 변경 없음)
- 설계 의도: 낱개 입출고는 수량을 거짓으로 맞출 수 있어 **부서장 승인**이 필요한 반면, BOM은 바로 빠져도 문제가 없어 — 둘을 **동시에 선택할 수 없도록 일부러 구현**.
- 경위: 김현우 사원이 의도 설명 + 동시 처리가 꼭 필요한 이유를 역질문 → 허동현 사원 "편의상 이유였고, 일부러 한 거면 어쩔 수 없다"고 수긍. 추가 변경 불필요로 종결.
