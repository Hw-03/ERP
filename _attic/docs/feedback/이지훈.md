### 입출고 — 3단계 입고 품목 선택 영역에 자체 스크롤바 추가
#입출고 #입고품목선택 #스크롤 #UX #추가요청 #미해결

전체 화면 스크롤바는 있지만, 3단계 입고 품목 선택용 스크롤 바가 없음. 입고 품목 선택용 영역에 자체 스크롤 바가 있으면 좋겠다는 의견.

→ 미해결
- 현재 동작 추정: 입고 품목 리스트 영역이 페이지 전체 스크롤에 묶여 있음 → 리스트가 길면 헤더·필터가 같이 스크롤됨.
- 기대 동작: 품목 선택 영역만 내부 스크롤 → 헤더·필터·요약(상위 N개·하위 N개)은 고정.
- 점검 후보:
  - `frontend/app/legacy/_components/_warehouse_steps/` 의 3단계 품목 선택 스텝
  - 리스트 컨테이너에 `overflow-y: auto` + 고정 높이(또는 `max-h-[...]`) 부여
- 작업량: 작음. CSS/스타일 조정 한 곳.

### 관리자(BOM 관리) — 자식 행 삭제가 동작하지 않음
#관리자 #BOM관리 #삭제 #버그 #추가요청 #미해결

발생부 (진공) [COCOON] BOM 수정 중 실패. 의도: `고압보드SMT 完 [Ver 2.12 신형] [COCOON]` → `발생부 고압B/D+튜브 최종 작업完 [Ver 2.12] [COCOON]` 로 교체. 수정 과정에서 기존 자식(`고압보드SMT 完 [Ver 2.12 신형] [COCOON]`) 삭제가 안 됨.

→ 미해결
- BOM 수정 자체(데이터 교체)는 사용자가 직접 처리 예정.
- **삭제 안 되는 버그 점검 필요**:
  - `frontend/app/legacy/_components/_admin/BOM*` (또는 BOM 관리 컴포넌트) — 우측 "현재 구성(N건)" 행의 삭제 아이콘 동작
  - 백엔드 — BOM 자식 삭제 라우터(`PATCH/DELETE /api/items/{id}/bom-children/...`)가 정상 응답하는지
  - 권한/상태 — `bom_completed_at` 이 박힌 부모는 자식 삭제 차단되는지 (완료 상태 락 가능성)
- 캡처에서 부모(7-VA-0017) 가 "완료" 배지 → 완료된 BOM 은 편집 모드 진입 시에만 수정 가능한지 점검 필요.
