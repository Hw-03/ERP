# 피드백 모음

DEXCOWIN MES 사용자 피드백을 직원별로 모아두는 폴더.

## 폴더 구조

```
docs/feedback/
  README.md         ← 이 파일 (사용 설명서)
  허동현.md          ← 직원별 파일 (이름 그대로)
  이필욱.md
  김건호.md
  ...
```

직원별로 파일 하나. 그 사람이 한 모든 피드백이 시간순으로 쌓임.

## 파일 양식

### 직원 파일 머리

```markdown
# 허동현
받은 피드백: 3건 (해결 2 / 미해결 0 / 보류 1)

---

(여기부터 피드백 항목)
```

통계 줄은 새 피드백 추가하거나 상태 바꿀 때마다 갱신.

### 피드백 항목 표준 모양

```markdown
## 2026-05-28
### 입출고 내역 — 재작업 행 우측 패널 자식 부품명 표시
#입출고내역 #재작업 #불편 #해결

재작업 행을 클릭했을 때 오른쪽에 열리는 정보가
'튜브 고정 보드 에노드 (나사)'로 이상하게 표시됨.

→ 해결 (2026-05-28, DesktopHistoryRightPanel.tsx 수정)

---
```

- **`## 날짜`** — 받은 날 (YYYY-MM-DD)
- **`### 화면 — 한 줄 요약`** — 어디서 / 무슨 일
- **태그 줄** — `#화면 #주제 #분류 #상태`
- **본문** — 직원이 적어준 원문 그대로
- **`→ 처리 메모`** — 해결/보류 사유, 수정한 파일 등
- **`---`** — 다음 항목과 구분선

## 태그 규약

### 상태 (필수, 한 개)
- `#미해결` — 받은 직후 기본값. 처리 대기.
- `#해결` — 코드 수정·답변 등으로 끝남.
- `#보류` — 정책 결정 대기, 거절, 다음 분기로 미룸 등 "지금은 안 함".

### 분류 (필수, 한 개)
- `#불편` — 안 되거나 불편함 (버그·UX 결함).
- `#요청` — 새 기능·기능 추가 요청.
- `#기타` — 질문·의견·정책 문의.

### 화면 (해당되면)
한글 붙여쓰기. 예: `#입출고내역` `#입출고불량` `#주간보고` `#품목관리` `#재고` `#결재` `#부서조정`

### 주제 (해당되면, 자유)
한글 붙여쓰기. 예: `#재작업` `#BOM` `#불량처리` `#회수` `#폐기`

### 즉석 태그
정말 급하면 그때그때 `#긴급` 같은 태그 자유롭게 추가. 미리 정해진 규약 없음.

## 직원에게 보낼 빈 양식

직원이 채워서 보내주는 양식:

```
이름 :
[피드백 1]
분류 :  ( ) 불편한 점   ( ) 추가 요청   ( ) 기타
어디서 :
내용 :

[피드백 2]
분류 :  ( ) 불편한 점   ( ) 추가 요청   ( ) 기타
어디서 :
내용 :
```

채워진 양식을 Claude에게 주면 알맞은 직원 파일에 정리해 추가함.

## 옵시디언 활용 팁

- `#미해결` 태그 클릭 → 전 직원 미해결 피드백 한 페이지로 자동 집계.
- `#입출고내역` 클릭 → 입출고 내역 관련 의견만 모음.
- `#재작업` 클릭 → 화면 가로질러 재작업 주제 의견 다 모음.
- 직원 파일은 이름 자체로 분류되므로 `#허동현` 같은 태그는 안 붙여도 됨.

## 추가·갱신 절차

1. 직원이 양식 채워서 보냄 (말·메일·메모 등).
2. 사용자가 Claude에게 양식 전달.
3. Claude가:
   - 해당 직원 파일 없으면 새로 만듦 (이름 + 통계 머리 포함).
   - 항목을 시간순으로 추가 (`#미해결` 시작).
   - 통계 줄 갱신.
4. 처리 후 상태 바꿀 때도 Claude가 통계 같이 갱신.
