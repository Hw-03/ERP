## 0. 이 문서 쓰는 법 (비개발자용 요약)

고칠 게 약 60개입니다. 한꺼번에 다 손대면 서로 꼬이므로 **순서와 묶음**을 정해 뒀습니다.

- **1단계(공통 먼저, 1명)**: 여러 화면이 같이 쓰는 "공통 부품" + 가장 위험한 버그. 이건 한 명만 건드려야 안 꼬임.
- **2단계(화면별, 최대 5명 동시)**: 화면 5개를 5명이 동시에. 각자 자기 화면 폴더만 만져서 안 부딪힘.
- **별도 트랙(큰 공사)**: 권한·기록·내보내기. 백엔드까지 필요 → 화면 손질 팀과 분리, 나중에 따로 설계.

나중에 실행할 땐: 1단계 1명 띄워 끝나면 → 2단계 5명 동시에 띄우고, 각 패키지의 **"에이전트 지시문"을 그대로 복사해 붙여넣기** 하면 됩니다.

|---|---|---|---|
| **Wave 1** | 공통 부품 안전·접근성 + 관리자 위험 버그 3건 | 에이전트 1명 (순차) | 없음 (제일 먼저) |

## 4. 별도 트랙 — 큰 공사 (이번 팀 작업 밖, 나중 별도 설계)

대기업 적합성을 위한 아키텍처 항목으로, 화면 손질이 아니라 **백엔드·데이터 구조까지 바뀌는 별개 공사**다. 본 계획의 Wave 1·2와 분리하며, 착수 전 **별도 설계 문서**가 선행되어야 한다.

- **권한 관리(RBAC)**: 현재 모든 사용자가 PIN만으로 거래 수정·관리자 기능 접근. 직원에 역할 등급 필드를 두고, 화면·기능별 노출/차단을 역할로 제어. 입출고내역의 수정 권한 개방, 관리자 권한등급 편집 불가가 여기 묶임.
- **변경 기록(감사 추적)**: 누가·언제·무엇을 바꿨는지 로그. 현재 등록일/수정일 2개 필드뿐. 관리자 "변경 이력" 탭, 대시보드 이력 5건 제한이 여기 묶임.
- **내보내기 재설계**: 기간·범위 필터 일관화, 내보내기 기록 영속화(현재 새로고침 시 소실), 주간보고 PDF/Excel. 백엔드 export API 정비 포함.

> 표시: **별도 설계 필요 — 백엔드 포함. 본 계획에서는 범위만 명시, 상세 설계는 별도 문서에서.**

## 5. 검증 기준

**작업 중(각 에이전트, 빠른 루프)**: 프론트 전용만 — `npm run lint:strict` + `npx tsc --noEmit` + `npm run test:coverage`. (프론트 UI 변경에 백엔드 pytest·OpenAPI·production build를 매번 돌릴 필요 없음 → 속도·낭비 방지)

**통합 시점(통합 담당, 1회)**: 합친 가지 전체에 대해 한 번만
```
powershell -ExecutionPolicy Bypass -File .\scripts\dev\verify_local.ps1
```
실측 게이트: backend pytest / lint:strict / tsc / vitest+coverage / production build / bundle-size / OpenAPI drift / git status. 통과 못 하면 완료 아님.

⚠️ **자동 접근성 검사 없음**: 위 게이트 어디에도 aria/scope/포커스 검사가 없다(jest-axe 등 미설치). 즉 "게이트 통과 = 접근성 완료"가 **아니다**. 접근성 검증은 §7의 검수 단계로만 담보된다.

접근성 목표 수준: **WCAG 2.1 AA** (테이블 헤더 scope, 폼 라벨, 모달 dialog/포커스, 선택상태 aria, 색 외 2채널 정보 전달).

## 7. 실행 메커니즘 (grill-me 2차 보강 — 가장 중요)

> 이 절이 "5명 동시"가 실제로 안전하게 돌아가게 하는 핵심. 빠지면 결과물이 엉킨다.

### 7-1. 작업실 격리
- Wave 2의 5개 메뉴 에이전트는 **각자 독립 git worktree**에서 작업(Agent `isolation: "worktree"`). 같은 폴더 동시 수정 금지.
- 폴더가 메뉴별로 disjoint(`_inventory_/_warehouse_/_history_/_weekly_/_admin_`)이라 worktree만 분리하면 충돌 거의 없음.

### 7-2. 가지·통합
- **새 전용 가지 `fix/ui-ux-2026-05`** 를 `main`에서 분기(현재 작업 가지 `feat/admin-bom-workbench-overhaul`와 섞지 않음 — CLAUDE.md 정책).
- ⚠️ **다른 세션 작업 중**: 가지 생성·착수는 그 작업이 커밋/머지된 뒤. 착수 직전 `git status`·충돌 확인.
- 흐름: Wave 1(1명) → `fix/ui-ux-2026-05`에 커밋 → Wave 2(5명) 각 worktree가 그 가지에서 분기 → **통합 담당이 메뉴별로 하나씩 순차 머지**(공통 파일은 Wave 1이 이미 끝냈으니 충돌원 없음) → 통합 1회 전체검증(§5) → 사용자 보고.
- `main` 자동 머지·자동 커밋 **안 함**. CLAUDE.md 정책상 사용자 요청 시에만.

### 7-3. 검수 단계 (Wave 3 = 재검수)
