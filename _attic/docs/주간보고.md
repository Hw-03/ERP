■ 이번 주 진행 사항 (5/18~5/22)

**1. 운영 인프라 / 배포 안정화**
실제 운영 환경에서 안정적으로 동작하도록, 5/18 야간에 풀스택 보강 시리즈(WS1~WS10)를
집중적으로 진행하였습니다.
- **컨테이너 정합** — docker-compose 운영 설정 정리, 운영 빌드와 개발 빌드 충돌 방지
- **DB 시동 가드** — 운영 환경에서 SQLite를 쓸 때 안전하게 켜지도록 가드 추가, 마이그레이션
  진단을 명확히 실패시켜(silent fail 차단) 이상 시 즉시 알 수 있도록 보강
- **세션 안정화** — 요청 처리 중 예외가 나면 DB 트랜잭션이 깨끗하게 롤백되도록 정비
- **헬스/관측 와이어링** — 서버 상태 점검 엔드포인트와 관측(로그/메트릭) 경로 정리
- **로그인 보안 하드닝** — URL 쿼리스트링으로 PIN이 노출되던 경로 제거 + 로그인 시도 레이트리밋
- **에러 로깅 보강** — 모든 예외에 풀스택 로깅을 남기도록 일괄 적용
- **동시 입고 경합 처리** — 같은 시점에 입고가 겹쳐도 422로 깨끗하게 응답
- **미지정 모드 크래시 가드** — 입출고 화면에서 알 수 없는 모드가 들어와도 화면이 죽지 않도록 처리

**2. 입출고 내역 화면 2·3차 정비 + 본 시스템 반영**
지난 주 시작한 입출고 내역 화면 재설계의 2·3차 정비분을 마무리하여 본 시스템(main)에 반영하였습니다.
- 상단 카드를 슬림하게 정리하고, 입출고 정정 모달을 하나로 통합
- BOM 묶음 내역의 "뒤로" 이동 동선 정비
- 필터를 대시보드와 같은 형태로 통일하고, 모델·공정·부서 단위 필터링 파라미터 추가
- API 명세(OpenAPI baseline)도 함께 갱신

**3. 모바일 UI/UX 전면 개편 (이번 주 두 번째 메인 작업)**
데스크톱이 안정화된 시점에 맞춰, 모바일 화면을 전면 재설계하였습니다.
- 공통 반응형 레이어를 먼저 도입하고, 그 위에 대시보드·입출고·내역·관리자 5개 화면을
  화면 크기에 맞춰 자동 재배치
- 디자이너 검수 결과 **PASS (95점)** 으로 모바일 화면 마무리 단계 완료
- 직전 보고서의 "추후 진행 예정" 6번(모바일 화면 마무리)이 이번 주에 완료되었습니다

**4. 불량 처리 흐름 재설계 Phase 1 (신규 핵심)**
실제 현장에서 발생하는 불량 처리 동선을 시스템에 맞게 재설계하였습니다.
- **라인 결재 권한** — 라인 단위로 불량 격리·처리에 결재 권한 부여
- **격리(Quarantine) → 처리(Action) 단계 분리** — 사유·격리 일자 등 컬럼 신설
- **불량 처리 허브 UI** — 격리 항목·처리 위저드(R 모달)·PA/PF 분해를 한 화면에서 처리
- **격리 추가 모달** + 대시보드에 불량 항목 빨간 표시
- 기존 입출고 위저드에서 불량 처리 단계는 분리하여 의도가 명확하도록 개편
- 회귀 테스트 113건(`test_defect_flow.py`) 신설하여 흐름 보호

**5. BOM 관리자 워크벤치 3-Column 개편 + 완료 워크플로**
관리자 화면의 BOM 워크벤치 레이아웃을 3-column으로 개편하고, BOM 작업 상태를 명시적으로
"완료" 처리할 수 있도록 워크플로를 추가하였습니다.
- BOM 완료 상태 컬럼 + 토글 엔드포인트 + 마이그레이션
- BOM 일괄 다운로드(export) 기능 보강

**6. 데이터 정합 — 권동환 사원 5월 재고 검증**
권동환 사원이 작성한 엑셀 재고와 시스템 DB가 일치하도록 단방향 동기화 스크립트를 정비하여,
5월 재고 데이터를 검증·반영하였습니다.
- 엑셀 ↔ DB 단방향 동기화 스크립트 + 자식 행 반영
- 13개 항목 메모를 DB에 일괄 반영
- 시스템 BOM과 실 재고 매칭 정확도 확보

**7. 입출고 CSV 외부 감시 시스템 (신규 인프라)**
외부 심사(예: 거래처, 감사) 시 입출고 이력을 깨끗한 CSV로 즉시 제출할 수 있도록,
월별 누적 미러 시스템을 구축하였습니다.
- 월 단위 자동 누적, 부서 내 작업 흐름 표시 추가

**8. `erp_code` → `ItemCode` 도메인 정리**
초기 "ERP" 명칭으로 만들어졌던 코드 도메인을 시스템 정식 명칭(MES 품목 코드)에 맞춰
일괄 정리하였습니다. (사용자 화면에는 변화 없음, 내부 코드 일관성)

**9. 외부 PC 첫 실행 안정화**
다른 직원 PC에서 처음 시스템을 실행할 때 안정적으로 켜지도록 `start.bat` 사전 검사와
`.gitattributes`를 보강하였습니다. (다음 단계 직원 테스트 배포 사전 준비)

**10. 사내 인수인계용 설명 문서 정리**
담당자 변경·휴가 등으로 인수인계가 필요할 때 후임이 바로 시스템을 이해하고 사용할 수
있도록, 내부 설명 문서의 폴더 구조와 항목별 설명을 일관된 톤으로 다듬어 두었습니다.

**11. 시스템 안정화 / 개발 환경 정비 (지속)**
- audit 로그·테스트 보강
- 격리 카탈로그(_attic) 후속 정리
- OpenAPI baseline 정합 유지

화면은 제 자리 PC가 켜져 있을 때 http://192.168.0.63:3000 에서 확인하실 수 있습니다.
