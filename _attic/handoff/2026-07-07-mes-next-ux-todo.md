- 품목 전환 진입 화면을 불량 탭의 `바로 처리` 화면처럼 다듬기
  - 증상: 현재 `품목 전환` 화면은 `사양 전환 / 구성 전환` 두 카드만 넓게 놓여 있어 같은 제품군의 전용 작업 화면처럼 보이지 않는다.
  - 사용자 혼란: 불량 탭의 `바로 처리`는 `이전 / 제목 / 진행 단계 / 큰 선택 카드` 구조가 명확한데, 품목 전환은 정보 위계가 약해 다음 작업 흐름을 예측하기 어렵다.
  - 개선 방향: 불량 `바로 처리` 화면을 기준으로 `이전`, `품목 전환`, `방식 선택` 흐름을 정리하고, `사양 전환`과 `구성 전환` 카드를 같은 화면 문법으로 재배치한다.
  - 우선순위: 높음.

- 품목 전환의 `사양 전환` 작업 화면 디자인 정리
  - 증상: `사양 전환` 상세 화면이 입력 필드와 비교/확인 영역만 나열된 느낌이라 작업자가 무엇을 먼저 해야 하는지 약하다.
  - 사용자 혼란: 소스 품목, 대상 품목, 전환 수량, 메모, 비교, 확인 버튼의 관계가 한눈에 잡히지 않는다.
  - 개선 방향: 디자인 스킬 기준으로 불량/입출고 전용 작업 화면 문법을 맞추고, `방식 선택 -> 소스/대상 선택 -> 비교 -> 확인` 흐름이 읽히게 재배치한다.
  - 우선순위: 높음.

- 품목 전환의 `구성 전환` 작업 화면 디자인 정리
  - 증상: `구성 전환` 상세 화면도 `사양 전환`과 거의 같은 나열형 UI라 BOM 차이 반영 작업의 의미가 드러나지 않는다.
  - 사용자 혼란: 단순 사양 변경과 달리 구성품 차감/회수 영향이 있는 작업인데, 화면에서는 그 위험도와 확인 순서가 충분히 강조되지 않는다.
  - 개선 방향: `구성 전환`은 BOM 차이, 추가 차감, 회수 입고, 부족 여부가 중심이 되도록 별도 요약 영역을 두고, `사양 전환`과 같은 레이아웃 기반에 위험 신호만 더 강하게 표현한다.
  - 우선순위: 높음.
