# 출하 PC UI 다음 개선 투두

작성일: 2026-06-30
상태: 수집 중

## 이전 투두 정리

기준 파일: `_attic/handoff/2026-06-29-shipping-performance-todo.md`
정리일: 2026-06-30

### 구현 완료로 정리

- 출하 첫 진입 성능 개선
  - 품목 2000건 로딩을 초기 진입에서 분리하고, 요청 작성/수정 진입 시점으로 늦김.
  - 출하 이력은 별도 초기 호출 대신 요청 목록의 `PICKED_UP` 상태에서 파생.
  - `verify_local.ps1` 기준 프론트 검증 통과.

- 상태 전환/잔여 요청 복구
  - `REQUESTED` 상세에서 다시 `준비 중으로 보내기` 가능.
  - `REQUESTED/PREPARING` 요청 취소 흐름 추가.
  - `PREPARED`는 준비 완료 취소 흐름으로 분리.
  - 기존 잔여 요청은 화면 기능으로 `PREPARING` 전환 확인.

- 출하 PC 화면 구조 개선
  - 허브, 요청 목록, 준비 중 목록, 이력 목록, 요청 상세를 전폭 작업 패널 문법으로 정리.
  - 내부 UUID 노출 제거.
  - 브라우저 URL/history 기반 출하 내부 화면 이동 반영.
  - 요청 상세의 액션 영역과 상태별 버튼 노출 정리.

- 출하 요청 작성/수정 마법사 개선
  - `PF 선택 -> BOM 구성 -> BOM 매칭 -> 요청 정보 -> 저장 및 전환` 단계형 흐름 적용.
  - 단계 탭과 헤더 중복 정리.
  - 페이지 전체 스크롤을 줄이고 긴 목록은 내부 스크롤로 처리하는 방향 적용.
  - 기준 PF 미선택 등 필수 조건 미충족 시 다음 단계 이동 비활성화.

- BOM 편집/매칭 UX 개선
  - 기본 BOM 다시 불러오기 버튼 제거.
  - 동일 BOM 확인 버튼 제거, 자동 매칭 결과 중심으로 정리.
  - 기존 BOM row의 품목 드롭다운 제거.
  - 품목 변경은 `제외 + 추가` 흐름으로 정리.
  - 새 PA/PF 이름과 품목코드 생성/재사용 안내를 최종 단계에 표시.

- 요청자/저장 액션 정리
  - 새 요청 생성 시 로그인 사용자 이름을 요청자로 사용.
  - 요청자 입력란은 읽기 전용 정보로 정리.
  - 5단계의 `요청 저장`, `준비 중으로 보내기`를 하단 액션바로 이동.

- 허브 카드 4차 정리
  - `바로 열기` CTA 제거.
  - 카드 하단 설명 문구 추가.
  - 건수 배지 가독성 개선.

### 보류 또는 후속 검토

- DB 락/세션 전략 개선
  - SQLite 읽기 요청까지 `BEGIN IMMEDIATE`를 잡는 구조는 출하 성능 문제의 근본 원인 후보지만, 전체 DB 동시성 정책과 관련되어 별도 백엔드 과제로 보류.
  - 읽기 전용 세션/엔진 분리 또는 조회 라우터 트랜잭션 전략 변경은 별도 계획 필요.

- 취소 이력 정책
  - 현재 `REQUESTED/PREPARING` 취소는 요청 삭제 중심으로 처리.
  - `CANCELLED` 상태를 만들어 이력으로 남기는 안은 보류.
  - `PICKED_UP` 취소는 v1 범위 밖으로 유지.

- 실사용 화면 미세 조정
  - 기존 큰 항목들은 구현됐지만, 실제 작업자가 화면을 보며 느끼는 여백, 밀도, 문구, 액션 위치는 계속 추가 투두로 수집한다.
  - 이 파일의 `수집 항목`에 앞으로 새 개선사항을 누적한다.

## 원칙

- 이 파일은 다음 출하 PC UI 개선사항을 구현 전까지 모아두는 투두 리스트다.
- 사용자가 명시적으로 구현을 지시하기 전까지 여기 적힌 항목은 구현하지 않는다.
- PC 실사용 흐름과 디자인 완성도를 우선한다.
- 모바일 출하 화면과 모바일 하단 탭바 frozen 스타일은 건드리지 않는다.
- 주간보고 frozen 파일은 건드리지 않는다.
- 구현 시 `$design` 스킬을 먼저 확인하고, 불량/입출고 탭의 업무형 화면 문법과 맞춘다.
- 색상은 `LEGACY_COLORS`, CSS 변수, 기존 token 흐름만 사용하고 inline hex는 추가하지 않는다.
