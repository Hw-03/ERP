## PC 입출고 > 품목 전환

### 1. 검색 선택 후 후보 목록 복귀와 선택 행 위치 고정
- 화면/영역: 소스·대상 품목 선택 단계의 검색창과 후보 목록
- 증상: 검색 결과에서 품목을 선택해도 검색어가 남고, 필터링된 목록만 유지되어 전체 후보 목록 안에서 선택 위치를 바로 파악하기 어렵다.
- 사용자 불편: 다음 선택이나 선택 변경을 위해 검색어를 직접 지우고 목록 위치를 다시 찾아야 한다.
- 확정 정책: 소스 또는 대상 품목을 검색으로 선택하면 해당 검색창을 비우고 전체 후보 목록으로 복귀한다. 선택된 행은 목록의 위에서 네 번째 행 위치가 되도록 스크롤을 이동한다.
- 구현 방향: 선택 처리 뒤 검색 상태를 초기화하고, 선택 행 ref 또는 목록 스크롤 계산으로 상단에 세 행이 남도록 스크롤 위치를 맞춘다. 소스와 대상 패널에 같은 동작을 적용한다.
- 수용 기준: 검색 선택 직후 검색창은 비어 있고 전체 목록이 보이며, 선택된 행은 `선택됨` 표시를 유지한 채 목록의 네 번째 행에 보인다.
- 브라우저 확인 시나리오: 소스와 대상에서 각각 검색어를 입력해 품목을 선택하고, 검색어 초기화·전체 목록 복귀·선택 행의 네 번째 위치를 확인한다.
- 우선순위: 높음

### 2. 소스 품목 용어를 기존 품목으로 통일
- 화면/영역: 품목 전환 전 단계의 소스 품목 라벨과 요약 정보
- 증상: 전환 전 품목을 `소스 품목`으로 표기한다.
- 사용자 불편: 현장 작업 관점에서 `기존 품목`보다 의미가 덜 직관적이다.
- 확정 정책: 품목 전환 화면에서 전환 전 품목을 가리키는 `소스` 용어는 모두 `기존`으로 바꾼다. 대상 품목 표기는 유지한다.
- 구현 방향: 단계 요약, 차이 확인, 최종 실행 확인 및 관련 보조 문구의 사용자 노출 텍스트를 일괄 점검해 `기존 품목` 문법으로 변경한다. 코드 내부 식별자와 API 필드는 변경하지 않는다.
- 수용 기준: 품목 전환 사용자 화면에 전환 전 품목 의미의 `소스` 문구가 남지 않고 `기존 품목`으로 일관되게 표시된다.
- 브라우저 확인 시나리오: 소스·대상 선택, 차이 확인, 최종 실행 각 단계에서 전환 전 품목 라벨을 확인한다.
- 우선순위: 중간

### 3. 차이 확인 단계 메모 라벨 가독성 보완
- 화면/영역: 차이 확인 단계 하단의 메모 입력 행
- 증상: `메모 (필수)` 라벨의 글자 크기가 작아 입력칸의 중요도가 충분히 드러나지 않는다.
- 사용자 불편: 필수 입력 항목을 한눈에 인지하기 어렵다.
- 확정 정책: 메모 라벨을 현재보다 한 단계 크게 해 입력칸과 같은 행에서 분명하게 보이게 한다.
- 구현 방향: 기존 한 줄 배치는 유지하고 메모 라벨의 폰트 크기와 굵기만 주변 하단 안내보다 높인다.
- 수용 기준: 차이 확인 하단에서 `메모 (필수)`가 사유 입력 placeholder보다 명확하게 식별되며 줄바꿈이나 입력칸 높이 변화가 없다.
- 브라우저 확인 시나리오: 차이 확인 단계에서 메모 입력 행의 라벨 크기와 정렬을 확인한다.
- 우선순위: 낮음

### 4. 최종 실행 화면의 품목 정보와 메모 영역 재배치
- 화면/영역: 최종 실행 단계의 BOM 차이 패널 머리말과 하단 실행 바
- 증상: 하단 실행 바에 기존 품목·대상 품목·메모가 동일한 비중으로 나열되어, 바로 위 BOM 차이 패널의 맥락과 분리되어 보인다.
- 사용자 불편: 어떤 BOM 구성 변화가 어느 품목 전환에 해당하는지 상단을 다시 대조해야 하고, 메모가 너무 작게 보인다.
- 확정 정책: 왼쪽 BOM 차이 패널의 `소스 BOM에서 빠지는 구성` 자리에는 기존 품목과 전환 수량을, 오른쪽 패널의 `대상 BOM 때문에 필요한 구성` 자리에는 대상 품목과 전환 수량을 표시한다. 하단 실행 바에는 메모를 넓고 크게 표시하고 `전환 실행` 버튼을 유지한다.
- 구현 방향: 최종 실행 단계에서 품목 정보는 각 BOM 패널의 헤더 영역으로 이동하고, 하단 실행 바는 메모 영역과 CTA 중심의 2열 구성으로 단순화한다. 패널의 회수 입고·추가 차감 보조 의미는 품목 정보와 충돌하지 않도록 유지하거나 명확한 보조 문구로 재배치한다.
- 수용 기준: 최종 실행에서 왼쪽 패널 머리말에 기존 품목, 오른쪽 패널 머리말에 대상 품목이 각각 표시된다. 하단에는 넓은 메모와 `전환 실행`만 남고, BOM 차이 목록과 버튼은 같은 화면에서 항상 확인할 수 있다.
- 브라우저 확인 시나리오: BOM 차이가 있는 전환을 최종 실행 단계까지 진행해 각 패널 품목 정보, 넓어진 메모, 실행 버튼 위치를 확인한다.
- 우선순위: 높음
