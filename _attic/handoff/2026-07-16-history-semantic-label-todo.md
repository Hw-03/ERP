### 7. 행 선택 전환 때 우측 상세 카드를 반짝이지 않게 하기

- 사용자 불편: 목록을 비교하며 거래를 훑을 때마다 시선이 끊기고, 카드 전체가 다시 열리는 것처럼 보여 선택 반응이 불안정하게 느껴진다.
- 확정 정책: 우측 상세 패널의 외곽 카드·너비·스크롤 프레임은 열린 상태에서 유지한다. 행이 바뀌면 새 거래의 제목·요약을 즉시 같은 카드 안에서 바꾸고, 비동기 세부 정보가 필요한 부분만 해당 영역에서 갱신한다. 이전 거래의 세부 정보가 새 거래 정보로 보이면 안 된다.

- 구현 검증: `DesktopHistoryRightPanel`을 서로 다른 단건 행과 서로 다른 묶음으로 rerender하는 테스트에서 SlidePanel과 DesktopRightPanel 외곽이 유지되고, 새 행 정보가 표시되며 이전 행의 비동기 응답이 덮어쓰지 않는지 검증한다.

- 브라우저 확인 시나리오: 1919×905 PC에서 목록의 인접 단건 행을 5회 이상 연속 선택하고, 품목 전환·창고/부서 이동·불량 묶음도 번갈아 선택한다. 카드 외곽이 깜빡이지 않는지, 선택 직후 제목·품목 코드·재고 변화가 일치하는지, 비동기 정보가 로드되는 동안 이전 거래의 내용이 남지 않는지 확인한다.

### 10. 모든 거래에 창고 또는 실제 부서를 남기기

- 결과: 부서 우선순위를 배치 도착 → 출발 → 로그 → 요청자 부서로 보완하고 공백·`미상` fallback을 제거했다. `20260720_0003` 마이그레이션은 근거가 있는 기존 53건을 창고 51건·출하 2건으로 영구 보정하며, 근거 없는 거래가 하나라도 있으면 쓰기 전에 실패한다. 실제 `backend/mes.db`에도 적용해 미분류 대상 0건을 확인했다.
