- 사용자 불편: 사용자에게 필요한 직책 정보가 보이지 않고, 영문 시스템값이 노출되어 화면 문맥과 맞지 않는다.
- 확정 정책: 직원의 `role` 값에서 직책 부분을 사용해 `부서 · 직책`을 표시한다. 예: `조립/사원` → `조립 · 사원`.
- 구현 방향: 로그인 시 현재 작업자 정보에 API 응답의 `role`을 보존하고, `MobileUserMenuSheet`에서는 부서명과 역할 문자열의 마지막 구간(직책)만 조합해 표시한다. 역할 문자열이 구분자 없이 들어오는 기존 데이터의 처리 방식은 구현 계획에서 실제 데이터 확인 후 명시한다.
- 수용 기준:

## 조립 체크리스트

### 2. 체크한 항목의 초록 완료 표시와 전체 해제 제공

- 화면/영역: 모바일 > 더보기 > 체크리스트 > 제품 선택 후 나타나는 모든 체크리스트 섹션과 항목.
- 증상: 현재 항목은 읽기 전용 목록으로 렌더링되어 눌러도 완료 여부를 알 수 없고, 모든 선택을 한 번에 취소할 방법도 없다.
- 사용자 불편: 작업자가 점검한 항목을 화면에서 즉시 구분할 수 없으며, 다시 점검하려면 완료 표시를 개별적으로 되돌릴 수밖에 없다.
- 확정 정책: 항목을 누르면 완료 상태를 토글한다. 완료 항목은 출하 품목 선택 카드처럼 면 전체를 강조하지 않고, 카드 테두리와 왼쪽 번호 원 및 번호 글자만 초록색으로 표시한다. 텍스트 색·배경·배치는 기존 디자인을 유지한다.
- 정책 확인 필요: 완료 상태를 제품 선택 화면으로 돌아가거나 페이지를 새로고침한 뒤에도 유지할지 결정해야 한다. 현재 화면에는 완료 상태를 저장하는 API·데이터 모델이 없다.
- 구현 방향: 제품별·섹션별·항목별 완료 상태를 구분할 수 있는 키를 두고, 모든 제품의 `sections[].items[]` 렌더링에 공통으로 토글과 완료 스타일을 적용한다. 선택된 제품 화면의 최하단에 현재 제품의 완료 표시를 모두 해제하는 `전체 해제` 버튼을 둔다. 출하의 API 저장/체크박스 디자인은 재사용하지 않는다.
- 수용 기준:
  - DX3000, ADX4000W, ADX6000 등 모든 선택 가능한 제품의 모든 섹션 항목이 눌림으로 완료·해제 전환된다.
  - 완료된 항목은 테두리와 번호 원/숫자만 초록색이고, 항목 배경이 출하 카드처럼 채워지지 않는다.
  - 항목을 다시 누르면 초록 표시가 사라진다.
  - 각 제품의 목록 최하단 `전체 해제`를 누르면 해당 제품의 모든 완료 표시가 사라진다.
  - 뒤로가기, 제품 전환, 새로고침 뒤 완료 상태의 유지 범위는 정책 결정과 일치한다.
- 브라우저 확인 시나리오: 각 제품을 차례로 열어 전원 OFF·전원 ON 등 서로 다른 섹션의 항목을 눌러 초록 테두리와 번호 원 표시를 확인한다. 일부 항목을 해제하고, `전체 해제`로 남은 항목이 모두 초기화되는지 확인한다. 서로 다른 제품으로 전환했을 때 완료 표시의 범위가 의도대로 분리되는지도 확인한다.
- 우선순위: 중간

### 3. 체크리스트 카드 그림자 제거

- 화면/영역: 모바일 > 더보기 > 체크리스트의 제품 선택 화면 및 제품 상세 체크리스트 화면 전체.
- 증상: 체크리스트의 상단 안내 카드, 제품 선택 카드, 섹션 카드에 공통 카드 그림자가 적용되어 있다.
- 사용자 불편: 조립 체크리스트는 작업 목록 중심 화면인데 카드가 떠 보이는 효과가 강해 화면이 가볍고 평평한 작업 문맥과 맞지 않는다.
- 확정 정책: 조립 체크리스트 안의 카드류는 그림자를 사용하지 않는다. 테두리·배경·둥근 모서리와 기존 간격은 유지한다.
- 구현 방향: `MobileAssemblyChecklistScreen`의 공통 카드 스타일에서 그림자만 제거해 제품 선택 화면과 DX3000·ADX6000 등 모든 제품 상세 화면에 동일하게 반영한다. 다른 모바일 화면과 카드 공통 스타일은 변경하지 않는다.
- 수용 기준:
  - 체크리스트 진입 안내 카드와 모든 제품 선택 카드에 그림자가 없다.
  - 제품 상세의 전원 OFF·전원 ON 등 모든 섹션 카드에 그림자가 없다.
  - 카드 테두리, 배경색, 모서리, 항목 간격과 탭 바는 기존대로 유지된다.
- 브라우저 확인 시나리오: 체크리스트 첫 화면에서 안내·DX3000·ADX6000 카드를 확인하고, 각 제품 상세에서 모든 섹션 카드를 스크롤해 그림자가 남아 있지 않은지 확인한다. 다른 모바일 화면의 카드 그림자는 변하지 않는지 함께 확인한다.
- 우선순위: 중간
