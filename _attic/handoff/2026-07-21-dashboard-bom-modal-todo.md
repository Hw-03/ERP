## 확정 결정 요약
- 대시보드 `BOM 구성 보기`는 현재의 좁은 중앙 카드가 아니라, 사용자가 표시한 데스크톱 작업 영역에 가까운 대형 읽기 전용 모달 안에서 본다.
- 모달의 세부 BOM 표현은 입출고 내역의 BOM 부모·구성품 행이 가진 계층, 접기·펼치기, 품목 식별 정보의 우선순위를 참고한다. 코드를 그대로 복제하거나 입출고 내역의 거래 수량·상태 의미를 대시보드 BOM에 섞지는 않는다.
- 하단 `닫기` 버튼과 이를 위한 전용 푸터는 두지 않는다. 닫기는 우측 상세 카드처럼 모달 우상단의 X 아이콘 버튼으로 제공하고, ESC·배경 클릭도 계속 지원한다.
- 모달을 여는 우측 패널 버튼은 펼침 조작이 아니므로 chevron을 제거하고, `BOM 보기` 대신 `하위 구성 보기`로 표기한다.
- 범위는 데스크톱 대시보드의 BOM 보기이며, 모바일 인라인 BOM과 입출고 내역의 실제 동작·데이터는 변경하지 않는다.

## 대시보드

### 1. BOM 구성 보기 모달을 작업 영역 크기로 확장하고 입출고 내역 BOM 문법을 반영
- 화면/영역: 데스크톱 대시보드 품목 상세 패널의 `BOM 보기`를 눌렀을 때 열리는 `BOM 구성 보기` 읽기 전용 모달.
- 증상: 현재 모달은 화면 중앙의 약 350px 폭 소형 카드로 열리고, 본문도 `max-h-[60vh]` 안에서만 스크롤된다. 하단의 `닫기` 버튼과 푸터가 높이를 차지해 BOM 트리에 쓸 수 있는 세로 공간도 줄어든다. 모달을 여는 우측 패널 버튼은 실제로는 펼치지 않고 모달을 열지만 chevron과 `BOM 보기` 표기를 사용한다. 다단 BOM의 품목명·부서·MES 코드·소요수량·현재 재고가 좁게 압축되어, 넓은 데스크톱 화면에서 구조를 한 번에 읽기 어렵다.
- 사용자 불편: BOM을 확인하는 데 충분한 화면 공간이 있는데도 작은 팝업 안에서 계속 스크롤해야 한다. 단순히 닫기 위한 하단 구역 때문에 목록을 덜 볼 수 있고, 모달 열기 버튼이 펼침처럼 보여 동작을 예상하기 어렵다. 입출고 내역에서 이미 익숙한 BOM 부모·구성품의 계층적 읽기 방식과도 인상이 다르다.
- 확정 정책: 모달 표면을 사용자가 표시한 대시보드 작업 영역 수준으로 확장한다. 데스크톱에서는 화면 좌우에 약 64px의 안전 여백을 남기는 넓은 모달을 기본으로 하며, 내부 트리만 독립 스크롤한다. 하단 `닫기` 버튼과 전용 푸터는 제거하고, 우측 상세 카드와 같은 문법의 X 아이콘 버튼을 모달 우상단에 둔다. ESC·배경 클릭·우상단 X 모두 모달을 닫는다. 모달을 여는 버튼은 chevron 없이 `하위 구성 보기`로 표기한다. BOM 부모와 구성품은 입출고 내역 BOM의 부모/자식 계층, chevron 펼침, 아이콘·배지·품목명·코드·수량 정보 우선순위를 참고해 표현한다.
- 구현 방향:
  - `InventoryDetailPanel`의 데스크톱 BOM `ConfirmModal` 호출에 대형 레이아웃과 우상단 닫기 제어를 지원하되, 다른 확인 모달의 공통 크기나 동작을 전역 변경하지 않는다. 모달 진입 버튼에서는 `ChevronRight`를 제거하고 레이블을 `하위 구성 보기`로 바꾼다.
  - 대형 모달은 뷰포트에 맞춰 폭을 제한하고, 낮은 화면 높이에서도 제목·우상단 X 영역은 고정한 채 트리 본문만 스크롤되게 한다. 창고·우측 상세 패널·빠른 작업 위에 겹치더라도, 배경 차단과 ESC·배경 클릭 닫기 동작은 기존 모달 문법을 유지한다.
  - 트리는 기존 `BomSubExpander` 데이터·다단 펼침을 재사용한다. 표현을 조정해야 하면 입출고 내역 `BomBatchDetail`의 `BOM` 부모 배지, 계층 indent/chevron, 부모 품목과 구성품의 시각적 위계만 참조한다.
  - 대시보드 BOM에는 현재 재고·소요수량처럼 BOM 조회가 제공하는 정보만 표시한다. 입출고 내역의 실제 입출고 방향, 작업 상태, 부족 판정, 거래 수량, 행 클릭 상세 선택을 이식하지 않는다.
  - ESC·배경 클릭·우상단 X 버튼·로딩·오류·빈 BOM 상태와 품목 변경 시 닫힘/초기화의 현재 계약은 유지하고 관련 테스트를 대형 레이아웃·트리 표현 기준에 맞춰 보강한다.
- 수용 기준:
  - 1919px 폭 데스크톱에서 모달이 좌우 약 64px 안전 여백을 제외한 작업 영역에 가깝게 넓어지며, 작은 중앙 카드처럼 보이지 않는다.
  - 모달 진입 버튼에는 chevron이 없고 `하위 구성 보기`로 표시된다. 하단 `닫기` 버튼과 푸터가 없고, 우상단 X 버튼은 키보드 포커스·접근 가능한 이름을 가지며 계속 접근 가능하다. 긴 BOM은 트리 본문만 스크롤되고 화면 높이가 낮아도 배경·헤더·트리가 겹치지 않는다.
  - 부모 BOM과 하위 구성품은 입출고 내역 BOM과 같은 계층 인지 방식(부모 강조, chevron, 들여쓰기, 아이콘/배지/품목명 우선순위)을 따르되, 대시보드 데이터 의미는 BOM 현재 재고·소요수량으로 유지한다.
  - 모바일 대시보드의 인라인 `하위 구성 보기`, 입출고 내역 BOM의 표·클릭·수량/상태 표시는 바뀌지 않는다.
- 브라우저 확인 시나리오:
  1. 데스크톱 대시보드에서 다단 BOM 품목을 선택한다. chevron 없는 `하위 구성 보기` 버튼을 눌러 모달이 작업 영역 크기로 넓어지고, 배경의 품목 상세·빠른 작업을 가리되 모달 내부 정보가 잘리지 않는지 확인한다.
  2. 1단계와 그 아래 자식을 여러 번 펼치고 접는다. 입출고 내역 BOM의 부모/구성품처럼 계층·chevron·품목명·MES 코드·소요수량·현재 재고를 빠르게 구분할 수 있는지 확인한다.
  3. 긴 품목명, 많은 구성품, 짧은 데스크톱 높이에서 하단 푸터 없이 본문만 스크롤되는지 확인한다. 우상단 X 버튼에 키보드로 포커스를 두고 닫은 뒤, ESC·배경 클릭으로도 각각 닫는다.
  4. 다른 품목으로 바꾼 뒤 다시 열어 이전 트리 펼침이나 스크롤 상태가 섞이지 않는지 확인한다. 모바일 대시보드와 입출고 내역 BOM을 함께 확인해 변경이 없는지 확인한다.
