  - 개선 방향: 상단에는 `소스 -Q`, `대상 +Q`를 크게 보여주고, 아래에는 변경되는 구성품만 `추가 차감`, `회수 입고`, `변동 없음 제외`로 나눠 표시한다. 부족 품목은 일반 행과 섞지 말고 상단 경고 영역에서 먼저 보이게 한다.
  - 우선순위: 높음.

- 품목 전환 Step 3에서 메모/사유 입력을 실행 전 판단 정보로 배치
  - 증상: 메모가 소스·대상 선택 단계에 같이 있으면 선택 필드 중 하나처럼 보이고, 왜 적어야 하는지 약하다.
  - 사용자 혼란: 품목 전환은 생산/재작업을 우회할 수 있는 강한 재고 작업인데, 사유 없이 실행하면 나중에 내역 추적이 어려워진다.
  - 개선 방향: 메모는 `차이 확인` 단계의 재고 영향 아래에 배치하고, placeholder를 `예: 구형 히팅 싱크를 신형 사양으로 전환`처럼 실제 감사/추적 문장으로 유도한다. 후속 정책에서 필수 여부를 결정할 수 있게 TODO에 남긴다.
  - 우선순위: 중간.
  - 정책 확인 필요: 품목 전환 메모를 항상 필수로 할지, 구성 전환만 필수로 할지 결정해야 한다.

- 품목 전환 Step 4 `실행`을 최종 확인과 완료 요약으로 분리
  - 증상: 실행 버튼만 마지막에 있으면 작업자가 클릭 직전에 이 작업이 어떤 입출고 내역으로 남는지 확신하기 어렵다.
  - 사용자 혼란: 실행 후 소스/대상/구성품 재고가 어떻게 바뀌고 어디에서 다시 확인할 수 있는지 모르면 현장 작업 후 검증 흐름이 끊긴다.
  - 개선 방향: 실행 전에는 모달 또는 마지막 화면에서 `소스 차감`, `대상 입고`, `추가 차감`, `회수 입고`, `입출고 내역 작업명`을 한 번 더 요약한다. 실행 후에는 완료 전용 화면에서 `새 품목 전환`, `입출고 내역에서 보기`, `입출고로 돌아가기` 버튼을 명확히 제공한다.
  - 우선순위: 높음.

- 품목 전환 단계 전환/뒤로가기 상태를 진행줄과 일관되게 연결
  - 증상: 단계가 나뉘어도 뒤로가기가 현재 입력 상태와 progress 상태를 정확히 반영하지 않으면 화면이 갑자기 되돌아간 것처럼 느껴질 수 있다.
  - 사용자 혼란: 방식 선택으로 돌아왔는데 이전 선택 효과가 남거나, 브라우저 뒤로가기와 화면 내 뒤로가기 동작이 다르면 작성 흐름을 잃을 수 있다.
  - 개선 방향: progress의 done/active/locked 상태와 내부 state를 1:1로 맞춘다. 이전 단계로 돌아가면 이후 단계의 preview/실행 요약은 폐기하고, 카드 높이/상단 탭 숨김은 짧은 opacity/height 애니메이션으로 자연스럽게 이어지게 한다.
  - 우선순위: 높음.

- 입출고 Step 1 복귀 시 작업 유형 카드 선택 효과 제거
  - 증상: Step 2에서 `작업 유형 선택`으로 돌아오면 이전에 눌렀던 `창고 입출고` 또는 `부서 입출고` 카드가 여전히 선택된 것처럼 파란 배경/테두리 효과를 가진다.
  - 사용자 혼란: 현재 화면은 다시 작업 유형을 고르는 상태인데, 이미 특정 작업이 선택되어 확정된 것처럼 보인다.
  - 개선 방향: Step 1에서는 작업 유형 카드를 기본 중립 상태로 표시하고, 선택 효과는 Step 2 이상에서 완료된 progress 요약으로만 표현한다.
  - 우선순위: 높음.

- 입출고 Step 1 progress 첫 칸의 기본 작업명 숨김
  - 증상: 아무 작업 유형도 새로 선택하지 않았는데 progress 첫 칸에 `원자재 입고`나 이전 선택값이 보인다.
  - 사용자 혼란: 사용자는 아직 선택하지 않았다고 느끼는데 화면은 이미 작업 유형이 정해진 것처럼 말한다.
  - 개선 방향: Step 1의 `작업 유형 선택` progress 칸은 선택 전에는 보조 문구를 비워두고, 사용자가 작업 유형을 선택해 Step 2로 넘어간 뒤에만 `창고 입출고`, `부서 입출고`, `품목 전환` 같은 요약을 표시한다.
  - 우선순위: 높음.

- 입출고 Step 5 최종 확인 품목 row의 불필요한 테두리 제거
  - 증상: Step 5 최종 확인에서 품목 묶음 row 내부 버튼에 파란 사각 테두리가 보여 선택 박스나 디버그 표시처럼 느껴진다.
  - 사용자 혼란: 이미 상위 카드와 row 구분선이 있는데 내부 버튼 테두리까지 겹쳐져, 실제로 클릭해야 하는 영역인지 단순 정보 row인지 판단이 흐려진다.
  - 개선 방향: row 클릭/펼침 기능은 유지하되, 내부 버튼의 시각적 사각 테두리는 제거하고 hover/pressed 상태만 기존 DEXCOWIN MES 톤으로 약하게 남긴다.
  - 우선순위: 높음.

- PC 작성 이탈/저장 확인 팝업을 모바일 수준으로 개선
  - 증상: PC에서 작성 중 다른 탭으로 이동할 때 뜨는 저장 팝업이 `저장`, `저장하지 않고 이동`, `저장하고 이동` 정도만 보여 내용이 너무 빈약하다.
  - 사용자 혼란: 사용자는 저장하면 어디에서 이어서 볼 수 있는지, 저장하지 않으면 무엇이 사라지는지, 계속 작성으로 돌아갈 수 있는지 바로 판단하기 어렵다.
  - 개선 방향: 모바일의 `작성 중인 입출고가 있어요` bottom sheet 문법을 참고해 PC 공통 모달에도 명확한 제목, 설명 문장, 주요 버튼(`임시저장하고 이동`), 위험 버튼(`저장 안 하고 나가기`), 보조 버튼(`계속 작성`)을 제공한다. 문구는 PC 맥락에 맞게 `임시저장하면 내 요청에서 이어서 진행할 수 있습니다. 저장하지 않고 이동하면 현재 작성 내용은 사라집니다.`처럼 정리한다.
  - 우선순위: 높음.

- PC 작성 이탈/저장 확인 팝업을 공통 컴포넌트 기준으로 정리
  - 증상: 같은 성격의 이탈/저장 확인 팝업이 입출고 외 다른 화면에서도 쓰이지만 화면마다 정보량과 버튼 문법이 다를 수 있다.
  - 사용자 혼란: 어떤 화면에서는 저장/이동 의미가 명확하고, 어떤 화면에서는 단순 버튼만 보여 같은 행동의 결과를 다르게 해석하게 된다.
  - 개선 방향: 입출고 PC 팝업만 단발로 고치지 말고, 공통 confirm/save-leave 모달 사용처를 먼저 찾은 뒤 모바일 문구 구조를 기준으로 PC 전역 문법을 통일한다. 각 업무명만 `입출고`, `출하 요청`, `불량 처리` 등으로 바꿔 들어가게 한다.
  - 우선순위: 높음.

### 출하

- 출하 허브 카드 제목/설명 위계 정리
  - 증상: 출하 허브 카드에서 제목 아래 작은 설명(`요청 목록, BOM 편집`, `준비 체크, 픽업`, `완료 요청, 재고 로그`)과 카드 하단 설명이 동시에 보여 정보가 중복된다.
  - 사용자 혼란: 카드 제목이 불량 허브처럼 큰 작업 선택지로 읽히지 않고, 작은 설명이 중간에 끼어 제목/설명 위계가 흐려진다.
  - 개선 방향: 카드 하단 설명은 유지하고 제목 아래 작은 설명은 제거한다. 카드 제목은 불량 허브 수준의 큰 타이포/무게로 키우고, 아이콘/카운트 배지는 현재 의미를 유지하되 제목 중심으로 재배치한다.
  - 우선순위: 높음.

- 출하 허브를 불량 허브와 같은 큰 선택 카드 문법으로 정리
  - 증상: 출하와 불량은 같은 허브형 3카드 화면인데, 출하는 요약 카드처럼 보이고 불량은 큰 작업 선택 카드처럼 보여 화면 문법이 다르다.
  - 사용자 혼란: 같은 1차 업무 진입 화면인데 탭마다 카드 크기, 제목 크기, 설명 위치가 달라 사용자가 화면을 다시 해석해야 한다.
  - 개선 방향: 디자인 스킬 기준으로 불량 허브의 큰 선택 카드 레이아웃을 참고해 출하 허브의 카드 높이, 제목 크기, 하단 설명 위치, 여백을 맞춘다. 출하 업무별 아이콘 색과 카운트 배지는 유지한다.
  - 우선순위: 높음.

### 불량

- 불량 허브 카드 배경/텍스트 색을 출하 허브와 통일
  - 증상: 불량 허브 카드의 배경/테두리/버튼 배경이 회색 계열로 보여 출하 허브의 흰 카드 톤과 다르다. 제목 텍스트도 아이콘 색과 같은 accent 색으로 보여 다른 탭의 검은 제목 문법과 다르다.
  - 사용자 혼란: 불량 허브가 같은 허브형 탭 안에서도 별도 화면처럼 보이고, 제목 색이 강해 아이콘 accent와 텍스트 위계가 섞인다.
  - 개선 방향: 카드 배경과 외곽 톤은 출하 허브처럼 흰색/밝은 중립 톤으로 맞춘다. 아이콘 색은 현재 불량 업무 accent를 유지하되, 카드 제목 텍스트는 다른 탭처럼 검은색/기본 텍스트 색으로 통일한다. 하단 설명 문구는 기존 위치와 크기를 유지한다.
  - 우선순위: 높음.
