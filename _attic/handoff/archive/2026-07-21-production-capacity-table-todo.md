- `출하 대기`·`빠른 생산`·`총생산` 수량 열과 실제 계산·병목 데이터 계약은 변경하지 않는다.

- 세 수량 설명은 포장 완료·테스트 완료·사내 재고 활용이라는 현장 언어로 고친다. 공용 자재 경고는 의미를 유지하되 최종 문구는 확인이 필요하다.

## 대시보드

### 1. 조립 완제품 목록을 모델 수·기준 모델 중심 헤더로 정리
- 화면/영역: 데스크톱 대시보드 `생산 가능수량` 모달의 `조립 완제품(AF) 기준` 목록. 조립 완제품명, `N종`, PF 변형 칩, 생산 가능수량 열이 한 행에 표시되는 영역.
- 증상: 목록 첫 헤더가 `조립 완제품 · 병목`으로 표시되지만, 이 영역에는 병목 부품명이 직접 보이지 않는다. 반면 행의 `N종`과 연결 PF 변형 칩은 각각 무엇을 뜻하는지 헤더에서 분리해 알려주지 않는다.
- 사용자 불편: 사용자가 모델군별 몇 종인지와 어떤 모델이 기준으로 고정되었는지를 한눈에 읽기 어렵고, 표시되지 않는 병목을 헤더가 강조해 목록의 목적을 혼동시킨다.
- 확정 정책: 첫 열은 조립 완제품명을 유지하고 `병목`은 제거한다. `N종`이 정렬되는 별도 열에는 `모델 수`, 기준 PF 변형 칩이 표시되는 넓은 열에는 `기준 모델` 헤더를 둔다. 행의 데이터·칩 선택/삭제 동작·수량 계산은 그대로 유지한다.
- 구현 방향:
  - `CapacityDetailModal`의 데스크톱 조립 완제품 목록 헤더와 행 그리드를 함께 조정해 `조립 완제품`, `모델 수`, `기준 모델`, `출하 대기`, `빠른 생산`, `총생산`이 같은 열 기준으로 정렬되게 한다.
  - `모델 수`에는 기존 그룹별 PF 변형 수(`N종`)만 표시한다. 새 카운트 기준이나 하드코딩된 종수는 만들지 않는다.
  - `기준 모델`에는 기존 `pinnedVariant` PF 변형 칩을 그대로 표시한다. 이 열은 기준으로 고정한 모델을 설명하므로, 모델군 전체 연결 관계나 병목 부품을 뜻하는 용어는 사용하지 않는다.
  - 병목 데이터는 계산과 상세 근거에 필요할 수 있으므로 API 타입·계산·다른 병목 표시 영역을 변경하지 않는다. 이번 범위는 해당 조립 완제품 표의 헤더와 정보 배치다.
- 수용 기준:
  - `조립 완제품 · 병목` 문구가 조립 완제품 목록 헤더에서 사라진다.
  - `DX3000 · 19종`과 같은 행에서 `19종`은 `모델 수` 열에, 기준으로 고정한 PF 변형 칩은 `기준 모델` 열에 명확히 대응한다.
  - 조립 완제품명·모델 수·기준 모델·세 생산수량 열이 표 전체에서 세로로 정렬되며, 긴 모델명·여러 칩에서도 수량 열이 밀리거나 겹치지 않는다.
  - 생산 가능수량 계산값, 펼치기, 칩 상호작용, 모바일 목록과 다른 병목 정보 표시는 바뀌지 않는다.
- 브라우저 확인 시나리오:
  1. `생산 가능수량` 모달을 열어 조립 완제품 목록 헤더에 `조립 완제품`, `모델 수`, `기준 모델`과 세 수량 열이 순서대로 보이는지 확인한다.
  2. DX3000처럼 모델 수가 많은 행과 적은 행을 비교한다. 각 `N종` 수와 기준 PF 칩이 맞는 열에 있고, `병목` 문구가 남아 있지 않은지 확인한다.
  3. 긴 기준 모델명·좁은 데스크톱 폭에서 행 높이, 줄바꿈/가로 스크롤, 수량 열 정렬이 깨지지 않는지 확인한다.
  4. 그룹 펼치기와 칩 제거 동작, 생산수량 값, 모바일 생산 가능수량 목록 및 별도 병목 안내가 기존대로인지 확인한다.
- 우선순위: 중간

### 2. 생산 가능수량의 현장 설명 문구를 정리하고 중복 보조 제목 제거
- 화면/영역: 데스크톱 대시보드 `생산 가능수량` 모달의 제목 아래 보조 제목, `출하 대기`·`빠른 생산`·`총생산` 설명 세 줄, 공용 자재 주의 배지.
- 증상: 현재 설명은 PF·AF·BOM·직계 1단계 같은 계산 용어를 앞세워, 실제 출하·테스트·포장 흐름을 바로 이해하기 어렵다. `조립 완제품(AF) 기준` 보조 제목은 목록과 설명에서 이미 전달하는 맥락을 반복하고, 공용 자재 경고는 길고 추상적이다.
- 사용자 불편: 현장 사용자가 어떤 상태의 재고인지 빠르게 판단하기보다 약어와 계산 방식을 먼저 해석해야 한다. 모달 상단의 보조 제목과 긴 경고는 핵심 수량 목록을 위한 화면 공간도 차지한다.
- 확정 정책:
  - `조립 완제품(AF) 기준` 보조 제목은 제거한다.
  - `출하 대기` 설명은 `박스 포장까지 완료되어 픽업을 기다리고 있는 재고`를 뜻하도록 바꾼다.
  - `빠른 생산` 설명은 `테스트가 완료된 완제품 재고와 포장 자재를 확인해, 빠르게 박스 포장까지 할 수 있는 수량`을 뜻하도록 바꾼다.
  - `총생산` 설명은 `튜브부터 박스까지 사내 재고를 사용해 이론적으로 생산할 수 있는 총합`을 뜻하도록 바꾼다.
- 정책 확인 필요: 공용 자재 경고의 최종 문구. 후보는 아래 두 줄로 표시한다.
  - `※ 공용 자재가 겹치는 모델은 표시 수량을 모두 동시에 생산할 수 없습니다.`
  - `한 모델에 자재를 사용하면 다른 모델의 생산 가능 수량은 줄어들 수 있습니다.`
  AF·PF 같은 내부 약어 없이도 공용 자재를 먼저 사용한 모델이 다른 모델 수량에 미치는 영향을 설명한다.
- 구현 방향:
  - `CapacityDetailModal`의 설명 전용 텍스트만 바꾸며, `출하 대기`·`빠른 생산`·`총생산`의 라벨, 색상, 계산식, API 필드명은 유지한다.
  - 현재 상단 보조 제목 렌더링과 공용 자재 주의 배지를 정리한다. 경고는 최종 정책이 확정된 경우에만 후보 문구로 교체하고, 경고의 색상·아이콘은 유지한다.
  - 설명은 한 줄 또는 자연스러운 두 줄 안에서 읽히도록 하되, 화면 폭에 맞춘 줄바꿈만 허용하고 추가 도움말·툴팁·새 정보 패널은 만들지 않는다.
- 수용 기준:
  - 모달 제목 아래에 `조립 완제품(AF) 기준`이 더 이상 표시되지 않는다.
  - 세 설명이 각각 포장 완료 후 픽업 대기, 테스트 완료 완제품+포장 자재로 빠른 포장, 사내 튜브~박스 재고로 이론적 총생산을 분명히 설명한다.
  - 공용 자재 경고는 최종 승인된 간결한 문구만 남기며, 표시 수량을 여러 모델에서 동시에 보장하지 않는다는 핵심 의미가 사라지지 않는다.
  - 수량 계산값, 색상, AF/PF 데이터, 모달 열기·닫기 동작은 바뀌지 않는다.
- 브라우저 확인 시나리오:
  1. `생산 가능수량` 모달을 열어 제목 바로 아래의 `조립 완제품(AF) 기준`이 제거되고, 세 수량 설명이 순서와 색상을 유지한 채 새 현장 언어로 표시되는지 확인한다.
  2. 출하 가능한 완제품, 테스트 완료 완제품과 포장 자재, 튜브·박스 재고라는 각 설명이 서로 겹치지 않고 이해되는지 현장 사용자 관점에서 확인한다.
  3. 확정된 공용 자재 경고 문구가 짧은 폭에서도 잘리지 않고, 여러 모델의 표시 수량을 동시에 생산할 수 없다는 의미를 유지하는지 확인한다.
  4. 데스크톱과 모바일에서 세 수량 값·색상·목록 스크롤·모달 닫기 동작이 기존과 같은지 확인한다.
- 우선순위: 중간
