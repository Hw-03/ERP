## 아직 TODO에 추가할 최근 관찰 4개

### A. 취소된 작업 상태 문법

- 취소 또는 무효화된 품목 전환/묶음 부모 행과 그 하위 영향 행을 일반 작업보다 흐리게 표시한다.
- 대표 품명과 수량 요약에는 취소선을 적용한다.
- 선택/호버/포커스는 취소 상태보다 우선하지만 취소선은 유지한다.
- 이는 재고를 되돌리는 표시가 아니라, 해당 이력이 현재 유효하지 않음을 나타내는 시각 문법이다.

### B. 흐름 열의 중복 재고 설명 제거

- 예: `여러 위치` 아래의 `부품 차감 + 완제품 입고`는 수량 열의 정보와 중복된다.
- 흐름 열은 조립, 여러 위치, 창고 → 조립 같은 위치/공정 흐름만 표시한다.
- 실제 재고 증감 설명은 수량 열에만 둔다.

### C. 재작업 내부 메모 태그 숨김

- 우측 상세 메모의 `[rework:scrap_child]`는 사용자 메모가 아니다.
- 확인한 저장 원인: `backend/app/services/dept_adjustment.py:499`에서 재작업 하위 폐기 트랜잭션을 구분하려고 `notes="[rework:scrap_child]"`를 저장한다.
- 프런트 공통 메모 정규화 helper에서 알려진 내부 `[rework:*]` 태그만 숨기고, 사용자 메모와 저장 데이터는 유지한다.
- 태그만 남은 메모라면 상세 패널의 메모 영역 자체를 표시하지 않는다.

### D. 수량 열 용어의 실제 재고 영향화

- 현재 생산 묶음의 `완제품 +N EA`, `부품 -N EA`는 품목 마스터 분류가 아니라 대표 생산품과 BOM 구성품을 나눈 내부 요약이다.
- 현장 화면에서는 `생산 입고 +N EA`, `구성품 차감 -N EA`처럼 실제 재고 동작을 먼저 보여 주는 방향을 검토한다.
- 분해·재작업은 `회수 입고 +N EA`, `폐기 차감 -N EA`처럼 정리하는 방향을 제안한다.
- 이 항목의 최종 용어는 사용자와 정책 확인 후 확정한다. TODO에는 `정책 확인 필요`로 적는다.
