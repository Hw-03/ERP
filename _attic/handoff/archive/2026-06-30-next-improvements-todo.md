## 개선 예정

### 1. 재작업 BOM 확인 화면 처리 수량 변경 UX

- 화면: 불량 > 불량 처리/바로 재작업 > BOM 확인
- 사용자가 느낀 문제:
  - 상단 `처리 수량`을 바꿀 때마다 BOM 영역이 새로 불러오는 것처럼 계속 반짝거린다.
  - 실제로 BOM을 매번 재조회하는지, 아니면 상태 재계산/렌더링 때문에 그렇게 보이는지 확인이 필요하다.
  - 상위 품목을 감싸는 빨간 박스가 굳이 필요한지 애매하다.
  - 위쪽 파란 요약 박스에서 이미 품목/처리 수량/작업 정보를 보여주기 때문에, 빨간 상위 품목 박스는 중복 정보처럼 보일 수 있다.
- 해결 방향 메모:
  - 처리 수량 변경 시 BOM 데이터를 다시 fetch하는지, 기존 BOM 데이터를 유지한 채 수량만 재계산하는지 코드와 네트워크 로그로 확인한다.
  - 재조회가 맞다면 BOM 구조 데이터는 품목 기준으로 캐시하고, 처리 수량 변경은 하위 수량 계산만 갱신하도록 분리한다.
  - 재조회가 아니더라도 깜빡임이 보이면 loading/animation/key 재마운트 원인을 찾아 제거한다.
  - 상위 빨간 박스는 제거/축소/색상 완화 세 가지 안을 비교한다.
  - 완전히 없앨 경우, 하위 품목 수량이 `처리 수량 기준으로 배분된다`는 정보가 어디에 남는지 확인한다.
  - 파란 요약 박스와 역할이 겹친다면 빨간 박스 대신 얇은 설명 줄이나 접힌 메타 정보로 대체하는 방향을 검토한다.
- 검증 메모:
  - 처리 수량을 여러 번 변경해도 BOM 하위 행이 깜빡이거나 스크롤 위치가 흔들리지 않아야 한다.
  - 네트워크/API 호출이 불필요하게 반복되지 않아야 한다.
  - 상위 빨간 박스를 제거하거나 완화해도 현재 선택 품목과 처리 기준 수량을 사용자가 잃지 않아야 한다.
