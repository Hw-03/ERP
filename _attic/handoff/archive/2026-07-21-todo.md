- 확정 정책: `모델 수`와 `기준 모델`을 `조립 완제품` 바로 오른쪽의 빈 영역으로 옮기고, 수량 열(출하 대기·빠른 생산·총생산)은 오른쪽에 유지한다.
- 구현 방향: 데스크톱 표의 공통 그리드 열 순서와 각 요약·상세 행의 셀 순서를 함께 조정해 열 머리글과 값의 정렬을 유지한다.
- 수용 기준: 데스크톱 표에서 조립 완제품, 모델 수, 기준 모델이 연속해 표시되고 세 수량 열은 우측 정렬 상태로 유지된다.

## 대시보드 빠른 작업

### 5. 입고·출고 빠른 작업의 이중 카드와 분리 배경 원복
- 화면/영역: 대시보드 > 품목 상세 오른쪽 패널 > 빠른 작업.
- 증상: 입고와 출고 각각에 바깥 카드가 추가되고 색 배경이 분리되어, 버튼 안에 두 번째 박스가 생긴 것처럼 보인다. 선택하면 각 방향 카드가 하위 선택지 높이만큼 늘어난다.
- 사용자 불편: 이전보다 빠른 작업 영역이 복잡하고 커져, 입고·출고 버튼을 단순하게 선택하기 어렵다.
- 확정 정책: 빠른 작업을 직전 디자인 변경 전 형태로 되돌린다. 입고·출고에는 별도 바깥 카드·패딩·분리 배경을 두지 않고, 선택된 방향의 선택지만 두 버튼 아래 전폭으로 펼친다.
- 구현 방향: 데스크톱 빠른 작업에서 방향별 감싸기 카드와 배경을 제거하고, 입고·출고 버튼의 기존 강조 색과 하위 선택지의 단일 전폭 펼침 구조를 복원한다. 입고·출고의 선택 동작과 각 선택지의 입출고 화면 이동은 유지한다.
- 수용 기준: 접힌 상태에서 입고·출고는 각각 하나의 버튼으로만 보이고 별도 바깥 박스나 분리 배경이 없다. 어느 방향을 선택해도 방향별 카드 자체가 세로로 늘어나지 않으며 하위 선택지는 두 버튼 아래 전폭으로만 표시된다.
- 브라우저 확인 시나리오: 품목을 선택한 뒤 빠른 작업에서 입고와 출고를 각각 한 번씩 열고 닫는다. 부서 입고·창고 반입 등 하위 선택지는 표시되지만 두 방향의 외곽 카드가 늘어나거나 이중 테두리가 생기지 않는지 확인한다.
- 우선순위: 중간

## 대시보드 BOM 구성 보기

### 6. 내용이 적어도 BOM 구성 보기 모달 높이 유지
- 화면/영역: 대시보드 > 품목 상세 > 하위 구성 보기 > BOM 구성 보기 모달.
- 증상: BOM 구성품 수가 적으면 모달이 내용 높이만큼만 줄어들어, 화면 중앙에 지나치게 낮은 가로형 패널로 표시된다.
- 사용자 불편: 상세 조회 모달의 정보 영역이 너무 작아 보여 내용을 더 볼 수 있는 화면처럼 느껴지지 않는다.
- 확정 정책: 내용이 적어도 모달을 사용자가 지정한 예시 크기(데스크톱 1916×908 화면 기준 약 500px 높이)로 크게 유지한다. 기존의 넓은 가로 폭과 최대 높이 제한은 유지한다.
- 구현 방향: 모달 패널에 데스크톱 최소 높이를 부여하고, BOM 내용 영역이 남은 높이를 사용하도록 한다. 구성품이 많을 때는 현재처럼 본문만 세로 스크롤하며 화면 높이를 넘지 않게 한다.
- 수용 기준: 구성품이 1~2개인 BOM도 약 500px 높이의 모달로 열린다. 구성품이 많은 BOM은 최대 높이 안에서 본문 스크롤로 전체 내용을 볼 수 있고, 제목·닫기 버튼은 계속 보인다.
- 브라우저 확인 시나리오: 구성품이 적은 품목과 많은 품목에서 각각 BOM 구성 보기를 열어 최소 높이, 최대 높이 제한, 본문 스크롤 및 닫기 동작을 확인한다.
- 우선순위: 중간

### 7. BOM 트리 본문을 기존 MES 디자인 문법에 맞추고 넓은 폭 활용
- 화면/영역: 대시보드 > 품목 상세 > 하위 구성 보기 > BOM 구성 보기 모달 > 구성품 트리 본문.
- 증상: 넓어진 모달 안에서 구성품 본문이 가는 연결선과 압축된 행만으로 표시되고, 본문 자체에는 카드·표면·간격 구분이 거의 없다. 품목명·수량·현재 재고가 왼쪽에만 몰려 오른쪽의 넓은 여백을 사용하지 못한다.
- 사용자 불편: 모달 헤더와 부모 품목 카드는 현재 MES 디자인 문법을 따르지만, 트리 본문은 다른 화면의 오래된 트리처럼 보여 한 화면 안에서도 이질감이 크고, 넓은 조회 화면의 정보 밀도가 낮다.
- 확정 정책: BOM의 계층 구조, 펼치기/접기, 구성품 수량·현재 재고 정보는 유지한다. 단, 본문 표현은 기존 MES의 카드·표면·여백·상태 배지 문법에 맞춰 정돈하고, 모달의 가용 폭을 품목 정보와 수량·재고 배치에 적극 사용한다.
- 구현 방향: 모달 전용 BOM 트리를 현재 패널 목록 스타일로 재구성해 계층 단위와 행 구분을 명확히 하고, 연결선은 계층을 이해하는 데 필요한 수준으로만 절제한다. 품목명은 왼쪽에서 충분히 읽을 수 있게 두고, 코드·필요 수량·현재 재고는 넓은 행의 오른쪽 정보 영역에 정렬한다. 부모 품목 헤더, 긴 품목명의 줄바꿈 및 본문 스크롤은 유지한다.
- 수용 기준: 모달의 본문이 헤더·부모 품목 카드와 같은 표면·간격·테두리 체계를 사용해 자연스럽게 이어진다. 각 구성품 행은 모달 폭을 활용해 품목명과 코드·수량·현재 재고가 균형 있게 배치되고, 큰 오른쪽 빈 공간이 남지 않는다. 여러 단계 BOM에서도 부모-자식 관계와 펼치기/접기 상태를 빠르게 구분할 수 있다.
- 브라우저 확인 시나리오: 2단계 이상 하위 구성품이 있는 품목을 열고 여러 노드를 펼치고 접는다. 짧은 이름과 긴 이름, 재고가 있는 항목을 함께 확인해 행의 가로 정보 배치, 계층 가독성·줄바꿈·스크롤이 유지되는지 확인한다.
- 우선순위: 중간
