## 2026-07-07 추가 피드백: Step 2 / Step 5 상단·버튼·여백

### Step 2 세부 작업과 부서

- 상단 단계 헤더 위 여백이 불필요하게 커 보인다.
  - `세부 작업과 부서 STEP 2 / 5` 헤더를 더 위로 붙인다.
  - Step 2+에서 상단 탭이 사라진 뒤 남는 공간이 자연스럽게 줄어드는지 확인한다.
- 세부 작업 선택 카드 영역은 높이를 살짝 줄인다.
  - 현재 `창고 -> 부서 / 부서 -> 창고` 카드가 화면을 많이 차지한다.
  - 도착 부서 선택 영역을 더 중요하게 보여주기 위해 세부 작업:도착 부서 높이 비율을 약 `4:6`에 가깝게 조정한다.
- 도착 부서 버튼 영역은 높이를 살짝 늘린다.
  - `튜브 / 고압 / 진공 / 튜닝 / 조립 / 출하` 버튼의 터치 영역과 시각적 비중을 키운다.
  - 430px 기준에서 하단 `다음 단계로` 버튼과 네비게이션을 침범하지 않는지 확인한다.

### Step 5 최종 확인

- 품목 행의 품목명 버튼 테두리를 제거한다.
  - 현재 품목명 자체가 별도 테두리 버튼처럼 보여 카드 안에서 이중 강조가 된다.
  - PC처럼 행 전체를 클릭하면 하위 구성이 펼쳐지는 방식으로 맞춘다.
  - 화살표는 유지하되, 클릭 가능 영역은 품목명 버튼이 아니라 행/card 전체로 확장한다.
- 저장 버튼 라벨 정렬을 Step 4와 Step 5에서 통일한다.
  - `저장` 버튼 안의 아이콘/텍스트 위치가 단계별로 달라 보인다.
  - 저장 버튼은 공통 액션 버튼 컴포넌트 또는 같은 class 조합으로 맞춘다.
- 메모 입력과 하단 액션바 사이 여백을 줄인다.
  - 현재 메모 input 아래 여백이 하단 액션바와 네비게이션 사이 간격보다 커 보여 어색하다.
  - 기준: `저장/제출 버튼`과 `하단 네비` 사이 간격과 비슷한 수준으로 맞춘다.
  - sticky 액션바가 본문을 가리지 않도록 하되, 불필요한 빈 공간은 줄인다.

### 더보기 상단 프로필/알림

- 프로필 카드와 알림 버튼은 더보기 상단에 유지한다.
- 모바일에서는 알림 버튼이 현재보다 커야 터치하기 편하다.
  - PC 알림 버튼과 완전히 같은 크기를 유지할 필요는 없다.
  - 모바일 기준 최소 터치 영역을 확보한다.
  - 프로필 카드 높이와 알림 버튼 높이가 어색하게 어긋나지 않게 맞춘다.

### 더보기 메뉴 3개 버튼

- `주간보고 / 출하 / 창고 지도` 3개 메뉴 버튼은 화면 폭을 가득 채우는 형태로 바꾼다.
- 현재처럼 개별 카드가 각각 떠 있는 느낌보다, 하나의 메뉴 리스트처럼 보이는 방향을 검토한다.
- 후보 방향:
  - 하나의 큰 리스트 컨테이너 안에 3개 행을 넣고 행 전체 클릭
  - 각 행은 full-width, 동일 높이, 동일 아이콘 크기
  - 행 사이 구분선은 약하게 두고, 바깥 라운드는 리스트 전체에만 적용
- 프로필/알림 영역은 그대로 두고, 아래 3개 메뉴만 화면을 더 넓게 쓰게 한다.
