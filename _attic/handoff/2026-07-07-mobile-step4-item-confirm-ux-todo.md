## 배경

모바일 입출고 작성 플로우의 `STEP 4 / 5` 품목 확인 화면에서 상위 자재와 하위 자재가 함께 표시되지만, 화면 폭이 좁아 정보 위계와 조작 규칙이 잘 보이지 않는다.

사용자 피드백:

- 하위 구성 목록에서 품목명을 제대로 확인하기 어렵다.
- 긴 품목명이 한 줄에서 잘려 식별이 어렵다.
- 상위 자재와 하위 자재의 포맷이 달라 통일성이 떨어진다.
- 같은 `-1` 수량 버튼인데 위치에 따라 크기와 스타일이 다르다.

## TODO

1. 수량 버튼 컴포넌트 공통화
   - 상위 자재와 하위 자재의 `-10 / -1 / 1 / +1 / +10` 버튼 크기, 높이, 폰트, 비활성 스타일을 통일한다.
   - 모바일 전용 `QuantityStepper` 또는 기존 로컬 helper 추출을 검토한다.
   - 수량 입력 영역은 카드 종류와 무관하게 같은 위치와 리듬을 갖게 한다.

2. 상위 자재 / 하위 자재 카드 포맷 통일
   - 상위 자재와 하위 자재 모두 같은 정보 순서를 따른다.
   - 권장 구조:
     - 품목명
     - 품목코드 / 공정 태그 / 투입 자재 태그
     - 수량 조정
     - 가능 재고 / 실행 후
   - 하위 자재라고 해서 버튼 크기나 여백이 달라지지 않게 한다.

3. 하위 구성 표시 방식 재설계
   - 현재 하위 구성 목록은 너무 좁고 깊게 보여서 실제 구성 확인이 어렵다.
   - 후보 방향:
     - 기본은 접힘 상태로 두고 `하위 N개 보기` 액션 제공
     - 펼쳤을 때는 카드 내부 미니 리스트보다 넓은 상세 영역, 바텀시트, 또는 별도 확장 패널 검토
     - 품목명은 2줄까지 허용하고, 품목코드와 수량은 보조 줄로 분리

4. 긴 품목명 표시 규칙 정하기
   - `ADX4000W TP-LINK (WAN1000M)-동글`처럼 긴 이름이 잘리는 문제를 해결한다.
   - 후보 방향:
     - 주요 품목명은 `line-clamp-2`
     - 하위 자재명도 2줄 허용
     - 전체명 확인이 필요한 경우 탭/롱프레스/상세 버튼으로 전체명을 보여준다.
   - 품목명 오른쪽에 코드와 뱃지가 몰리지 않게 아래줄로 분리한다.

5. Step 4 화면 밀도 재조정
   - 상위 품목 카드는 요약형으로 줄이고, 하위 구성은 별도 확장 영역으로 빼는 방향을 검토한다.
   - sticky 제출 버튼과 본문이 겹치거나 마지막 카드가 가려지지 않게 하단 여백을 재점검한다.
