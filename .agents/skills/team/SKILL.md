# Team

`/team [작업 설명]`으로 호출한다. 작업 설명 없이 호출하면 대화 맥락에서 파악한다.

## 판단

아래 중 하나라도 해당하면 팀 구성. 아니면 한 줄 알림 후 솔로 진행.

- 독립적으로 병렬 처리 가능한 작업이 2개 이상
- 서로 다른 전문 영역이 동시에 필요
- 혼자 처리하기엔 분량이 너무 많음

## 팀 구성

작업에 맞게 역할과 인원을 동적으로 결정한다. 최소 인원으로 구성.

모델 배정:
- **Haiku** — 단순 반복, 검색, 정리
- **Sonnet** — 일반 코딩, 분석
- **Opus** — 복잡한 설계, 아키텍처, 최종 통합

## 솔로 결정 시

> "이 작업은 혼자가 효율적입니다 → 진행합니다."
