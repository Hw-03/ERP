# .env.example

> [!summary] 역할
> 원본 프로젝트의 `.env.example` 파일을 Obsidian에서 추적하기 위한 미러 노트다.

## 원본 위치

- Source: `backend/.env.example`
- Layer: `backend`
- Kind: `source-file`
- Size: `1280` bytes

## 연결

- Parent hub: [[backend/backend|backend]]
- Related: [[backend/backend]]

## 읽는 포인트

- 실제 수정은 원본 파일에서 한다.
- Vault 노트는 구조 파악과 인수인계를 돕는 설명 레이어다.

## 원본 발췌

````text
# ── 로컬 개발 (SQLite, 별도 DB 설치 불필요) ─────────────────────
# DATABASE_URL 을 설정하지 않으면 자동으로 SQLite 사용
# DATABASE_URL=sqlite:///./erp.db

# ── 운영 환경 (PostgreSQL) ────────────────────────────────────────
# DATABASE_URL=postgresql://erp_user:erp_pass@localhost:5432/erp_db

SECRET_KEY=change-me-in-production

# ── 포트 / 호스트 (start.bat, docker-compose 와 정렬) ──────────────
# BACKEND_PORT=8010
# FRONTEND_PORT=3000

# ── 로깅 (Phase 4) ────────────────────────────────────────────────
# 콘솔 + backend/logs/erp.log (RotatingFileHandler 5MB x 5 backup).
# LOG_LEVEL=INFO
# LOG_DIR=backend/logs

# ── CORS 추가 origin (콤마 구분) ───────────────────────────────────
# 기본 정규식이 RFC1918 사설 IP + localhost 를 허용한다.
# 외부 PC/도메인 추가 시 콤마로:
# CORS_EXTRA_ORIGINS=https://erp.example.com,http://192.168.10.5:3000
````

## 정책

- `main` 브랜치는 코드만 유지한다.
- `vault-sync` 브랜치는 같은 코드에 `vault/` 인수인계 문서를 더한다.
- 코드와 노트가 다르면 실제 코드가 우선이다.
