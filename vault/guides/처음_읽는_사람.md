# 처음 읽는 사람

이 볼트는 DEXCOWIN MES 프로젝트를 처음 넘겨받는 사람이 길을 잃지 않도록 만든 설명형 프로젝트 지도입니다.

## 가장 먼저 알 것

- `vault/ERP/`는 실제 프로젝트 폴더 구조를 따라갑니다.
- 각 폴더의 `📁_폴더명.md`는 그 폴더의 안내판입니다.
- 실제 파일을 설명하는 문서는 같은 위치의 `원본파일명.md`입니다.
- `vault/guides/`는 읽는 순서를 도와주는 문서 5개만 둡니다.
- 설명이 실제 코드와 다르면 실제 코드가 진실입니다.

## 추천 순서

1. [[ERP/📁_ERP]] — 프로젝트 전체 입구
2. [[guides/전체_컨텍스트]] — 회사 업무 흐름과 시스템 구조
3. [[guides/위험지대_지도]] — 실수하면 큰 영향을 주는 곳
4. [[guides/용어사전]] — 현장 용어와 코드 용어 연결
5. 관심 영역의 폴더 안내판 — 예: [[ERP/backend/📁_backend]], [[ERP/frontend/📁_frontend]], [[ERP/scripts/📁_scripts]]

## 읽는 요령

개발자가 아니라면 코드 전체를 읽으려고 하지 않아도 됩니다. 먼저 폴더 안내판에서 “이 폴더가 왜 있는지”를 보고, 거기서 추천하는 5개 파일만 따라가면 됩니다.

개발자나 AI 에이전트라면 각 파일 설명의 “연결되는 파일”을 따라가면 화면에서 API, 서비스, DB까지 흐름을 추적할 수 있습니다.
