# 화면 정의서 — DEXCOWIN MES

이 문서는 DEXCOWIN MES 화면을 **현장 업무 관점**에서 한눈에 보도록 정리한 지도입니다.
화면이 무슨 일을 하고, 누가 볼 수 있고, 어떤 서버 API와 연결되는지를 담았습니다.
숫자(품목 수 등)는 수시로 바뀌므로 적지 않았습니다 — 실제 값은 `python _attic/backend-scripts/facts.py`로 확인하세요.

---

## 진입 구조 — 화면이 어떻게 뜨는가

- 브라우저로 루트 주소(`/`)에 들어오면 Next.js가 [[ERP/frontend/app/legacy/page.tsx]]를 렌더합니다.
- 로그인 게이트(`MesLoginGate`)를 먼저 통과해야 합니다. 직원이 본인을 고르고 **PIN(기본 0000)**을 넣어 로그인합니다.
- 로그인 후 화면 폭에 따라 **두 가지 껍데기(Shell)** 중 하나가 보입니다.
  - **데스크톱**(`lg` 이상, 넓은 화면): [[ERP/frontend/app/legacy/_components/DesktopLegacyShell.tsx]] — 왼쪽 세로 사이드바 + 상단바 + 본문.
  - **모바일**(좁은 화면): [[ERP/frontend/app/legacy/_components/mobile/MobileShell.tsx]] — 하단 탭바 + 상단 헤더 + 본문.
- 두 껍데기는 같은 데이터를 보지만 **레이아웃만 다릅니다.** 탭(화면) 종류는 거의 같습니다.

> 한 줄 요약: **PC에서는 사이드바, 휴대폰에서는 하단 탭.** 안에 들어가는 화면은 같습니다.

### 탭 = URL `?tab=`
어느 화면에 있는지는 주소창의 `?tab=` 값으로 정해집니다. 알림을 클릭하면 `?tab=...&section=...` 형태로 특정 화면·구역으로 바로 이동합니다(딥링크).

## 데스크톱 사이드바 탭 목록

사이드바는 [[ERP/frontend/app/legacy/_components/DesktopSidebar.tsx]]에서 정의합니다. 위쪽 6개는 일반 탭, 맨 아래 "관리"는 따로 떨어진 관리자 진입구입니다.

| 순서 | 탭 이름 | 한 줄 설명(코드의 subtitle) | URL `?tab=` 값 |
|---|---|---|---|
| 1 | 대시보드 | 현황과 안전재고 확인 | `dashboard` |
| 2 | 입출고 | 입고와 출고 작업 처리 | `warehouse` |
| 3 | 창고 지도 | 위치별 재고 한눈에 | `warehouseMap` |
| 4 | 불량 | 격리·폐기·반품 처리 | `defect` |
| 5 | 입출고 내역 | 입출고 이력 조회 | `history` |
| 6 | 주간보고 | 생산·재고 흐름 | `weekly` |
| (하단) | 관리 | 마스터와 운영 설정 | `admin` |

> 모바일에는 **창고 지도 탭이 없습니다.** 모바일 탭은 대시보드 / 입출고 / 불량 / 내역 / 주간보고 / 관리 6개입니다([[ERP/frontend/app/legacy/_components/mobile/MobileShell.tsx]]).

### 상단바(공통 도구)
[[ERP/frontend/app/legacy/_components/DesktopTopbar.tsx]]는 모든 탭 위에 항상 떠 있습니다.
- 화면 제목 + 아이콘, 상태 메시지(작업 성공/실패 알림 띠)
- 오른쪽: **로그인한 사람 이름·부서·역할 배지**, 그 안 드롭다운에서 **PIN 변경 / 로그아웃**
- **알림 벨**(`NotificationBell`) — 결재·인수인계 도착 알림. 클릭하면 해당 화면으로 딥링크 이동.
- **동기화(새로고침) 버튼**

## 화면별 정의

접근 직급 규칙(코드 근거: [[ERP/frontend/app/legacy/_components/_warehouse_steps/_constants.ts]]):
- **입출고·불량 화면 진입**: 직원의 `io_enabled`가 꺼져 있지 않으면 가능(`canEnterIO`). 꺼져 있으면 "접근 거부" 안내.
- **부서 승인함**: 부서 결재자(`department_role`이 정/부) 또는 관리자(`level === "admin"`)만(`isDepartmentApprover`).
- **창고 승인함**: 창고 담당(`warehouse_role`이 정/부)만.
- **관리 화면**: 관리자 PIN을 따로 풀어야 함(화면 자체에 PIN 잠금).

| 화면명 | 위치(탭) | 접근 직급 | 주요 기능 | 연결 API(대표) | 주의사항 |
|---|---|---|---|---|---|
| **대시보드** | `dashboard` | 로그인 누구나 | 재고 현황 표·KPI(전체/부족/품절)·부서/모델/공정 필터·생산가능 수량 패널·품목 상세(우측 패널, 최근 거래 5건)·"입출고로 보내기" | `GET /api/items`, `GET /api/inventory/transactions`, `GET /api/production/capacity` | 재고 표는 100건씩 페이지. 품목 코드는 `mes_code` 기준. 안전재고 미달 건수가 사이드바 배지로 뜸. |
| **입출고** | `warehouse` | `io_enabled` 직원 | 입고/출고/부서 조정/불량 등록 작성, **작업 중**(임시저장 누적), **내 요청**, (권한별)**창고 승인함**·**부서 승인함**·**인수인계** | `POST /api/io/preview`, `PUT /api/io/draft`, `POST /api/io/submit`, `GET /api/stock-requests/.../count`, `GET /api/handover/inbox/count` | 진입 권한 없으면 "접근 거부". 섹션 탭은 권한 따라 늘었다 줄었다 함. **인수인계 작성=튜브 부서, 인수 확인=고압·진공 부서 또는 부서 결재자.** |
| **창고 지도** | `warehouseMap` (데스크톱 전용) | 로그인 누구나 | 창고를 **바닥→앵글 정면→줄→층** 순으로 드릴다운, 자리별 박스·품목·수량 확인, 품목명/코드 검색(`/`키 포커스), 브라우저 뒤로가기 지원 | `GET /api/warehouse-map/map` | 모바일에는 탭이 없음. 위치 데이터는 관리 탭 "창고 구조 편집/위치 배정"에서 만듦. |
| **불량** | `defect` | `io_enabled` 직원 | 격리 항목 목록(부서별, 다중 선택), KPI 카드(격리/1년경과/결재대기/오늘처리), 새 불량 격리·R 즉시 폐기·R 즉시 반품, 일괄 정상복귀/폐기/반품, PA·PF 분해 마법사 | `GET /api/defects/locations`, `GET /api/defects/kpi`, `POST /api/defects/quarantine`, `POST /api/defects/unquarantine` | 작업자 로그인 필수. 생산라인 소속이면 기본 "내 부서"만 보임. BOM 있는 품목은 "분해" 흐름으로 빠짐. |
| **입출고 내역** | `history` | 로그인 누구나 | 거래 이력 표(무한 스크롤), 기간/부서/모델/거래종류 필터, 검색, 달력 패널(월/연 보기), 거래 상세·정정·메타 수정, 묶음(배치) 펼치기 | `GET /api/inventory/transactions`, `GET /api/inventory/transactions/summary`, `GET /api/inventory/.../monthly-counts` | 상단 KPI 박스는 표시 전용(클릭 필터 아님). 정정하면 새 보정 거래가 위에 쌓임(원본은 남음). |
| **주간보고** | `weekly` | 로그인 누구나 | 한 주(월~일) 생산·재고 흐름 요약 보고서. 상단 주(week) 선택기로 주간 이동 | `GET /api/inventory/weekly-report?week_start=...&week_end=...` | **화면 동결(완성) 상태** — 명시 요청 없이 손대지 말 것([[ERP/frontend/app/legacy/_components/DesktopWeeklyReportView.tsx]]). |
| **관리(관리자)** | `admin` | 관리자 PIN 해제 필요 | 좌측 메뉴 9개 섹션: 모델/품목/직원/부서 관리, BOM 관리, 내보내기(엑셀·CSV), 외부 제출용 로그, 창고 구조 편집, 위치 배정 + 보안(PIN 변경) | 섹션별 상이(`/api/items`, `/api/employees`, `/api/departments`, `/api/bom`, export URL 등) | 진입 시 **관리자 PIN 잠금** 통과해야 함. 다른 탭으로 나갈 때 입력 중이면 이탈 확인(dirty guard). |

## 입출고 화면 섹션 탭 (권한별 노출)

입출고 탭 안에는 다시 작은 섹션 탭이 있습니다([[ERP/frontend/app/legacy/_components/_warehouse_sections/WarehouseSectionTabs.tsx]]). 권한에 따라 보이는 탭이 달라집니다.

| 섹션 | 항상 보임? | 누구에게 보이나 |
|---|---|---|
| 요청 작성 | 예 | 입출고 진입 가능자 전원 |
| 작업 중 | 예 | 전원 (임시저장 누적 개수 배지) |
| 내 요청 | 예 | 전원 |
| 창고 승인함 | 아니오 | 창고 담당(정/부)만 |
| 부서 승인함 | 아니오 | 부서 결재자 또는 관리자만 |
| 인수인계 | 아니오 | 튜브 부서(작성) 또는 고압·진공 부서/부서 결재자(인수 확인) |

## 모바일 화면 개요

모바일은 [[ERP/frontend/app/legacy/_components/mobile/MobileShell.tsx]]에서 하단 탭바로 전환합니다. 화면 본체는 `mobile/screens/` 아래에 있습니다.

- 대시보드 — `MobileDashboardScreen`
- 입출고 — `MobileWarehouseScreen` (마법사 형태 단계 입력)
- 불량 — `MobileDefectScreen`
- 내역 — `MobileHistoryScreen`
- 주간보고 — `MobileWeeklyScreen`
- 관리 — `MobileAdminScreen`

모바일 특징:
- **입출고·불량 탭은 `io_enabled` 직원에게만** 하단 탭바에 노출됩니다(권한 없으면 탭 자체가 안 보임).
- 알림은 데스크톱과 동일한 알림 벨을 헤더에 둡니다.
- 작업 결과는 화면 아래 **토스트 메시지**로 뜹니다(자동으로 사라지지 않고 닫기 버튼으로 닫음 — 느린 네트워크·접근성 배려).
