# 2026년 6월 변경 요약 (복귀용)

휴직 기간 동안 DEXCOWIN MES에 들어온 변경을 한곳에 모았습니다. 기간은 **2026-05-22 ~ 2026-06-24**입니다. 현장에서 바로 보이는 변화부터, 화면 뒤에서 바뀐 규칙, 모바일 개편, 코드 정리 순으로 정리했습니다.

> 양이 많습니다. 시간순이 아니라 **주제별**로 묶었습니다. 정확한 커밋이 궁금하면 `git log --since=2026-05-22 --until=2026-06-25` 로 직접 확인하면 됩니다.

---

## 한눈에 — 가장 큰 변화 6가지

1. **생산 가능 수량을 AF 기준으로 재정의** — 출하 대기·빠른 생산·총생산 3지표로 바뀜.
2. **창고 박스 자동 차감** — 박스에서 출고하면 그 박스 수량이 자동으로 줄어듦(활성화 플래그 OFF가 기본).
3. **창고 지도 편집을 2탭으로 통합** — 박스 관리·앵글 편집.
4. **모바일 화면 전면 완성** — 하단 탭바·입출고 위저드·불량 허브·창고지도까지 모바일 전용으로 재설계.
5. **인수인계서 + 결재 알림 + 인수 권한 제한** — 튜브→고압/진공 재고 자동 이동, 받는 부서만 인수 확인.
6. **결재 권한을 role로 일원화 + 작업자 사번 전 거래 기록** — 직원 등급(level) 제거, 감사 추적 강화.

## 신규 기능

현장에서 직접 보이거나 쓰게 되는 기능입니다.

### 생산·재고
- **생산 가능 수량 AF 재정의 (6/15, 6/23 보강)** — 기존 즉시/최대 2지표를 **출하 대기(ship_ready)·빠른 생산(fast_production)·총생산(total_production)** 3지표로 재정의했습니다. AF(조립·완성) 기준으로 계산하며, 모델별 "기준 PF"를 지정할 수 있습니다. 구 지표는 전환이 끝날 때까지 병행 유지됩니다. 계산 기준은 [[ERP/backend/app/services/production_capacity.py]].
- **거래 취소 (6/15)** — 입출고 내역을 지우지 않고 **재고만 롤백**하는 기능입니다. 창고/부서 수량을 분리해 표시하고, 일반 역재생 방식으로 재고 효과를 되돌립니다.
- **안전재고 규칙 정립 (6/22)** — 안전재고 = `items.min_stock`(부족 알림이 이 값으로 작동). 규칙은 발주 대상인 R 품목만 "1대당 BOM 사용량 × 200"으로 설정하고 A/F는 비웁니다. 개발 DB에는 적용 완료, 운영 DB에는 아직 전파 전입니다.

### 창고 지도
- **창고 박스 자동 차감 (6/22, Phase 1~3c)** — 박스 추적을 켜면 출고 시 해당 박스의 수량이 자동으로 줄어듭니다. 박스 드래그 이동·미배치 알림·중간 삽입까지 들어갔습니다. **활성화 플래그(box-tracking)가 현재 OFF라 평소엔 기존 방식**으로 동작합니다. 차감 로직은 [[ERP/backend/app/services/inv_transfer.py]]의 `_deplete_boxes_if_tracking()`.
- **창고 지도** — 앵글/박스를 실제 데이터로 그려 어디에 무엇이 얼마나 있는지 지도로 봅니다. 재고대조 기준은 창고 재고(`warehouse_qty`)입니다. 모델은 [[ERP/backend/app/models/warehouse.py]].
- **편집 탭 2개로 통합 (6/23)** — 이전 3탭(박스 배치/재고 관리/앵글 편집)을 **박스 관리·앵글 편집 2탭**으로 합쳤습니다. ※ 용어는 "앵글"입니다("선반" 아님).

### 인수인계·알림
- **인수인계서** — 튜브 부서에서 고압/진공 부서로 넘기는 인수인계 문서입니다. 작성·제출하면 받는 쪽이 PIN으로 인수 확인을 하고, 확인하는 순간 품목 수량만큼 튜브에서 받는 부서로 재고가 자동 이동합니다(거래 로그 동반). 인쇄도 됩니다. 모델은 [[ERP/backend/app/models/handover.py]]. **임시저장(draft) 단계가 추가됐습니다(6/15).**
- **결재 알림 시스템** — 요청 도착·승인·반려·인수인계 도착을 직원별로 쌓아 보여줍니다(30초 폴링).

### 입출고·접근
- **입출고 접근 권한 io_enabled (6/23)** — 입출고 화면 접근을 로그인 직원의 `io_enabled` 권한과 연동했습니다.
- **승인 전 요청 수정 + 내 요청 더보기 (6/24)** — 아직 승인되지 않은 요청을 수정할 수 있고, 내 요청 목록을 더 볼 수 있습니다.
- **낱개 출고 주간보고 집계 (6/24)** — 낱개 출고가 주간보고에 반영되지 않던 버그를 고쳤습니다.
- **출고출하 disassemble 창고에서 가져오기 (6/24)** — 분해 출고 시 창고에서 가져오는 흐름을 지원합니다.
- **로그인 직원 초성검색 (6/23)** — 이름 초성으로 직원을 찾습니다.
- **재작업 분해 시 자식 격리 이동 (~6/5)** — 회수 대상이 아닌 자식 부품이 사라지던 동작을 막아 재고 소실을 방지합니다.
- **입출고 임시저장 누적 (~6/5)** — 임시저장이 기존 슬롯을 덮어쓰지 않고 batch 단위로 쌓입니다.

## 기준체계·규칙 변경

눈에 잘 안 보이지만 영향이 큰 규칙 변경입니다.

- **결재 권한을 role로 일원화 (6/15)** — 직원 등급(level)을 없애고, 결재 권한을 **창고/부서 role 기준**으로 통일했습니다.
- **인수 확인 권한 = 받는 부서만 (6/16)** — 고압/진공 소속 직원만 인수 대기함을 보고 인수 확인을 할 수 있습니다. 생산부 결재권자라도 소속이 고압/진공이 아니면 인수 불가(의도된 설계). 일반 부서 결재는 부서 무관 그대로입니다.
- **작업자 사번 전 거래 기록 (6/15)** — 모든 거래 경로에 작업자 사번(`producer_employee_id`)을 남기고, 외부 감사 CSV에 처리자 사번 컬럼을 추가했습니다.
- **mes_code 전역 unique 복원 (~6/5)** — 품목코드가 시스템 전체에서 유일해졌습니다. 소프트삭제된 코드도 코드 값을 영구 점유하므로 **삭제한 코드를 다른 품목이 재사용할 수 없습니다**(이력 추적성). 일련번호는 삭제분 포함 최댓값+1.
- **결재 규칙 단일 원천 (~6/5, ADR-0005)** — 입출고 배치가 창고/부서 결재·즉시 반영 중 무엇을 타는지 정하는 규칙을 [[ERP/backend/app/services/approval_rules.py]] 한 곳으로 모았습니다. FE↔BE 드리프트 테스트가 함께 걸려 있습니다.
- **주간보고 집계 정비 (6/16)** — 생산=PRODUCE 전용으로 분리하고 입고를 떼어냈으며, 전주 재고를 정확화했습니다. (주간보고 화면은 동결 영역)
- **재고 리셋 고아 방지 가드 (6/15)** — 재고 리셋 스크립트가 미결 요청을 고아로 만들던 문제에 가드를 넣었습니다.

## 모바일 전면 개편 (6/10 ~ 6/23)

PC와 별개로 **모바일 전용 화면을 대대적으로 재설계**했습니다. 데스크톱 레이아웃은 건드리지 않고 모바일 분기만 손봤습니다.

- **하단 탭바** — 슬라이딩 pill 애니메이션, "더보기"를 5번째 전폭 탭으로 전환. ※ 탭바 디자인은 **동결 영역**입니다(임의 수정 금지).
- **입출고 위저드** — 스텝별 버튼/텍스트 확대, sticky 푸터, 스크롤 제거 등 터치 환경에 맞게 재구성.
- **불량 허브** — 첫 화면 카드 키오스크화, KPI·격리 목록 노출.
- **내역 달력** — 모바일은 월~금 5열로 표시.
- **창고지도** — 모바일 가로 전용 화면으로 전면 재설계.
- **대시보드·생산 가능 현황** — 검색바 sticky 분리, 기본 접힘 등.
- **모바일↔데스크톱 정합** — 두 화면이 같은 데이터를 보이도록 맞췄습니다.

## 구조 정비 (동작 무변경)

기능은 그대로 두고 코드만 정리한 부분입니다. 복귀 후 코드를 읽을 때 참고하세요.

- **`legacy/` → `mes/` 폴더 개명 (6/5)** — 현재 운영 화면 폴더를 `frontend/app/legacy` → `frontend/app/mes`로 바꿨습니다("낡은 코드"라는 오해 방지). 내부 식별자 `xray-erp` 등은 의도적으로 유지됩니다.
- **직원별 품목 순서 커스터마이징 (6/8)** — `employee_item_orders` 테이블을 신설해, 직원이 품목 표시 순서를 직접 정렬·저장할 수 있습니다.
- **DB 접근 계층(DAL)·의존성·부트스트랩 분리** — DB 조회/저장을 [[ERP/backend/app/repositories/📁_repositories]]로 모으고, 권한·인증을 `dependencies/`로, DB 초기화를 `bootstrap/`로 분리했습니다.
- **`schemas.py` → `schemas/` 패키지 분리 (6/5)** — API 스키마를 도메인별 파일로 나눴습니다(`from app.schemas import X`는 그대로 동작). → [[ERP/backend/app/schemas/📁_schemas]]
- **서비스 추출** — 생산 입고 오케스트레이션을 `services/production_receipt.py`로, 품목 정렬을 `sortItemsForPicker` 순수 함수로 빼내는 등 큰 함수를 책임별로 쪼갰습니다(동작 보존).
- **테스트 보강** — 안정성 핵심 서비스 회귀 테스트를 늘리고, e2e에 전용 DB·로그인 우회를 붙여 쓰기 시나리오까지 확장했습니다. UTC 기준 시간대 의존 flake도 수정했습니다.
- **CLAUDE.md 영어화 + 드리프트 방지 규칙 (6/19)** — 프로젝트 지침을 영어화하고, 문서·코드가 조용히 어긋나는 것을 막는 규칙을 추가했습니다.

위 항목들은 커밋 메시지에 "동작 보존"으로 표기돼 있으니, 의심되면 해당 커밋을 직접 확인하면 됩니다.
