# 2026년 6월 변경 요약 (복귀용)

휴직 기간 동안 DEXCOWIN MES에 들어온 변경을 한곳에 모았습니다. 기간은 **2026-05-22 ~ 2026-06-05**입니다. 현장에서 바로 보이는 변화부터, 화면 뒤에서 바뀐 규칙, 코드 정리 순으로 정리했습니다.

---

## 신규 기능

현장에서 직접 보이거나 쓰게 되는 기능입니다.

- **창고 지도** — 앵글/박스를 실제 데이터로 그려서 어디에 무엇이 얼마나 있는지 지도로 봅니다. 화면에서 본 위치/수량을 실제 재고와 대조(재고대조)할 수 있습니다. 박스 hover, 줄 구분, 한 박스에 여러 품목코드 표시, 브라우저 뒤로가기 같은 사용성도 함께 들어갔습니다.
- **인수인계서** — 튜브 부서에서 고압/진공 부서로 넘기는 인수인계 문서입니다(F1). 작성(=제출)하면 받는 쪽이 PIN으로 인수 확인을 하고, 확인하는 순간 품목 수량만큼 튜브에서 받는 부서로 재고가 자동 이동합니다. 인쇄도 됩니다. 받는 쪽에는 도착 알림이 뜹니다.
- **결재 알림 시스템** — 요청이 도착하거나(F2), 승인/반려되면(F3) 알림이 갑니다. 그동안 "내 요청이 어떻게 됐는지" 직접 들어가 봐야 했던 부분이 줄었습니다.
- **재작업 분해 시 자식 격리 이동** — 불량 재작업으로 분해할 때, 회수 대상이 아닌 자식 부품이 그냥 사라지던 동작을 막았습니다. 이제 회수 외 자식은 격리로 이동해서 재고에서 소실되지 않습니다.
- **입출고 임시저장 누적** — 임시저장이 기존 슬롯을 덮어쓰던 동작을 없앴습니다. batch 단위로 새 슬롯이 쌓이므로, 저장해둔 작업이 다음 저장에 지워지지 않습니다.

## 기준체계 변경

데이터를 다루는 규칙이 바뀐 부분입니다. 눈에 잘 안 보이지만 영향이 큽니다.

- **mes_code 전역 unique 복원** — 품목코드(mes_code)가 시스템 전체에서 다시 유일해졌습니다. 소프트삭제(휴지통)된 코드도 코드 값을 영구히 점유하므로, **삭제했던 코드를 다른 품목이 재사용할 수 없습니다**. 이력 추적성을 위한 결정입니다. 일련번호는 삭제분을 포함한 최댓값+1로 매기므로 평소 신규 등록에는 영향이 없습니다. (중간에 "삭제 후 재등록 허용"을 위한 부분 unique 단계를 거쳤다가, 최종적으로 전역 unique로 정리됐습니다.)
- **결재 규칙 단일 원천** — 입출고 배치가 창고 결재/부서 결재/즉시 반영 중 무엇을 타는지 정하는 규칙이 여러 파일에 복제돼 있던 것을, `approval_rules.py` 한 곳으로 모았습니다. 한 곳만 고치면 프론트와 백엔드가 조용히 어긋나던 위험을 줄였습니다. 배경은 [[ERP/_attic/docs/adr/ADR-0005-approval-rules-single-source.md]]에 있습니다.
- **입출고 내역 용어 통일** — 화면 라벨을 "새 불량 / 불량 해제 / 불량 처리"로 맞췄습니다(이전의 "새 격리 / 격리 해제 / 폐기" 표현 정리). 용어 기준은 [[guides/용어사전]]을 따릅니다.

## 구조 정비 (동작 무변경)

기능은 그대로 두고, 코드만 정리한 부분입니다. 복귀 후 코드를 읽을 때 참고하세요.

- 큰 함수를 책임별 헬퍼로 쪼개는 리팩터가 여러 건 있었습니다(production, io_dispatch, io_preview, sr_execution 등). 줄 수는 줄었지만 동작은 그대로입니다.
- 불량 위치 목록 조회의 N+1 쿼리를 제거했습니다(사유 로그 일괄 조회).
- 에러 응답 방식을 표준으로 통일했습니다(여러 라우터의 직접 예외 발생 → 공통 처리, 프론트 메시지 추출과 호환).
- 안정성 핵심 서비스 회귀 테스트를 110개대로 늘리고, e2e는 전용 DB와 로그인 우회를 붙여 쓰기 시나리오까지 확장했습니다.

위 항목들은 커밋 메시지에 "동작 보존"으로 표기돼 있으니, 의심되면 해당 커밋을 직접 확인하면 됩니다.

## 코드 정비 (Phase 1 — main 병합 완료)

휴직 직전 요청한 "클린코드" 방향으로, **동작은 그대로 두고 구조만** 정비했습니다. PR #27로 올라가 2026-06-05 main에 병합됐고, **이 볼트 미러에도 반영돼 있습니다.**

- **`schemas.py` → `schemas/` 패키지 분리** — API 데이터 약속(스키마)을 도메인별 파일로 나눴습니다. 기존 `from app.schemas import X` 호출은 그대로 동작합니다. → [[ERP/backend/app/schemas/📁_schemas]]
- **`transactions.py` 필터 헬퍼 분리** — 거래 이력 쿼리 필터/응답 빌더를 `_tx_filters.py`로 빼냈습니다(엔드포인트 동작 무변경). → [[ERP/backend/app/routers/inventory/_tx_filters.py]]
- **`legacy/_warehouse_v2` 오해 방지 README** — "낡은 코드가 아니라 현재 운영 중인 메인 UI"임을 적은 안내.

검증: 백엔드 테스트 전체 통과, OpenAPI 스펙 무변경(외부 API 동일). 같은 날 인수인계 인쇄(로고·A4 레이아웃)와 피드백 현황 문서도 함께 main에 병합됐습니다.
