# 위험지대 지도

이 문서는 “잘못 건드리면 실제 운영에 큰 영향을 줄 수 있는 곳”만 따로 모은 지도입니다.

## 가장 위험한 영역

- [[ERP/backend/app/models/📁_models]] — DB 테이블 구조(영역별 파일로 나뉜 패키지)
- [[ERP/backend/app/schemas.py]] — API 요청/응답 계약
- [[ERP/backend/app/services/inventory.py]] — 실제 재고 수량 변경
- [[ERP/backend/app/services/stock_requests.py]] — 입출고 요청과 승인 상태
- [[ERP/backend/app/services/stock_math.py]] — 사용 가능 재고와 승인 대기 수량 계산
- [[ERP/backend/app/services/approval_rules.py]] — 결재 규칙 상수의 단일 원천. 한 줄만 바꿔도 창고 결재/부서 결재/즉시 반영 판정이 통째로 흔들립니다(ADR-0005). FE↔BE drift 테스트가 함께 걸려 있습니다.
- [[ERP/backend/app/routers/stock_requests.py]] — 입출고 요청 API
- [[ERP/backend/app/routers/settings.py]] — 관리자 설정과 위험 기능
- [[ERP/frontend/lib/api-core.ts]] — 프론트 전체 API 호출 공통 처리
- [[ERP/scripts/ops/backup_db.py]] / [[ERP/scripts/ops/restore_db.py]] — DB 백업/복구

## 새로 추가된 위험 영역 (2026-06)

- **창고 지도 재고대조** — [[ERP/backend/app/models/warehouse.py]]. 박스에 담긴 품목 수량을 창고 재고(`warehouse_qty`) 기준으로 대조합니다. 대조 기준을 `quantity`(부서 보유분 포함)로 착각하면 "창고에 있어야 할 양"이 어긋납니다.
- **결재 알림 / 결재 규칙** — [[ERP/backend/app/models/notification.py]] · [[ERP/backend/app/services/approval_rules.py]]. 결재 판정 규칙(`approval_rules`)을 바꾸면 알림이 누구에게 가는지, 어떤 작업이 결재를 타는지가 함께 바뀝니다. 규칙·알림을 따로 보지 말고 같이 확인합니다.
- **인수인계 자동 이동** — [[ERP/backend/app/models/handover.py]]. '인수 확인' 한 번으로 품목 수량만큼 튜브→고압/진공 재고가 실제로 이동합니다(거래 로그 동반). 인수 확인 경로를 건드릴 때는 재고 이동과 거래 로그까지 함께 검증합니다.

## 수정 전 확인 원칙

1. 현재 DB 백업이 있는지 확인합니다.
2. 연결된 라우터, 서비스, 스키마, 화면을 같이 봅니다.
3. 재고 수량이나 요청 상태가 바뀌는 작업은 거래 로그까지 확인합니다.
4. 프론트 공용 API 처리를 바꾸면 대시보드, 입출고, 내역, 관리자 화면을 함께 확인합니다.
5. `_attic`의 과거 자료를 현재 운영 기준으로 착각하지 않습니다.
