# 위험지대 지도

이 문서는 “잘못 건드리면 실제 운영에 큰 영향을 줄 수 있는 곳”만 따로 모은 지도입니다.

## 가장 위험한 영역

- [[ERP/backend/app/models.py]] — DB 테이블 구조
- [[ERP/backend/app/schemas.py]] — API 요청/응답 계약
- [[ERP/backend/app/services/inventory.py]] — 실제 재고 수량 변경
- [[ERP/backend/app/services/stock_requests.py]] — 입출고 요청과 승인 상태
- [[ERP/backend/app/services/stock_math.py]] — 사용 가능 재고와 승인 대기 수량 계산
- [[ERP/backend/app/routers/stock_requests.py]] — 입출고 요청 API
- [[ERP/backend/app/routers/settings.py]] — 관리자 설정과 위험 기능
- [[ERP/frontend/lib/api-core.ts]] — 프론트 전체 API 호출 공통 처리
- [[ERP/scripts/ops/backup_db.py]] / [[ERP/scripts/ops/restore_db.py]] — DB 백업/복구

## 수정 전 확인 원칙

1. 현재 DB 백업이 있는지 확인합니다.
2. 연결된 라우터, 서비스, 스키마, 화면을 같이 봅니다.
3. 재고 수량이나 요청 상태가 바뀌는 작업은 거래 로그까지 확인합니다.
4. 프론트 공용 API 처리를 바꾸면 대시보드, 입출고, 내역, 관리자 화면을 함께 확인합니다.
5. `_attic`의 과거 자료를 현재 운영 기준으로 착각하지 않습니다.
