# 📁 .git

## 이 폴더는 무엇을 위한 곳인가

`.git`은 Git이 브랜치, 커밋 이력, 원격 저장소 정보를 보관하는 내부 저장소입니다.

## 현장 업무와의 관계

제품 기능과 직접 연결되지는 않지만, `main`은 코드만 유지하고 `vault-sync`는 코드와 Obsidian 설명을 함께 가진다는 브랜치 정책을 지키는 기반입니다.

## 언제 보면 좋나

- 브랜치와 커밋 이력이 왜 중요한지 이해해야 할 때
- `main`과 `vault-sync`의 차이를 설명해야 할 때
- Git 저장소가 어떻게 변경 이력을 관리하는지 확인해야 할 때

## 파일 설명 방식

실제 `.git/` 폴더는 Obsidian 미러 안에 복제하지 않습니다. Git 내부 폴더를 그대로 만들면 저장소가 꼬일 수 있으므로, 이 프록시 노트 하나만 둡니다.

## 조심할 점

`.git` 내부 파일을 직접 수정하거나 삭제하면 저장소가 망가질 수 있습니다. 브랜치 변경, 커밋, 병합은 Git 명령으로만 처리합니다.
