# .env.example — .env.example 설명

## 이 파일은 무엇을 책임지나

`.env.example`는 프로젝트 파일입니다. 프로젝트 구조 안에서 `backend/.env.example` 위치에 있으며, 필요할 때 역할과 연결 파일을 확인하기 위한 설명을 둡니다.

## 업무 흐름에서의 의미

현장 화면에서 발생한 요청이 실제 데이터 조회나 변경으로 이어질 때 이 백엔드 영역이 관여합니다.

## 언제 보면 좋나

- 이 파일이 맡은 화면/API/데이터 흐름을 확인해야 할 때
- 수정 전에 영향 범위를 빠르게 파악해야 할 때

## 중요한 내용

자동으로 뽑을 수 있는 함수/클래스 목록은 적지만, 파일 위치와 확장자로 볼 때 위 역할을 맡습니다.

## 연결되는 파일

- [[ERP/backend/📁_backend]] — 이 파일이 속한 폴더의 안내판입니다.

## 핵심 발췌

```text
# ── 로컬 개발 (SQLite, 별도 DB 설치 불필요) ─────────────────────
# DATABASE_URL 을 설정하지 않으면 자동으로 SQLite 사용
# DATABASE_URL=sqlite:///./mes.db

# ── 운영 환경 (PostgreSQL) ────────────────────────────────────────
# DATABASE_URL=postgresql://mes_user:mes_pass@localhost:5432/mes_db
# 운영 표식: 설정 시 DATABASE_URL 이 SQLite/미설정이면 backend 기동 거부(WS2 가드).
# 개발/테스트는 비워둘 것(SQLite 허용). docker-compose 는 APP_ENV=production 주입.
# APP_ENV=production
# REQUIRE_POSTGRES=1   # APP_ENV 대신 명시적으로 강제할 때

SECRET_KEY=change-me-in-production

# ── 포트 / 호스트 (start.bat, docker-compose 와 정렬) ──────────────
# BACKEND_PORT=8010
# FRONTEND_PORT=3000

# ── 로깅 (Phase 4) ────────────────────────────────────────────────
# 콘솔 + backend/logs/mes.log (RotatingFileHandler 5MB x 5 backup).
# LOG_LEVEL=INFO
# LOG_DIR=backend/logs

# ── CORS 추가 origin (콤마 구분) ───────────────────────────────────
# 기본 정규식이 RFC1918 사설 IP + localhost 를 허용한다.
# 외부 PC/도메인 추가 시 콤마로:
# CORS_EXTRA_ORIGINS=https://mes.example.com,http://192.168.10.5:3000
```
