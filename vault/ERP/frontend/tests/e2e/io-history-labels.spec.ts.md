# io-history-labels.spec.ts

## 이 파일은 뭐예요?
입출고 V2 화면과 입출고 내역 탭에서 도메인 라벨이 단일 사전(glossary)을 따르는지 회귀 검증하는 e2e 스펙입니다. work type 3개(원자재 입고·창고 입출고·부서 입출고)가 정확한 이름으로 노출되고, 폐기된 구 라벨 목록이 화면에 없는지 확인합니다.

## 언제 보나요?
- 라벨 이름 변경·화면 리팩터링 후 구 용어가 남지 않았는지 확인할 때 (P0-1 회귀 방어)
- 새 라벨을 추가하거나 폐기 목록에 추가해야 할 때

## 중요한 내용
- `expectLabelAbsent(page, word)` — `getByText(word, {exact:true})`로 정규식 메타문자(예: `|`) 오해석 방지
- 폐기 라벨 목록(11개): 재작업·새 격리·격리 해제·폐기·격리 폐기·창고 반출·창고 반입·외부 입고·부서 작업·생산 | 입고·분해 | 출고
- 시드 데이터 불필요 — 라벨 존재 여부만 검증하므로 가장 먼저 활성화 가능한 spec
