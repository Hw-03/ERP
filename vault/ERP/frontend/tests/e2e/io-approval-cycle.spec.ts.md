# io-approval-cycle.spec.ts

## 이 파일은 뭐예요?
창고 결재의 제출→승인→큐 소멸 풀사이클을 2개의 브라우저 컨텍스트로 검증하는 e2e 스펙입니다. 일반직원(E01)이 창고→부서 요청을 제출해 큐에 올린 뒤, 창고 정(E22)이 창고 승인함에서 PIN 0000으로 승인하면 큐가 비워지는지 확인합니다.

## 언제 보나요?
- 결재 큐 적재·소멸 로직이 변경됐을 때 회귀 확인
- 자가승인(창고 역할이 본인 요청 제출 시 즉시 반영) 정책 때문에 풀사이클 테스트가 실패할 때 원인 파악

## 중요한 내용
- 2-세션 패턴: `ctxA`(E01 제출)와 `ctxB`(E22 승인)를 `browser.newContext()`로 독립 분리
- 자가승인 주의: E22가 창고 정이므로 직접 요청하면 큐 미적재 → E01로 제출해야 큐에 올라감
- 승인 2단계: "승인" 버튼 → PIN 텍스트박스(`0000` 입력) → "승인 확정" 버튼
- 종착 검증: "승인 대기 중인 요청이 없습니다." 텍스트로 큐 소멸 확인
