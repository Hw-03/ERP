# 처음 읽는 사람을 위한 ERP Vault 안내

> [!summary] 역할
> 이 문서는 ERP 프로젝트를 처음 인수인계 받는 사람이 첫날에 읽는 안내서다.
> 코딩을 조금 아는 고등학생도 전체 흐름을 잡을 수 있게, 현재 구조와 읽는 순서를 쉬운 말로 정리한다.

## 이 Vault는 뭐예요?

이 Vault는 `C:\ERP` 프로젝트를 Obsidian에서 보기 좋게 정리한 인수인계용 문서 묶음이다.  
원본 코드를 그대로 읽기 어려운 사람도, 폴더 구조와 핵심 설명을 함께 보면서 전체를 따라갈 수 있게 만든다.

## 가장 중요한 한 문장

> 이 ERP는 공장 자재가 어디에 얼마나 있고, 언제 어떻게 움직였는지를 기록하는 내부 운영 시스템이다.

회계나 급여 시스템이 아니라, **품목 / 재고 / 입출고 / 생산 / 출하** 흐름이 중심이다.

## 지금 프로젝트 구조에서 먼저 알아둘 것

| 위치 | 의미 |
|---|---|
| `backend/` | API 서버와 DB 로직 |
| `frontend/` | 실제 사용자 화면 |
| `docker/` | Docker 실행 설정 |
| `data/` | 원본 엑셀/CSV 자료 |
| `docs/` | 규칙, 진행 기록, 디자인 자료 |
| `scripts/` | 수동 실행 보조 스크립트 |
| `_vault/` | Obsidian 전용 가이드/대시보드/템플릿 |

## 이번 브랜치에서 달라진 핵심

1. `docker-compose*.yml` 이 루트가 아니라 `docker/` 폴더 아래로 정리됐다.
2. `schema.sql` 은 루트가 아니라 `backend/schema.sql` 기준으로 보는 게 맞다.
3. 백엔드에는 `models` 라우터, `integrity`/`stock_math` 서비스가 새로 생겼다.
4. 프론트엔드는 `legacy` 안에 모바일 전용 구조(`mobile/hooks`, `mobile/io`, `mobile/screens`)가 크게 추가됐다.
5. 예전 모바일 탭 컴포넌트 일부는 `_components/_archive/` 로 이동했다.

| 파일 | 왜 중요한가 |

## 헷갈리기 쉬운 것

> [!warning] `legacy` 는 구버전이 아니라 현재 실제 UI다
> 이름 때문에 속기 쉽지만, 지금은 여기가 메인 화면이다.

> [!warning] 모바일 UI는 예전 탭 파일만 보면 안 된다
> 지금은 `mobile/` 아래의 `screens`, `io`, `hooks`, `primitives` 구조를 같이 봐야 한다.

> [!warning] `_vault/` 는 원본 프로젝트 폴더가 아니다
> 여기는 Obsidian용 안내 문서 공간이다. 실제 코드 구조와 동일한 원본은 `backend`, `frontend`, `docker`, `data`, `docs`, `scripts` 를 보면 된다.

## 처음 수정하기 전에 지켜야 할 것

1. 실제 코드와 Vault 설명이 다르면 항상 실제 코드를 우선한다.
2. DB 관련 수정은 `models.py`, 관련 라우터, 관련 서비스를 함께 본다.
3. UI 수정은 `frontend/app/legacy/` 기준으로 본다.
4. 코드 규칙은 `docs/ITEM_CODE_RULES.md` 를 먼저 본다.
5. 모르는 용어는 용어 사전이나 FAQ에 기록한다.
