# 처음 읽는 사람을 위한 ERP Vault 안내


> [!summary] 역할
> ERP 프로젝트를 처음 인수인계 받는 사람이 첫날 읽는 안내서다.

## 이 Vault는 뭐예요?

`C:\ERP` 프로젝트를 Obsidian에서 읽기 좋게 정리한 문서 묶음이다. 실제 코드는 `backend`, `frontend`, `scripts`, `docs`, `data`, `docker`에 있고, `vault/`는 그 구조를 설명하는 레이어다.

## 가장 중요한 한 문장

> 이 ERP는 공장 자재가 어디에 얼마나 있고, 언제 어떻게 움직였는지를 기록하는 내부 운영 시스템이다.

## 먼저 알아둘 구조

| 위치 | 의미 |
|---|---|
| `backend/` | FastAPI API 서버, DB 모델, 라우터, 서비스 |
| `frontend/` | Next.js 사용자 화면. 현재 실제 UI는 `app/legacy` |
| `scripts/ops/` | 백업, 복구, 헬스체크, 정합성 점검 |
| `scripts/migrations/` | 데이터/코드 기준 변경용 일회성 스크립트 |
| `docs/` | 규칙, 진행 기록, 운영 문서 |
| `data/` | 현장 엑셀/CSV 기준 자료 |
| `.github/workflows/` | CI 설정 |

## 헷갈리기 쉬운 것

> [!warning] `legacy`는 현재 실제 UI다
> 이름은 legacy지만 지금 사용자가 보는 주 화면은 `frontend/app/legacy`다.

> [!warning] 예전 단일 파일을 찾지 말 것
> `backend/app/routers/inventory.py`, `AdminTab.tsx` 같은 큰 파일은 최신 구조에서 하위 패키지/섹션으로 분리됐다.

> [!warning] 브랜치 정책
> `main`은 코드만 유지하고, `vault-sync`만 `vault/`를 포함한다.

## 처음 수정하기 전 체크

- 화면 변경: `frontend/app/legacy`와 `frontend/lib/api.ts`를 같이 본다.
- API 변경: `backend/app/routers`, `schemas.py`, 관련 service/test를 같이 본다.
- DB 변경: `backend/app/models.py`, `backend/schema.sql`, migration/seed 영향을 확인한다.
- 운영 작업: `scripts/ops`와 백업 절차를 먼저 확인한다.
