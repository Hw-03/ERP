# 🗂 폴더 지도

> [!summary] 역할
> ERP 레포 루트의 모든 폴더 / 파일이 뭔지 한 줄로 정리. 신입이 ls 결과 보고 "이게 뭐지?" 했을 때 첫 답.

## 한눈에

> [!info] 분류
> - **코드 영역** (배포 핵심): `backend/`, `frontend/`, `scripts/`, `docker/`
> - **문서/설정**: `docs/`, `.github/`, 루트 파일들
> - **격리/보관**: `_attic/`, `_dev/`
> - **vault**: `vault/` (vault-sync 브랜치 한정)
> - **무관 (gitignore)**: `.git/`, `.claude/`, `.pytest_cache/`

## 1. 코드 영역 (배포에 들어감)

### `backend/`
FastAPI API 서버. SQLAlchemy ORM + Pydantic. 진입점은 [[erp/backend/app/main.py]].
- [[erp/backend/app/routers/]] — 15개 라우터, 99개 endpoint
- [[erp/backend/app/services/]] — 14개 비즈니스 로직 모듈
- [[erp/backend/app/models.py]] — 31개 ORM 엔터티
- [[erp/backend/tests/]] — 38개 테스트 파일

### `frontend/`
Next.js 14 App Router. 실제 UI 는 `app/legacy/` 안에 있음 (이름과 달리 활성).
- [[erp/frontend/lib/api.ts]] — 11개 도메인 API re-export 허브
- [[erp/frontend/app/legacy/_components/]] — 184개 .tsx 파일

### `scripts/`
운영/개발 스크립트.
- `scripts/dev/` — 개발 보조 (verify_local.ps1, 매칭 알고리즘 등)
- `scripts/ops/` — 운영 (백업/복구/헬스체크/부하 테스트)

### `docker/`
Docker compose 정의 (실험/배포용). 정규 운영은 `start.bat` 사용.

## 3. 격리 / 보관 (만지지 마)

### `_attic/`
> [!danger] 격리 보관소 — 함부로 손대지 말 것
> 옛 `_archive/`, `_backup/`, `frontend/_archive/` 가 모두 이 안으로 통합됨.
> - `_attic/_archive/` — 폐기된 코드 / 옛 버전 / 참고 자료
> - `_attic/backend/` — backend 백업
> - `_attic/frontend/` — frontend 옛 UI
> - `_attic/ai/` — AI handover 문서
> - `_attic/data/` — 옛 데이터
> - `_attic/docs/` — 옛 문서
> - `_attic/scripts/`, `_attic/outputs/` — 기타
>
> CLAUDE.md 명시: "Do not casually edit `_attic/`; it contains archived source material, backups, and old working notes."

### `_dev/`
> [!info] 개발 baseline 보관
> `_dev/baselines/openapi.json` — OpenAPI 스펙 베이스라인. CI 의 OpenAPI drift 검증에 사용. 라우터/스키마 수정 시 [[erp/scripts/dev/verify_local.ps1|verify_local.ps1]] 가 갱신 명령을 알려준다.

## 4. Vault

### `vault/`
> [!info] vault-sync 브랜치 전용
> Obsidian vault. 신입이 지금 보고 있는 노트들이 다 여기.
> `main` 브랜치에는 없음 (`.gitignore` 의 `vault/` 룰). `vault-sync` 브랜치만 포함.
> 안에 [[_vault/guides/]] (가이드 16종) + [[_vault/dashboards/]] (대시보드/캔버스) + 코드 미러 697개.

## 5. 무관 (gitignore — repo 에 안 들어감)

### `.git/`
Git 메타데이터. 자동 관리.

### `.claude/`
> [!info] Claude Code 로컬 설정
> 사용자(현우)가 쓰는 [Claude Code](https://claude.com/claude-code) CLI 의 로컬 설정 폴더.
> `.gitignore` 영역이라 **다른 사람 PC 에는 없고 origin 에도 안 올라감**.
> 신입이 자기 도구로 Claude Code 를 쓰면 자기 폴더에도 생김. 자기 설정.
> - `settings.local.json` — 권한/모델/MCP 설정
> - `scheduled_tasks.lock` — 스케줄 잠금

### `.pytest_cache/`
> [!info] pytest 캐시
> backend 테스트 실행 시 pytest 가 자동 생성하는 캐시 폴더.
> `.gitignore` 영역. 지워도 다음 테스트 시 재생성됨.
> 신입이 backend 에서 pytest 돌리면 자기 PC 에도 생김.

## 7. 신입이 자주 묻는 것

> [!question] 왜 `_attic/` 같은 게 그대로 남아 있나?
> CLAUDE.md 의 명시 정책. 격리/보관소를 "정리하자" 식으로 손대면 운영 백업이 사라질 수 있음. 만지려면 사용자(현우)에게 먼저 묻는다.

> [!question] `.claude/` 가 다른 PC 에 없는데 망가진 거 아니에요?
> 아님. `.gitignore` 라 사람마다 다름. 본인이 Claude Code 쓰면 자기 PC 에 자동 생김.

> [!question] `_dev/baselines/openapi.json` 가 뭔가요?
> OpenAPI 스펙의 baseline 파일. backend 라우터/스키마가 바뀌면 이 파일도 갱신해야 CI 가 통과. [[erp/scripts/dev/verify_local.ps1]] 가 자동 검사.

> [!question] `vault/` 에 코드가 다 미러됐는데 왜 코드는 또 backend/ 에 있나요?
> vault 는 **분석 지도**지 코드 사본이 아니다. 실제 수정은 항상 `backend/`, `frontend/` 같은 원본에서만. [[바이브_코딩_컨텍스트]] 참조.

> [!question] 모든 폴더 다 봐야 하나요?
> Day 1 은 [[처음_읽는_사람]] + [[ERP_MOC]] 만 보면 충분. 폴더 지도는 ls 결과 보고 "이건 뭐?" 할 때 다시 오는 reference.
