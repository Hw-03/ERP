# defect-handling-for-operators.txt — 불량 처리 시스템 현장 설명서

> [!summary] 역할
> 작업자/담당자용 불량 격리 프로세스 가이드. 컴퓨터 기반 "어디서/언제/왜/누가/얼마나" 추적 시스템 소개.

## 1. 이 파일의 역할
- 기존 문제점: 불량 격리 기록 흐릿함, 누가 했는지 모름, 폐기/반품/분해 구분 없음, 다른 부서 결재 막힘
- 신규 시스템: [불량 격리] 단추 → 부서/품목/수량/사유 입력 → 바로 빠짐 (결재 불필요)
- 재고 화면: 불량 격리 "빨간 칸"으로 표시, 대시보드 "조립부 불량 5개" 강조
- 평소 업무(입고/생산/출고) 동일, "불량 발견 시" 한 가지 단추만 추가

## 2. 실제 원본 위치
`docs/defect-handling-for-operators.txt` — 약 60줄 (텍스트)

## 3. 주요 내용
- 불량 격리 절차: 어디서든 → [불량 격리] → 정보 입력 → 끝
- 결과: 정상 재고에서 제외, "불량 격리" 빨간 칸으로 이동
- 부서별 모니터링: 대시보드에서 한눈에 확인

## 4. 어디서 쓰이는지
- 현장 교육 자료
- 부서 게시판/인트라넷 공지
- 신입 온보딩

## 5. ⚠️ 위험 포인트
- **기술적 내용 없음** — 순수 업무 프로세스 설명만 (UI/API 명시 없음)
- 불량 후 처리 (폐기/반품/분해) 프로세스는 문서 미포함
- 다단계 승인 흐름 제거됨 — 감시 필요

## 6. 수정 전 체크
- 현장 담당자 읽음 확인 (가독성 평가)
- 불량 격리 단계별 스크린샷 추가 권장
