불량 처리 시스템 새로 만들어요 — 현장 설명서
============================================================
대상: 작업자·담당자분들
작성: 2026-05-21


■ 한 줄 요약
─────────────────────────────────────────────
지금처럼 불량 발견하면 그냥 빼두는 식 말고,
컴퓨터로 "어디서 / 언제 / 왜 / 누가 / 얼마나" 추적되게 만들 거예요.



■ 왜 이걸 만드나
─────────────────────────────────────────────
지금 불량 처리가 좀 두루뭉술해서요.

· 누가 격리해뒀는지 나중에 모름
· 진공부 같은 데는 1년씩 묵은 불량이 쌓여 있는데 안 보임
· 폐기인지 반품인지 분해 회수인지 기록이 흐릿
· 다른 부서 일은 결재 길이 막혀서 멈춤

이걸 다 정리하면, 적어도 "어디서 막혀 있나" "뭐가 얼마나 쌓였나"
는 한 화면에서 다 보이게 됩니다.



■ 평소 일하던 흐름은 그대로
─────────────────────────────────────────────
입고하고, 생산하고, 출고하고 — 이건 똑같아요.
바뀌는 건 "불량이 나왔을 때" 한 가지 단추를 더 누른다 정도.



■ 불량 발견하면 어떻게?
─────────────────────────────────────────────
어디서든 (입고 검수 / 생산 중 / 자체 검사 / 출고 직전)
불량이 보이면 똑같이 한 가지로 처리해요.

  → [불량 격리] 단추를 누른다
  → 부서, 품목, 수량, 사유를 적는다
  → 끝. 결재 안 받아도 됨, 바로 빠짐

그러면 그 부품은 "정상 재고"에서 빠지고
"불량 격리"라는 빨간 칸으로 옮겨갑니다.

대시보드에서도 빨간 글씨로 "조립부 불량 5개" 이런 식으로 떠요.
일하면서 흘리지 마시고, 발견하면 그 자리에서 격리만 해주세요.



■ 격리는 부담 없이, 일단 빼두세요
─────────────────────────────────────────────
"확실치 않은데 격리해도 되나?" 걱정 마세요.

· 잘못 격리했으면 → [정상 복귀] 한 번 누르면 다시 돌아옴
· 결재 없음, 즉시 처리됨

쌓아둬도 됩니다. 묵혀도 됩니다.
어차피 나중에 한꺼번에 처리할 거니까요.



■ 격리한 거 나중에 어떻게 처리하나
─────────────────────────────────────────────
[불량 처리 허브]라는 화면이 새로 생겨요.
거기 들어가면 격리된 게 다 보입니다.

부서별로 묶여서, 오래된 순서로, 1년 넘은 건 ⚠ 표시까지.

거기서 항목 하나 누르면 [처리] 단추가 있어요. 누르면:

  ◆ 원자재(부품 한 개짜리)인 경우
     - [정상 복귀] : 다시 써도 됨
     - [폐기]      : 그냥 버림
     - [공급처 반품]: 사 온 데로 돌려보냄
     ※ 폐기·반품은 부서장 결재 받습니다

  ◆ 완성품·조립품(여러 부품으로 만든 거)인 경우
     - 아래에 BOM(부품 목록)이 좌라락 펼쳐져요
     - 본인을 어떻게 할지: 복귀 / 전부 폐기 / 분해
     - 분해를 고르면, 자식 부품 하나하나마다
         "이거 살릴까 버릴까" 선택
     - 살린 부품은 정상 재고로 자동 입고
     - 버린 부품은 폐기 기록
     ※ 결재 한 번 받으면 한꺼번에 처리됨



■ "재작업"은 어디서 하나
─────────────────────────────────────────────
재작업이라는 별도 메뉴는 안 만듭니다.

이유: 작업하고 있는 동안에도 어차피 그 부품은 격리되어 있죠.
시스템이 굳이 "지금 손대고 있는 중"인지 알 필요 없어요.

재작업 다 끝나면, 그때 [처리]를 눌러서:
- 살려낸 부품들 → 정상 입고
- 못 살린 부품들 → 폐기

이런 식으로 결과만 기록하면 됩니다.



■ 원자재는 검수에서 바로 처리 OK
─────────────────────────────────────────────
사 온 부품(원자재)이 입고 검수에서 불량이면,
"굳이 격리부터 → 처리"까지 두 번 할 필요 없습니다.

- [공급처에 바로 반품]
- [바로 폐기]

이 두 단추가 따로 있어요. 한 번에 끝.
(단 사유는 적어야 하고, 부서장이나 창고장 결재는 받습니다)



■ 결재는 누가 봐주나
─────────────────────────────────────────────
튜브·고압·진공·튜닝·조립·출하 — 이 라인들에는 따로
부서장이 없죠. 다 생산부 소속이고, 이필욱 부장님·김건호
과장님이 통째로 봅니다.

근데 지금 시스템은 "조립부 일은 조립부 사람만 결재" 식으로
짜여 있어서, 다른 라인 결재가 이상하게 막히는 일이 있어요.
이걸 풀어드리는 거예요.

새로 바뀌면, 어느 라인 일이든 결재 가능한 분:

  · 이필욱 부장님 (생산부 정)        ◎ OK
  · 김건호 과장님 (생산부 부)        ◎ OK
  · 창고 정/부                       ◎ OK

이 중 누구 한 명이 결재만 누르면 처리됩니다.
한 분 자리 비셔도 다른 분이 봐주실 수 있어요.



■ 사유는 꼭 적어주세요 (제일 중요)
─────────────────────────────────────────────
모든 액션마다 사유를 받습니다.
귀찮으시겠지만, 이게 빠지면 시스템 만든 의미가 없어요.

· 종류 선택: 외관 불량 / 치수 불량 / 기능 불량 / 검사 통과 / 기타
· 보충 설명: 자유롭게 한 줄 (선택)

예) "스크래치 다수 / 우측 끝단"
    "치수 +0.05 over"
    "출력 부족, 재검사 시 통과 가능성 있음"

나중에 같은 종류 불량이 자꾸 나면 통계로 잡혀요.
"이번 달 외관 불량 30건, 어느 공정에서 많이 나오나"
이런 게 자동으로 정리됩니다.



■ 진공부 1년 묵은 거, 이제는 안 묻혀요
─────────────────────────────────────────────
허브 화면 맨 위에 [1년 이상] 카운트가 따로 뜹니다.
누르면 그 항목들만 추려져서 나와요.

쌓아둬도 잊혀지지 않게,
어쩌면 분기마다 "묵은 불량 처리의 날" 같은 운영도 가능해집니다.



■ 한 번 입력하면 다 따라옵니다
─────────────────────────────────────────────
격리든 폐기든 반품이든 분해든,
시스템에 한 번 누르면:

· 재고 수량이 자동으로 맞춰지고
· 입출고 내역에 기록이 남고
· 결재 이력이 남고
· 대시보드 빨간색이 줄고

따로 엑셀에 적거나 메모하실 필요 없어요.



■ 정리
─────────────────────────────────────────────
바뀌는 거 딱 4가지:

  1. 불량 보이면 [불량 격리] 누르고 사유 적기
  2. 격리해둔 거 나중에 [처리]에서 결과 입력
  3. 사 온 원자재 불량은 검수에서 바로 [반품] [폐기] 가능
  4. 결재가 생산부장·창고장으로 넓어져서 막힘 없음

나머지는 다 똑같습니다.
