■ 이번 주 진행 사항 (4/25~4/30)

**1. 입출고 화면 단계형(Wizard) UX 재구성**
기존에 한 화면에 모든 옵션이 몰려있던 입출고 화면을 단계별 진행 방식으로 재구성하였습니다.
좌측에 작업 설정, 우측에 품목 선택·수량 입력 영역을 명확히 구분하였고,
각 단계가 완료돼야 다음 단계로 넘어가는 방식으로 입력 실수를 줄였습니다.

**2. 창고 승인 기반 입출고 요청 흐름 구현**
생산부에서 입출고를 요청하면 창고 담당자가 확인·승인한 뒤 처리되는 흐름을 구현하였습니다.
기존에는 누구나 즉시 입출고를 처리할 수 있었으나, 이제는 요청→승인→처리 단계가 구분됩니다.
부서 간 재고 이동 시 책임 소재를 명확히 하고 실수를 방지하기 위한 구조입니다.

**3. 직원별 저장형 입출고 장바구니 구현**
담당자별로 진행 중인 입출고 작업을 저장해두는 장바구니 기능을 추가하였습니다.
화면을 이탈하거나 다른 작업을 하더라도 기존 작업 내용이 유지되어 이어서 처리할 수 있습니다.

**4. 작업자 PIN 로그인 및 수정 이력 관리**
직원별 PIN 번호로 시스템에 로그인하는 기능을 구현하였습니다.
직원 정보나 거래 내역이 수정될 경우 누가, 언제 수정했는지 감사 이력이 자동으로 기록됩니다.

**5. BOM 기능 강화**
자재명세서(BOM) 관리 화면을 단계형(Wizard) 방식으로 개선하였습니다.
또한 특정 자재가 어떤 완제품의 BOM에 사용되는지 역방향으로 조회할 수 있는 기능을 추가하였습니다.

**6. 코드 품질 대규모 정비 (Phase 5)**
시스템 내부 코드를 전면적으로 정비하였습니다.
- 데이터 조회 속도 최적화 (불필요한 중복 쿼리 제거)
- 오류 발생 시 응답 형식 전체 통일
- 테스트 자동화(CI) 도입 — 코드 수정 시 자동으로 오류 감지

**7. 시스템명 및 용어 정리**
시스템 내 표기를 실제 사내 환경에 맞게 전면 정리하였습니다.
X-Ray에서 DEXCOWIN MES로 명칭을 교체하고, 현장에서 사용하는 용어 기준으로 화면 문구를 통일하였습니다.

**8. 품목 분류 체계 개편 (진행 중)**
기존 category 방식 대신 공정 유형 코드(process_type_code) 기준으로 품목을 분류하는 방식으로 개편 작업을 진행하였습니다.
현재 백엔드·프론트엔드 전반에 걸쳐 교체 중이며 마무리 단계입니다.

**9. 버그 수정 및 기타**
- 직원 목록에서 일부 인원이 비활성 처리되던 버그 수정
- 한글 인코딩 깨짐 현상 수정
- 기존 MES 시스템과 현재 시스템의 차이점을 정리한 비교 분석 보고서 작성

화면은 제 자리 PC가 켜져 있을 때 http://192.168.0.63:3000 에서 확인하실 수 있습니다.
