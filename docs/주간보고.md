■ 이번 주 진행 사항 (5/1~5/8)

**1. 주간 재고 변화 보고 기능 신설**
공정별로 이번 주 재고가 얼마나 변했는지 한눈에 확인할 수 있는 주간보고 탭을 새로 추가하였습니다.
공정 단위로 입고·출고·순변화를 요약하여 카드 형태로 표시하고, 품목별 상세 수치는 테이블로 확인할 수 있습니다.
매주 보고 업무에 직접 활용할 수 있도록 화면 구성과 데이터 표시 방식을 다듬었습니다.

**2. 자동 테스트 강화 — 커버리지 91%**
코드 수정 시 오류를 자동으로 잡아내는 테스트를 추가하였습니다.
자동화 테스트 적용 범위(Coverage)가 87.6%에서 91.3%로 향상되었으며,
화면 JS 번들이 2MB를 초과하면 자동으로 알림이 뜨는 번들 크기 게이트도 추가하였습니다.

**3. 코드 정리 — 거대 파일 21개 분해**
하나의 파일에 기능이 과도하게 집중되어 있던 코드 파일 21개를 역할 단위로 분리하였습니다.
파일당 평균 줄 수가 300~550줄에서 150~270줄 수준으로 줄었으며,
향후 기능 수정 및 추가 작업이 더 쉬워졌습니다.

**4. 내부 라이브러리 경로 정비**
UI 색상·형식·부서 정보 등을 관리하는 내부 라이브러리 구조를 정비하였습니다.
기존에 중간 단계를 거쳐 불러오던 항목들을 직접 불러오도록 경로를 단순화하였고,
오래된 임시 wrapper 파일들을 제거하여 코드 복잡도를 낮췄습니다.

**5. 공통 UI 컴포넌트 정비**
화면마다 제각각 만들어 쓰던 버튼·카드·슬라이드 패널 등을 공통 컴포넌트로 추출하였습니다.
FilterChip, SlidePanel, KpiCard 등 3종의 공통 UI 컴포넌트를 신설하여 화면 간 일관성을 높였고,
색상 지정 코드가 50회 이상 반복되던 패턴을 헬퍼 함수로 통합하였습니다.

화면은 제 자리 PC가 켜져 있을 때 http://192.168.0.63:3000 에서 확인하실 수 있습니다.
