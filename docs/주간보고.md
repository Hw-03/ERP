■ 이번 주 진행 사항 (5/4·5/6·5/8)

**1. 자재 코드·품목명 1차 정비**
NAS에 보관되어 있던 자재 재고 현황 파일(품질경영 F704-03, 생산부 자재 재고 현황)을 기준으로
자재 코드 체계와 품목명을 정리하여 1차 안을 확정하였습니다. 현재 848개 품목 기준이며,
실제 사용 과정에서 보완이 필요한 부분은 수정 가능성이 남아있습니다.

**2. BOM 세팅 도구 개발 (다음 주 완성 예정)**
1차 정비된 자재 코드·품목명을 기반으로 BOM을 손쉽게 입력·편집할 수 있는
세팅 도구를 개발하였습니다. 마스터 엑셀과 동기화한 848개 품목을 검색하여 완제품의
하위 자재와 소요량을 등록할 수 있고, 진행도 저장·불러오기 및 일괄 다운로드를 지원합니다.
다음 주 마무리 후 실제 BOM 데이터를 입력할 예정입니다.

**3. BOM 관리자 화면 워크벤치 개편**
시스템 내부의 BOM 관리 화면을 워크벤치(workbench) 방식으로 전면 개편하여,
완제품 검색·하위 자재 조회·소요량 편집을 한 화면에서 처리할 수 있도록 하였습니다.
BOM 세팅 도구로 입력한 데이터를 시스템에서 그대로 활용할 수 있는 구조입니다.

**4. 자재(품목) 폼 편집 기능 강화**
관리자 화면에서 자재를 등록·수정할 때 ERP 코드, 옵션 코드, 모델 슬롯을 직접 편집할 수
있도록 폼을 확장하였습니다. 등록·수정 폼이 같은 컴포넌트(ItemFormFields)를 공유하도록
정비하여 입력 항목 누락이나 화면 간 차이가 발생하지 않도록 하였습니다.

**5. 부서간 재고 조정 4단계 위저드 (실험)**
부서 간 재고를 조정하는 흐름을 4단계 위저드(출발·도착 부서 → 품목 → 수량 → 확인)로
구현하여 시험 적용해 보았습니다. 입출고 화면 전체 재설계 시 실제 담당자 동선에 맞게
다시 다듬을 예정이며, 이번 주는 패턴 검증 차원의 작업입니다. 백엔드 API도 함께 신설하였습니다.

**6. 주간 재고 변화 보고 화면 신설**
공정별로 한 주간 재고가 얼마나 변했는지 확인할 수 있는 주간보고 탭을 추가하였습니다.
공정 단위 입고·출고·순변화 요약 카드와 품목별 상세 테이블, 달력 형태의 히스토리 보기를
함께 제공하여 매주 보고 업무에 바로 활용할 수 있도록 하였습니다.

**7. 시스템 안정화 — 자동 테스트·코드 정리**
앞으로 입출고 흐름과 BOM을 본격적으로 다듬으려면 시스템 자체가 안정적이어야 하므로,
이번 주 시간을 들여 내부 정비도 함께 진행하였습니다.
- 자동화 테스트 적용 범위(Coverage) 87.6% → 91.3% 향상
- 화면 JS 번들 2MB 초과 자동 감지 게이트 추가
- 한 파일에 기능이 몰려있던 코드 21개 분리 (300~550줄 → 150~270줄)
- 공통 UI 컴포넌트 5종(FilterChip·SlidePanel·KpiCard·EmptyState·LoadingSkeleton) 신설
- 색상·형식·부서 정보 내부 라이브러리 경로 단순화

**8. 모바일 화면 개편 (사전 작업)**
모바일 화면은 본격 사용은 가장 후순위지만, 시간 여유가 있을 때 미리 틀을 잡아두기 위해
하단 5탭(홈·재고·입출고·내역·메뉴) 구조와 입출고 허브(IO Hub) 골격을 만들어 두었습니다.
입출고 흐름 재설계가 끝난 후 실제 사용 환경에 맞춰 다시 정리할 예정입니다.

**9. 버그 수정 및 기타**
- BOM 검색창 한글 입력(IME) 시 글자 깨짐·포커스 이탈 현상 수정
- 창고 재고를 초과하는 출고 요청을 사전에 차단
- BOM 자식 후보 200건 컷오프 제거 (전체 자재 검색 가능)
- 로그인 화면 로고 인트로 애니메이션 적용
- 입출고 내역 달력 셀 높이 고정 및 배지 초기 로드 안정화
- 산출물 디렉토리 정리 — 개발현황 엑셀(`docs/개발현황.xlsx`)을 보고 문서 폴더로 이전

화면은 제 자리 PC가 켜져 있을 때 http://192.168.0.63:3000 에서 확인하실 수 있습니다.
