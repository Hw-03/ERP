■ 이번 주 진행 사항 (5/11·5/12 기준, 5/13~15 진행 중)

**1. 입출고 화면 v2 — 4단계 마법사 도입 (이번 주 핵심 작업)**
지난 주 보고드렸던 "입출고 화면·내역 흐름 재설계"에 본격적으로 착수하였습니다.
실제 담당자 작업 동선을 분석하여 작업유형(입고/출고/이동/조정) → 대상(부서·창고)
→ 품목·수량 → 확인의 4단계 마법사 구조로 재구성하였습니다.
- **작업유형 분기** — 입고/출고/이동/조정을 sub_type 기준으로 명확히 분리하여
  화면 곳곳에 같은 버튼·태그가 일관되게 표시되도록 정비
- **수량 보정(조정)** 흐름을 입고·출고 방향으로 분리하여 의도가 분명하게 드러나도록 개편
- **작성 화면(Compose) UX 정리** — 하단 네비게이션바 제거, 컬럼 분리, 입력 완료 시
  다음 단계 자동 진행
- **권한 가드** — 작업유형별 접근 가능 부서·역할 사전 차단
- **부서 입출고 + BOM 가드** — 부서 재고를 BOM 기준으로 검증

**2. 입출고 v2 안정성 — 중복 제출 방지(멱등성)**
네트워크 지연이나 더블 클릭으로 같은 입출고가 두 번 기록되는 문제를 막기 위해
요청 단위 식별자(client_request_id)를 도입하였습니다.
같은 요청은 서버에서 한 번만 반영되며, 충돌 시 사용자에게 적절한 메시지(409/503)를
표시합니다. API 명세(OpenAPI baseline)도 함께 갱신하였습니다.

**3. BOM 세팅 도구 — 실 데이터 입력 진행 (805건)**
지난 주 마무리한 BOM 세팅 도구로 실제 BOM 데이터를 입력하고 있습니다.
- 조립 + 진공 합본 698건 프리셋 임베드
- 고압 99건 추가 → 합본 796건 + DB 동기화
- 추가 튜닝(NF) 9건 → 현재 **805건** 진행
완료 후 시스템 BOM과 최종 동기화 예정입니다.

**4. 관리자 화면 전면 정비**
관리자 화면을 PIN 입력 → 8개 섹션 구조로 재정리하였습니다.
- BOM 관리 화면을 가로 분할 레이아웃으로 개편
- 대시보드 필터 정비 및 KPI 카드 컴팩트화
- 화면 전반의 정보 밀도와 가독성 개선

**5. 로그인 화면 단일 카드 재구성**
기존 3단계(부서 → 사번 → PIN) 로그인 화면을 한 화면에서 콤보박스 + PIN 입력으로
처리하도록 단순화하였습니다. 실제 작업자 동선 기준으로 가장 빠르게 들어올 수 있도록
정리한 것입니다.

**6. 셀렉트(드롭다운) 디자인 통일**
화면마다 제각각이던 드롭다운 컴포넌트를 공통 AppSelect로 통일하였습니다.
색상·간격·키보드 동작이 같은 규칙을 따르도록 정비하였고, 입출고 v2 마법사·관리자
화면·로그인 등 모든 곳에 일괄 적용하였습니다.

**7. 생산 가능 수량 정의 재정비**
완제품을 만들 수 있는 수량의 정의를 다음과 같이 명확히 하였습니다.
- **즉시 생산(immediate)** — 현재 부서 재고만으로 만들 수 있는 수량
- **최대 생산(maximum)** — 창고 재고까지 옮겨오면 만들 수 있는 수량
화면에서는 status를 4분기(생산 가능/일부 가능/창고 보충 필요/불가)로 시각화하였습니다.

**8. 개발 환경 정비**
- 운영 빌드와 개발 빌드의 디렉터리 분리(`.next-prod`) — 동시 실행 시 충돌 방지
- 개발 서버 재시작 헬퍼 추가
- Next.js 14의 CLI 인자 미지원 이슈를 next.config.js의 phase 분기로 해결
- `.dev/*.log` 등 개발 산출물 git 추적 제외

화면은 제 자리 PC가 켜져 있을 때 http://192.168.0.63:3000 에서 확인하실 수 있습니다.
