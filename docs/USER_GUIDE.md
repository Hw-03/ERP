# 사용자 가이드

DEXCOWIN ERP 데스크톱 셸(`/legacy`)을 처음 사용하는 현장 사용자를 위한 화면별 사용법.

## 공통

- 좌측 상단 **DEXCOWIN** 로고 옆 사이드바로 화면을 전환한다(대시보드 · 입출고 · 입출고 내역 · 관리자).
- 우측 상단 **새로고침** 버튼은 현재 화면 데이터를 다시 불러온다.
- 우측 상단 **테마 토글**(달/해 아이콘)로 다크/라이트를 전환한다.
- 우측 상단 **상태 pill**:
  - 초록 점: "방금 완료 …" — 직전 작업이 정상 처리됨
  - 파란 점: 일반 상태 메시지
  - 빨간 점: 오류 또는 데이터를 불러오지 못함
  - 노란 점: 재고 부족·주의

빈 상태(데이터 없음 / 검색 결과 없음 / 필터로 가려짐)와 로드 실패는 어느 화면에서나 같은 시각 언어로 안내된다.

## 1. 대시보드 (재고)

전체 재고 현황을 빠르게 파악하고 필요 시 조치로 이어가는 화면.

KPI 카드 순서: **전체 → 정상 → 부족 → 품절** (이 순서는 고정).

- KPI 카드를 누르면 해당 상태의 품목만 필터링된다.
- 부서 칩 / 모델 칩 / 분류 칩으로 좁힐 수 있다. 모든 칩이 OFF면 "전체"와 같은 결과가 나온다.
- "조치 필요" 카드는 부족·품절 등 즉시 보충이 필요한 항목을 모아 보여준다. 행을 누르면 곧바로 입출고 화면에서 해당 품목으로 이동한다.
- 결과가 0건이면 "필터 초기화" 버튼으로 한 번에 풀 수 있다.

## 2. 입출고 (단계형 wizard)

작업은 **5단계** 카드 흐름으로 진행한다. 각 단계가 끝나면 다음 카드가 자동으로 열린다.

### 1단계 · 담당자

- 본인 카드를 누른다. 부서별 색상이 지정되어 있다.
- 직원이 10명을 넘으면 "추가 N명" 카드로 펼친다.

### 2단계 · 작업 유형

| 작업 유형 | 의미 |
|---|---|
| 원자재 입출고 | 창고 기준 입고/출고 |
| 창고 이동 | 창고↔생산부서 이동 |
| 부서 입출고 | 생산부서 기준 입고/출고 |
| 패키지 출고 | 등록된 묶음 출고 |
| 불량 등록 | 불량 격리 처리 (되돌릴 수 없음) |
| 공급업체 반품 | 공급업체 반품 처리 (되돌릴 수 없음) |

- 이동 방향, 대상 부서, 불량 발견 위치 등 작업에 필요한 옵션이 추가로 나타난다.
- 빨간 톤으로 표시되는 작업은 되돌릴 수 없는 작업이다. **이 작업으로 진행 →** 버튼을 눌러야 다음으로 넘어간다.

### 3단계 · 품목 선택

- 부서 / 모델 / 분류 / 검색으로 좁히고 품목 행을 클릭해 선택한다.
- 한 번에 100개씩 노출되며 **100개 더 보기** 버튼으로 더 불러올 수 있다.
- 선택한 품목이 필터로 가려지면 노란 배너 "선택한 N건이 현재 필터로 가려졌습니다 · 필터 해제" 가 표시된다. 버튼을 누르면 모든 필터가 풀린다.
- 패키지 출고는 품목 대신 패키지 1건을 선택한다.

### 4단계 · 수량 · 메모

- 각 행에서 ±1 / ±10 버튼이나 직접 입력으로 수량을 조정한다.
- "현재 재고" 옆에 "실행 후 재고"가 함께 표시된다. **출고 시 재고가 음수가 되면** 행이 빨강 배경 + "재고 부족" 라벨로 강조된다.
- 메모는 200자 권장. 200자를 초과하면 카운터와 입력란 테두리가 빨강으로 변한다(저장은 가능).

### 5단계 · 실행

- 화면 위에 노란 안내가 떠 있으면 그 사유부터 해결한 뒤 실행 버튼이 활성화된다.
  - 출고 후 재고가 음수면 "출고 후 재고가 음수입니다 — 수량을 다시 확인하세요" 가 표시되어 한 번 더 점검하도록 안내한다.
- 실행 버튼을 누르면 최종 확인 팝업이 뜬다. **처리 중에는 배경 클릭이나 ESC로 닫히지 않는다(중복 실행 방지)**.
- 처리 결과:
  - 모두 성공 → 화면 중앙에 "입고 완료 / 출고 완료" 카드가 잠깐 떴다 사라지고, 데이터가 자동 새로고침된다. 우측 상단 pill 이 초록 "방금 완료 …" 로 바뀐다.
  - 일부 성공 → 노란 결과 모달에 성공 N건/실패 M건이 나오며 **실패 항목만 다시 시도** 버튼이 제공된다. 성공한 건은 이미 처리되어 있으므로 재시도해도 이중 처리되지 않는다.
  - 모두 실패 → 빨간 결과 모달에 사유가 나오며 **재시도** 버튼이 제공된다.

## 3. 입출고 내역

조회 / 추적 / 감사용 화면.

- 기본 노출 필터: **검색 · 기간 · 유형**. 그 외 필터는 화면을 깔끔하게 두기 위해 기본 숨김 처리되어 있다.
- 달력 뷰 / 목록 뷰를 토글로 전환할 수 있다. 달력에서 날짜를 누르면 해당 일자 거래 목록이 옆에 펼쳐진다.
- 거래 행을 누르면 우측에 상세 패널이 열려 처리 전·후 재고, 담당자, 메모, 참조 번호를 보여준다.
- 데이터가 없거나 조건에 맞는 거래가 없으면 친절한 빈 상태가 안내된다.

## 4. 관리자

기준정보·구성관리·시스템을 한곳에 모은 화면. 각 섹션은 PIN 잠금 또는 별도 안내가 있을 수 있다.

- **기준정보**: 품목, 직원, 모델, BOM, 출하 묶음(패키지) 관리
- **구성관리**: BOM 등록은 단계적으로 진행. 단일 화면이 아닌 자재 추가 → 수량 조정 흐름.
- **위험 영역(삭제·초기화·PIN 변경)**: 시각적으로 분리되며 확인 절차를 거친 뒤에만 진행된다. **위험 영역은 본 가이드의 범위를 넘어서며 운영자(관리자)만 사용해야 한다.**
- 빈 상태(예: BOM이 없을 때)는 일관된 안내 카드로 표시된다.

## 자주 묻는 상황

| 상황 | 어떻게 |
|---|---|
| 화면이 깨끗한데 무엇을 해야 할지 모르겠다 | 사이드바에서 **대시보드** 로 이동 → KPI 또는 조치 필요 카드를 따라간다. |
| 검색해도 결과가 없다 | 화면이 "필터로 가려졌습니다 · 필터 초기화" 를 보여주는지 확인한다. |
| 출고 실행 버튼이 비활성이다 | 5단계 위 안내문이 사유를 보여준다. 보통 담당자/품목/수량/재고 부족 중 하나다. |
| 일부만 처리되었다고 나온다 | 결과 모달의 "실패 항목만 다시 시도" 를 누르면 된다. 성공분은 이미 반영되었다. |
| 데이터를 불러오지 못했다 | 빨간 띠 안내의 "새로고침" 을 누른다. 그래도 안 되면 운영자에게 알린다. |
