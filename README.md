# ERP 자재 마스터 DB 통합 프로젝트

정밀 X-ray 발생 장치 제조사의 **부서별로 파편화된 엑셀 자재 파일**을 분석하여
**단일 표준 마스터 DB**로 통합하고, 향후 **SQL 기반 ERP 시스템**으로의 이관을 준비한다.

## 프로젝트 개요

원자재(RM) → 튜브(TA/TF) → 고압(HA/HF) → 진공(VA/VF) → 조립(BA/BF) → 최종 완제품(FG) 의
**4개 공정 BOM 구조**를 가진 제조 환경에서, 부서별로 파편화된 엑셀 자재 파일을
하나의 통합 마스터로 정규화한다.

### 입력 파일

| 파일 | 설명 | 기준점 | 메인 시트 |
|---|---|---|---|
| **A**: `F704-03 (R00) 자재 재고 현황.xlsx` | 원자재 마스터 (전체) | ✅ Baseline | `26.03월` |
| **B**: `2026.03_생산부 자재_조립,출하파트.xlsx` | 조립/출하팀 자재 |   | `조립 자재` |
| **C**: `2026.03_생산부 자재_고압,진공,튜닝파트.xlsx` | 고압/진공/튜닝팀 자재 (튜브 공정 자재 포함) |   | `고압` (부서 컬럼 + 분류 컬럼으로 4개 공정 구분) |

> 파일 C의 `부서` 컬럼에는 `고압`/`진공`/`튜닝`만 있고 `튜브` 부서는 없지만,
> `분류` 컬럼에 `튜브`/`튜브 A,SSY`가 존재하여 튜브 공정 자재가 식별된다.

## 산출물

| 파일 | 설명 |
|---|---|
| `erp_integration.py` | 통합 스크립트 (로딩/정규화/매칭/카테고리/출력) |
| `ERP_Master_DB.csv` | **통합 마스터 DB** (items 테이블) |
| `ERP_Source_Links.csv` | 마스터 항목 ↔ 엑셀 원본 행 매핑 (item_source_links 테이블) |
| `ERP_Integration_Report.md` | 통합 결과 리포트 (통계, 샘플) |
| `ERP_Unmatched_A_Items.csv` | 파일 A 미매핑 항목 (수동 검토용) |
| `ERP_Excluded_Items.csv` | 비활성 부서(`구버전`/`미사용`/`사용중`) 항목 (마스터에서 제외, 감사용) |

3개의 엑셀 파일은 스크립트와 동일한 디렉터리에 있어야 한다.

## 카테고리 코드

| 코드 | 설명 | 판정 규칙 |
|---|---|---|
| `RM` | 원자재 (Raw Material) | 파일 A에 존재하는 모든 항목 |
| `BA` | 조립 반제품 (Body Assembly Sub) | 파일 B 단독, ASS'Y 키워드 없음 |
| `BF` | 조립 완제품 (Body Assembly Final) | 파일 B 단독, ASS'Y 키워드 포함 |
| `HA` | 고압 반제품 (High-voltage Sub) | 파일 C 단독, 부서=고압 (튜브 제외) |
| `HF` | 고압 완제품 (High-voltage Final) | 파일 C 단독, 부서=고압, ASS'Y 키워드 |
| `VA` | 진공 반제품 (Vacuum Sub) | 파일 C 단독, 부서=진공/튜닝 |
| `VF` | 진공 완제품 (Vacuum Final) | 파일 C 단독, 부서=진공/튜닝, ASS'Y 키워드 |
| `TA` | 튜브 반제품 (Tube Sub) | 파일 C 단독, 분류=`튜브` 또는 튜브 부품 키워드 (`튜브 고정`, `하우징 파이프`, `고압보드 SMT`, `고압베어B/D` 등) |
| `TF` | 튜브 완제품 (Tube Final) | 파일 C 단독, 분류=`튜브 A,SSY` 또는 튜브 어셈블리 |
| `FG` | 최종 완제품 (Finished Goods) | 이름이 회사 제품 모델명과 동일 |

> **비활성 부서 분리**: 파일 C에서 부서가 `구버전`/`미사용`/`사용중`인 항목(7건)은
> 매칭/마스터 DB에서 제외되고 `ERP_Excluded_Items.csv` 로 별도 보관됨 (감사용).

## 매칭 로직

파일 A(원자재 마스터)의 각 품목을 파일 B/C의 품목과 대조하여 같은 부품인지 판정한다.

**정규화**: 유니코드 NFKC → 소문자 → 공백/특수문자/괄호/언더스코어 제거

**점수 기반 매칭** (가중치가 누적되어 임계값 이상이면 매칭):
1. **MAKER 품번 매칭** (+6): 파일 A의 MAKER 품번 전체가 B/C 품목명에 포함
2. **MAKER 품번 토큰 매칭** (+3 ×토큰수)
3. **규격 매칭** (+5): 파일 A의 규격 전체가 B/C 품목명에 포함 (강력한 차별화 신호)
4. **규격 토큰 매칭** (+3 ×토큰수)
5. **품명 매칭** (+2~3): 파일 A의 품명이 B/C 품목명에 포함
6. **품명 토큰 매칭** (+1~2 ×토큰수)
7. **역방향 매칭** (+2): 짧은 B/C 이름이 A의 품명+규격+MAKER 조합에 포함

**임계값**: 3점 이상만 매칭으로 인정 (`SCORE_THRESHOLD = 3`).

### 매칭 예시

| 파일 A 품명 | 파일 A 규격 | 파일 B/C 품목 | 매칭? |
|---|---|---|---|
| `고압 다이오드` | `DL1500` | `DL1500(다이오드)` | ✅ spec 매칭 |
| `TUBE` | `60KV D-081B` | `캐논튜브 [D-081B] _60KV` | ✅ spec 토큰 `D-081B`, `60KV` |
| `네온램프` | `NEON-5T.5*13 red` | `네온램프(NEON-5T)` | ✅ 품명 + 스펙 토큰 |
| `시멘트저항` | `RK92-11XD 307J` | `RK92-11XD307J (시멘트 저항)` | ✅ MAKER + 품명 |

## 파일 A 미매핑 항목의 성격

미매핑 94건 중 대부분은 **X-ray 튜브 공정의 내부 부품 / BOM 하위 원자재**로,
파일 B/C에 독립 항목으로 등록되어 있지 않은 것이 정상이다:

- 전극 (70kV / 100kV)
- 텅스텐 필라멘트
- 게터 / 에노드 / 포커서
- 세라믹 바디 / 세라믹 스템 / 세라믹 슬리브
- 필러 / 전극 필러

이들은 파일 C의 `캐논튜브`, `세라믹튜브`, `CEI튜브` 등 튜브 공정 완제품(`TF` 카테고리)
또는 파일 A의 RM `TUBE` 항목의 하위 구성 부품이며, 향후 SQL ERP 이관 시
`bom_lines` 테이블에 부모-자식 관계로 등록하여 해소할 예정.

## 향후 SQL ERP 이관

`schema.sql`에 PostgreSQL 기준 스키마를 정의해 두었다:

| 테이블 | 설명 |
|---|---|
| `items` | 자재 마스터 (CSV 1:1 매핑) |
| `item_source_links` | 엑셀 원본 행 추적 (감사용) |
| `product_models` | 회사 제품 모델 마스터 (DX3000, ADX6000 등) |
| `bom_headers` / `bom_lines` | BOM (자재 명세서) 버전 관리 |
| `stock_snapshots` | 월별 재고 스냅샷 이력 |

**뷰**:
- `v_current_bom` - 현재 유효 BOM
- `v_stock_summary_by_category` - 카테고리별 재고 요약

CSV를 `\COPY` 명령으로 적재하는 방법도 스키마 파일 하단에 예시로 포함.
